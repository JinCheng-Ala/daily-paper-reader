## 摘要
利用预训练策略、基础模型或人类操作员的先验知识，为从头学习机器人技能提供了一种高效的替代方案。然而，这些智能体提供的动作往往是次优的、带噪声的，或与特定任务的专家行为不一致。我们提出GLOVES，一系列基于流的自适应方法，通过将非专家动作向专家动作分布迁移来校正它们。GLOVES并非用完全自主性替代智能体控制，而是执行选择性动作级自适应，在保持智能体意图的同时提升任务成功率。学习到的流还通过反向流评估提供了一种自然的分布内评分机制。我们将此信号用作干预门控：与专家分布一致的动作保持不变，异常或分布外（OOD）动作则被校正。这样，仅在必要时提供辅助。GLOVES仅需有限的专家监督，使用少量演示或可重复利用的成功技能片段。通过学习局部专家动作模式并在执行过程中拼接，GLOVES提供了一种轻量级的共享控制模块，用于跨任务和环境的鲁棒动作自适应。代码和演示见ripl.github.io/GLOVES_web。

## Abstract
Leveraging prior knowledge from pretrained policies, foundation models, or human operators offers an efficient alternative to learning robot skills from scratch. However, these agents often provide actions that are suboptimal, noisy, or misaligned with task-specific expert behavior. We propose GLOVES, a family of flow-based adaptation methods that correct non-expert actions by transporting them toward an expert action distribution. Rather than replacing agentic control with full autonomy, GLOVES performs selective action-level adaptation, improving task success while preserving agent intent. The learned flow also provides a natural in-distribution scoring mechanism through reverse flow evaluation. We use this signal as an intervention gate: actions that appear consistent with the expert distribution are passed through unchanged, while anomalous or out-of-distribution (OOD) actions are corrected. In this way, assistance is only provided when necessary. GLOVES requires only limited expert supervision, using a small number of demonstrations or reusable successful skill segments. By learning local expert action patterns and stitching them during execution, GLOVES provides a lightweight shared-control module for robust action adaptation across tasks and environments. Code and demos are available at ripl.github.io/GLOVES_web.

# 基于流的策略自适应无需策略更新（GLOVES）论文总结

## 1. 论文的核心问题与整体含义（研究动机和背景）

现代机器人系统常依赖预训练策略、人类操作员或视觉-语言-动作模型（VLA）等智能体提供的先验知识。然而，这些智能体的动作往往存在噪声、偏差、延迟或无效（如由于领域偏移），直接零样本应用会降低任务成功率。传统解决方案是使用专家演示或强化学习进行微调，但微调并不总是可行或理想：智能体可能是人类，VLA仅能通过黑盒API访问，演示数据难以大规模收集，奖励难以设计，更新大型模型成本过高。因此，论文提出 **在运行时自适应智能体的动作，而不更新智能体本身**，即动作级自适应。GLOVES（一系列基于流的自适应方法）被提出，通过从有限专家演示中学习专家动作分布，将非专家动作“运输”到该分布，同时在必要时选择性干预，保留智能体意图。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 学习一个条件专家流（conditional flow），将高斯噪声映射为专家动作块（action chunk）。
- 在推理时，智能体提出动作块，GLOVES通过 **反向流评分（reverse flow score）** 判断该块是否处于专家分布内（OOD检测）。
- 若处于分布内，则直接执行；若为OOD，则使用三种编辑方法之一进行校正，最终得到修正后的动作块。

### 关键技术细节

#### 2.1 条件流匹配（Conditional Flow Matching）
- 使用 **整流流（rectified flow）** 设置：线性插值路径 \(x_t = (1-t)x_0 + t x_1\)，速度场 \(u_t = x_1 - x_0\)。
- 训练时，神经网络 \(v_\theta(x_t, t | c)\) 预测真实速度，最小化 \(\mathbb{E}[\|v_\theta((1-t)x_0+tx_1, t | c) - (x_1 - x_0)\|_2^2]\)，其中 \(c\) 为图像观测条件。

#### 2.2 OOD检测（Out-of-Distribution Detection）
- 利用流的可逆性：对于测试动作块 \(a_u\)，反向积分得到潜在变量 \(z = F_\theta^{-1}(a_u)\)，其先验负对数似然 \(\|z\|_2^2\) 作为OOD分数。
- 近似方法：采用单步反向欧拉估计 \(\hat{z} \approx a_u - v_\theta(a_u, 1)\)，结合 conformal prediction 确定阈值。

#### 2.3 三种动作编辑方法

1. **FPAS（Flow-Prior Action Sampling）**  
   - 以智能体动作 \(a_u\) 为初始点，迭代采样局部扰动，利用OOD分数和接近原始动作的正则项进行加权更新，找到既接近专家分布又保留意图的动作。

2. **FEEG（Flow Editing with Energy Guidance）**  
   - 从智能体动作的噪声版本开始（\(x_t = t a_u + (1-t)\epsilon\)），沿专家流积分，并加入能量引导项 \(-\lambda_{\text{align}} \nabla_{x_t} E(\hat{x}_1)\)，其中 \(E(x_1) = \frac{1}{2}\|x_1 - a_u\|_2^2\) 鼓励终端动作接近原始智能体动作。

3. **IFAE（Inversion-Free Action-Chunk Editing）**  
   - 同时训练两个流：智能体流（\(v_u\)）和专家流（\(v_e\)）。编辑状态 \(a^{(k)}\) 从 \(a_u\) 向专家分布运输，通过配对中间样本 \(q_k\)（智能体流）和 \(p_k\)（专家流），结合速度差和源一致性正则进行更新。无需显式反转噪声。

- 所有方法均支持OOD门控：仅当动作块超出专家分布时进行编辑，否则直接通过。

## 7. 优点：方法或实验设计上的亮点

### 方法优点
- **无需更新智能体**：完全在推理时进行动作级自适应，适用于不可微、黑盒或人类操作员。
- **统一的OOD检测与校正**：使用同一个学习到的专家流完成两个任务，无需训练额外模块。

- **轻量级**：仅需少量专家演示，模型规模适中（ResNet-18 + DiT），适合实际部署。

## 8. 不足与局限

- **变体选择不明确**：FPAS、FEEG、IFAE在不同任务上的优势缺乏系统性解释，哪些因素（如动作空间维度、任务接触特性）决定最佳变体尚待研究。
- **依赖专家分布的可建模性**：IFAE需要训练智能体流，当智能体动作分布复杂或高度退化时可能失效。
