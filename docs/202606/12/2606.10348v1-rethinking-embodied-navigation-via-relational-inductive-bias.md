## 摘要
目标导航要求智能体通过视觉观察在未知环境中定位目标。现有方法通常依赖开放词汇检测器或视觉语言模型（VLM）来回答“去哪里搜索”，但往往忽视“什么不可信”——即哪些语义线索不可靠。开放词汇感知容易受到系统性误导证据的影响：假阳性、过时的静态先验以及由于缺乏具身验证而导致的重复失败探索，这会污染地图构建和决策制定。此类错误根植于真实场景中的结构化对象关系。为解决此问题，我们提出DB-Nav，一种通过双重关系偏置重塑搜索空间的框架。它将目标中心关系分解为激活偏置（传播上下文证据）和抑制偏置（通过感知混淆和动作级证伪抑制不可靠区域）。这些偏置统一为关系激活-抑制探索图，利用在线观测和失败访问来调节前沿探索值。在ObjectNav基准上的实验表明，DB-Nav在成功率（SR）和路径长度加权成功率（SPL）上显著优于现有方法，提供了一个轻量级、可解释且稳健的导航框架，无需昂贵的在线VLM推理。

## Abstract
Object navigation requires an agent to locate a target in an unknown environment through visual observations. Existing methods typically rely on open-vocabulary detectors or vision-language models (VLMs) to answer where to search, but often overlook what not to trust - which semantic cues are unreliable. Open-vocabulary perception is prone to systematic misleading evidence: false positives, outdated static priors, and repeated failed exploration due to lack of embodied verification, which contaminates mapping and decision-making. Such errors are rooted in structured object relations in real-world scenes. To address this, we propose DB-Nav, a framework that reshapes the search space via dual relational biases. It factorizes target-centric relations into an Activation Bias (propagates contextual evidence) and an Inhibition Bias (suppresses unreliable regions via perceptual confusion and action-level falsification). These biases are unified into a Relational Activation-Inhibition Exploration Graph that modulates frontier exploration values using online observations and failed accesses. Experiments on ObjectNav benchmarks show that DB-Nav significantly outperforms existing methods in success rate (SR) and Success weighted by Path Length (SPL), offering a lightweight, interpretable, and robust navigation framework without costly online VLM reasoning.

- **研究对象**：具身目标导航（ObjectNav），要求智能体在未知环境中通过视觉观察逐步定位目标物体。
- **现有方法局限**：
  - 主流方法依赖开放词汇检测器或视觉语言模型（VLM）来回答“去哪里搜索”（where to search），但忽视了另一个同等关键的问题：“什么不可信”（what not to trust）。
  - 开放词汇感知产生的语义线索常包含系统性误导证据：例如视觉相似性导致假阳性（如将椅子误检为沙发）、静态先验一旦被观测矛盾后难以更新、缺乏具身体验验证导致重复失败探索——这些错误会不断污染地图更新和导航决策。
- **根本原因**：此类错误并非随机噪声，而是根植于真实场景中对象关系的结构化偏差。目标出现和混淆都与对象关系紧密相关。
- **核心观点**：可靠的具身搜索应动态利用正向关系线索（哪些区域值得探索），同时抑制负向关系线索（哪些区域的证据不应过度信任），而非将所有语义证据视为同等有益。

## 2. 方法论

### 核心思想
- 提出 **DB-Nav**（Dual Bias Navigation）框架，通过 **双关系归纳偏置**（dual relational biases）动态重塑导航搜索空间：
  - **激活偏置（Activation Bias）**：基于目标与上下文物体的共现关系，将正向搜索证据传播到有希望的区域。
  - **抑制偏置（Inhibition Bias）**：通过显式建模感知混淆（视觉上相似的目标）和动作级证伪（访问后未发现目标），抑制不可靠区域。
- 双偏置统一为一个 **关系激活–抑制探索图**（Relational Activation–Inhibition Exploration Graph），将在线视觉观测和失败访问事件转化为语义源，动态调节候选前沿（frontier）的探索值。

### 关键技术细节

1. **目标中心关系偏置构建**
   - 为每个目标相关类别 ℓ 分配三维角色向量 b_g(ℓ) = [c_g(ℓ), a_g(ℓ), i_g(ℓ)]^T ∈ {0,1}^3：
     - c_g(ℓ)=1：目标/同义词 → 提供目标承诺证据（commitment）
     - a_g(ℓ)=1：上下文共现物体 → 提供正向激活（activation）
     - i_g(ℓ)=1：视觉相似干扰物 → 提供负向抑制（inhibition）
   - 例如：目标类别为[1,1,0]^T，上下文为[0,1,0]^T，干扰物为[0,0,1]^T。

2. **多标签对象-角色竞争**
   - 在线感知输出包含位置、置信度和类别标签。同一物理物体可能在不同视角被赋予不同标签，因此DB-Nav维护每个物体簇的多标签证据。
   - 计算每个标签的可靠性得分 ρ_o(ℓ) = m_o(ℓ)·q_o(ℓ)（累计观测质量和融合置信度）。
   - 通过多标签竞争选择主导标签 ℓ*_o = argmax ρ_o(ℓ)，并将其转换为对象级证据强度 r_o = σ(ρ_o(ℓ*_o))。
   - 最终得到对象级关系证据 h_o = r_o·b_g(ℓ*_o) = [h^c_o, h^a_o, h^i_o]^T（承诺、激活、抑制证据）。

3. **关系激活–抑制传播**
   - 在每个决策步骤构建局部关系图 G_t = (V_t, E_t)，包含：对象证据节点 O_t、上下文假设节点 H_t、负面证据节点 Y_t、候选前沿 F_t。
   - 使用距离衰减核函数 κ(p_v, p_u) 进行局部传播（半径 R）。
   - **激活值** A_t(f) = 直接目标证据传播 + 上下文假设传播。
   - **抑制值** I_t(f) = 干扰物抑制证据传播 + 失败访问负面证据传播。

4. **偏置条件化探索与目标承诺**
   - 前沿 f 的更新分数：Ẽ_t(f) = (基础语义响应 S0_t(f) + A_t(f)) / (1 + I_t(f) + 归一化路径代价 C̄_t(f))。
   - 选择最大化更新分数的前沿作为下一探索目标。
   - 目标承诺门：当对象级承诺证据 h^c_o 超过阈值 θ 时，触发目标承诺（直接导航至该对象）；否则继续探索。

### 算法流程（文字说明）
- 输入目标类别 → 构建目标中心角色向量 → 在线感知获取对象观测 → 多标签竞争确定每个对象的角色与证据强度 → 在图结构中传播激活和抑制 → 更新所有前沿分数 → 若承诺证据足够则切换到目标追逐，否则探索最优前沿 → 遇到失败访问事件则生成/增强负面证据节点 → 重复直至成功或达到步数上限。

## 8. 不足与局限
