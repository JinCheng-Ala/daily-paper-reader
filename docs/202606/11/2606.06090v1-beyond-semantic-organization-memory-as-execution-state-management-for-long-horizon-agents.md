## 摘要
基于LLM的智能体越来越多地处理具有相互依赖决策的长周期任务，其中每个动作都会重塑未来的约束条件，中间错误可能级联放大。现有的RAG和智能体记忆系统通过语义相似性组织历史记录，在决策时检索内容相关的条目。我们认为这种设计与执行状态依赖关系不匹配：它割裂了决策轨迹，混合了有效和错误的轨迹，阻碍了连贯的状态重建和错误隔离。我们提出MAGE（记忆作为智能体引导的探索），一种主动的执行状态管理器，将交互存储在层次化的状态树中。智能体从活跃的根到当前路径派生其状态，结合子目标摘要、最近轨迹以及来自先前分支的提示。四个耦合操作维护这棵树：Grow记录新轨迹，Compress总结完成的子目标，Maintain验证摘要，Revise恢复目标边界并在新分支上继续。这种设计在保持状态完整性的同时约束上下文增长，并将有缺陷的片段与活跃路径隔离。在MemoryArena上的实验表明，与基线相比，MAGE将平均任务成功率提高了7.8-20.4个百分点，同时将token消耗减少了55.1%。

## Abstract
LLM-based agents increasingly tackle long-horizon tasks with interdependent decisions, where each action reshapes future constraints and intermediate errors can cascade. Existing RAG and agent memory systems organize histories by semantic similarity, retrieving content-relevant entries at decision time. We argue that this design mismatches execution-state dependencies: it fragments decision trajectories and mixes valid and erroneous traces, hindering coherent state reconstruction and error isolation. We propose MAGE (Memory as Agent-Guided Exploration), an active execution-state manager that stores interactions in a hierarchical state tree. The agent derives its state from the active root-to-current path, combining subgoal summaries, recent traces, and hints from prior branches. Four coupled operations maintain the tree: Grow records new traces, Compress summarizes completed subgoals, Maintain validates summaries, and Revise restores a target boundary and resumes on a new branch. This design bounds context growth while preserving state integrity and isolating flawed segments from the active path. Experiments on MemoryArena show that MAGE improves the average task success rate by 7.8--20.4 pp over baselines, while reducing token consumption by 55.1%.

## 1. 论文的核心问题与整体含义

- **研究动机**：基于LLM的智能体在执行长周期任务（如购物、旅行规划、搜索、数学推理）时，决策相互依赖，每个动作都会改变未来约束，中间错误会级联放大。现有记忆系统（包括RAG和智能体记忆系统）普遍按照**语义相似性**组织历史信息，在决策时检索与当前查询内容相关的条目，而非反映执行状态的因果依赖关系。
- **核心问题**：这种设计导致两个严重缺陷：
  - **状态碎片化**：将决策轨迹割裂为语义片段，无法重建完整的执行状态链。
  - **错误隔离困难**：有效和错误的轨迹在相同的语义空间中混合，错误无法追溯源头，导致污染后续推理。
- **整体含义**：本文提出记忆应当从被动的事实存档转变为**主动的执行状态管理器**，从而在保持上下文紧凑的同时维护状态完整性和错误隔离能力。

## 2. 论文提出的方法论

### 核心思想
- 将智能体的交互历史组织为一棵**两层层次化的状态树**，树中根到当前节点的路径天然构成完整的执行状态。
- 智能体通过**四个耦合操作**主动维护这棵树，形成闭环的状态管理循环。

### 关键技术细节

#### 层次状态树结构
- **底层（Bottom Layer）**：记录每个动作-观察对（action-observation）作为原始节点，按执行顺序链接，保存细粒度状态。
- **顶层（Top Layer）**：压缩已完成的子目标（subgoal），存储摘要节点，每个顶层节点覆盖一段连续的底层节点。
- 运行时，智能体的执行状态 $S$ 由三部分构成：
  - **压缩状态 $C$**：根到当前顶层节点的摘要链。
  - **原始状态 $R$**：最近未压缩的少量底层节点。
  - **执行提示 $H$**：当前节点和父节点的子节点信息（历史替代分支）及诊断反馈。

#### 四个状态转换操作

| 操作 | 功能 | 认知对应 |
|------|------|----------|
| **Grow** | 执行新动作时追加底层节点，若与历史分支相同则复用节点，避免重复。更新 $R$ 和 $H$。 | 扩展轨迹 |
| **Compress** | 在子目标完成或 $R$ 超长时，将最近的底层段压缩为顶层摘要节点，清空 $R$，更新 $C$ 和 $H$。保留决策边界，便于后续回滚。 | 分块压缩 |
| **Maintain** | 每次压缩后立即用LLM验证刚生成的摘要及其覆盖的底层节点是否符合任务指令。若通过则继续；若失败，记录诊断信息 $f$ 并返回失败信号（含目标步 $pt.id$）。 | 错误监控 |
| **Revise** | 当检测到错误时，将顶层和底层指针回滚到目标边界，清除失败分支，保留有效前缀。将诊断信息 $f$ 注入 $H$，使新分支避免重蹈覆辙。 | 选择性纠错 |

## 6. 论文的主要结论与发现

1. **性能显著提升**：MAGE在所有领域上平均SR超过所有基线，比长上下文提升7.8-20.4个百分点（具体领域差异：Web Shopping 6.0pp，Travel Planning 10.0pp，Web Search 8.1pp，Formal Reasoning部分持平或略高）。
2. **token效率更高**：MAGE比长上下文平均减少55.1%的token消耗（32.9%-71.4%），且比多数记忆系统更省，因为辅助LLM调用仅在压实时发生。
3. **状态完整性是关键**：消融实验证明，去掉Compress、Maintain或Revise都会显著降低SR，说明边界感知压缩、验证和纠错三者缺一不可。
4. **错误隔离有效**：通过树结构的修订机制，错误分支被隔离，有效前缀保留，防止污染后续决策。
