## 摘要
AI智能体通常使用任务成功率、奖励、延迟和成本进行评估。这些指标虽然有用，但常常忽略了智能体行为的重要方面：智能体是否过度探索、是否过于刻板地重复、是否有效使用工具、是否随时间减少不确定性、或在重复运行中是否保持鲁棒性。本文提出了基于熵的AI智能体评估(EEA)，这是一种通过熵度量智能体行为的轻量级框架。EEA并非将智能仅视为最终任务完成，而是研究智能体决策过程的结构。该框架引入了动作熵、轨迹熵、工具熵、信息增益、探索效率和鲁棒性熵。这些指标旨在补充而非取代传统评估方法。我们还提供了一个实用的Python实现，旨在与LangChain、Google ADK、自定义智能体循环以及存储的可观测性追踪等智能体框架集成。

## Abstract
AI agents are commonly evaluated using task success, reward, latency, and cost. These metrics are useful, but they often miss important aspects of agent behavior: whether an agent explores too much, repeats itself too rigidly, uses tools effectively, reduces uncertainty over time, or remains robust across repeated runs. This paper proposes Entropy-Based Evaluation of AI Agents (EEA), a lightweight framework for measuring agent behavior through entropy. Rather than treating intelligence as only final task completion, EEA studies the structure of the agents decision process. The framework introduces action entropy, trajectory entropy, tool entropy, information gain, exploration efficiency, and robustness entropy. These metrics are intended to complement, not replace, traditional evaluation methods. We also present a practical Python implementation designed to integrate with agent frameworks such as LangChain, Google ADK, custom agent loops, and stored observability traces.

## 2. 论文提出的方法论

### 核心思想
将智能体的一次运行表示为事件序列（工具调用、模型调用、规划步骤等），然后基于信息论中的熵概念，计算一系列行为指标，刻画行为多样性、策略变化、不确定性减少等特征。

### 六种熵指标

| 指标名称 | 定义与公式 | 含义 |
|----------|------------|------|
| **动作熵（Action Entropy）** | \( H_A = -\sum_i p(a_i) \log_2 p(a_i) \)，其中 \( a_i \) 为动作 | 度量动作选择的多样性：低熵表示集中（可能刻板），高熵表示多样化（可能混乱） |
| **轨迹熵（Trajectory Entropy）** | 基于完整策略序列的分布计算熵 | 度量不同运行间策略的多样性：低熵表示路径固定，高熵表示策略丰富 |
| **工具熵（Tool Entropy）** | 基于工具调用频率分布计算熵 | 度量工具使用的集中度：低熵表示依赖单一工具（专业化），高熵表示广泛使用（可能低效） |
| **信息增益（Information Gain）** | \( IG = H_{\text{before}} - H_{\text{after}} \)，其中 \( H \) 为信念分布的熵 | 度量智能体在任务过程中减少的不确定性量；正值表示有效推理，负值表示更困惑 |
| **探索效率（Exploration Efficiency）** | \( EE = \frac{S}{H_A + \epsilon} \)，\( S \) 为成功率 | 结合成功与行为随机性，奖励高效成功的探索，同时允许困难任务有较高探索 |
| **鲁棒性熵（Robustness Entropy）** | 比较多次运行的轨迹熵与结果熵 | 理想状态：中等轨迹熵 + 低结果熵 + 高成功率 + 可控成本 |

### 算法流程（文字说明）
1. 收集智能体运行轨迹（事件序列、结果标签、成本、延迟等）；
2. 将轨迹转换为标准化的 `AgentRun` 表示（JSON 格式）；
3. 计算各熵指标；
4. 可选组合为 **Entropic Agent Score (EAS)**，一种可配置的聚合分数，综合奖励信息增益、成功以及行为熵和成本。

### 实现细节
- 提供 Python 包 `entropy-agent-eval`，框架无关，支持 JSON/JSONL 加载、命令行接口、通用事件记录器、LangChain 和 Google ADK 适配器、基准测试工具和可选绘图功能。
- 不强制依赖任何特定智能体框架，可整合实时回调、导出日志、数据库追踪、可观测性系统等。

## 3. 实验设计

### 实验一：受控基准测试（Controlled Benchmark）
- **场景**：四种已知行为的参考智能体模式，每个模式在6个任务上各运行3次，共72次运行。
- **数据集/任务**：事实问答、多跳推理、编码/调试等6个合成任务。
- **对比方法**：
  1. direct-llm（直接回答）
  2. react-search（基于搜索的ReAct模式）
  3. react-search-code（搜索+编码的ReAct模式）
  4. planner-executor（规划-执行模式）
- **评估方式**：记录轨迹、成功标签、结果标签、延迟、成本、信息增益（假设前后分布）。

### 实验二：学习路线图智能体（Learning Roadmap Agent）
- **场景**：为学习者制定个性化学习路线图（要求包含概述、先修知识、每周计划、项目、资源、评估检查点等）。
- **任务集**：3个路线图请求（生产级LLM代理、Python数据分析仪表盘、AI前端应用）。
- **对比框架**：
  - LangChain（使用OpenAI chat模型）
  - Google ADK（使用Gemini 2.5 Flash）
- **评价**：两个实现使用相同的规划辅助、提示、评分标准、EEA追踪模式。

1. **EEA能捕捉传统指标无法显示的行为差异**：例如，direct-llm轨迹熵最低（固定两步路径），但成功率也最低；planner-executor成功率和信息增益最高，但成本和动作熵也更高；react系列显示出不同的工具使用特征。
