## 摘要
AI智能体通常使用任务成功率、奖励、延迟和成本进行评估。这些指标虽然有用，但往往忽略了智能体行为的重要方面：是否过度探索、过于僵化地重复、有效使用工具、随时间推移减少不确定性、或在重复运行中保持鲁棒性。本文提出基于熵的AI智能体评估（EEA），一种通过熵来衡量智能体行为的轻量级框架。EEA不将智能视为仅最终任务完成，而是研究智能体决策过程的结构。该框架引入了动作熵、轨迹熵、工具熵、信息增益、探索效率和鲁棒性熵等指标。这些指标旨在补充而非取代传统评估方法。我们还提供了一个实用的Python实现，旨在与LangChain、Google ADK、自定义智能体循环以及存储的可观测性追踪等智能体框架集成。

## Abstract
AI agents are commonly evaluated using task success, reward, latency, and cost. These metrics are useful, but they often miss important aspects of agent behavior: whether an agent explores too much, repeats itself too rigidly, uses tools effectively, reduces uncertainty over time, or remains robust across repeated runs. This paper proposes Entropy-Based Evaluation of AI Agents (EEA), a lightweight framework for measuring agent behavior through entropy. Rather than treating intelligence as only final task completion, EEA studies the structure of the agents decision process. The framework introduces action entropy, trajectory entropy, tool entropy, information gain, exploration efficiency, and robustness entropy. These metrics are intended to complement, not replace, traditional evaluation methods. We also present a practical Python implementation designed to integrate with agent frameworks such as LangChain, Google ADK, custom agent loops, and stored observability traces.

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 将每个智能体的运行轨迹表示为一系列事件（工具调用、模型调用、规划步骤、动作、最终答案等），通过计算不同事件分布的熵来量化行为模式。
- 度量不依赖模型内部参数，仅需可观测的执行轨迹和可选的假设分布。

### 关键技术细节与公式
- **动作熵** (Action Entropy, HA)：衡量单次运行中动作的多样性。  
  \( H_A = -\sum_i p(a_i) \log_2 p(a_i) \)  
  低熵表示动作重复，高熵表示动作多样（可为探索或混乱）。
- **轨迹熵** (Trajectory Entropy)：衡量多次运行或不同智能体之间完整策略路径的多样性。路径如 `search -> read -> answer` 视为一个策略。
- **工具熵** (Tool Entropy)：衡量工具使用分布的集中或分散程度。低值表示依赖特定工具，高值表示广泛探索或低效。
- **信息增益** (Information Gain, IG)：衡量智能体在任务过程中降低假设不确定性的程度。  
  \( IG = H_{\text{before}} - H_{\text{after}} \)  
  其中 \( H_{\text{before}} \) 和 \( H_{\text{after}} \) 分别为收集证据前后信念分布的熵。正值表示不确定性减少。
- **探索效率** (Exploration Efficiency, EE)： \( EE = \frac{S}{H_A + \epsilon} \)，其中 \( S \) 为成功率，\( \epsilon \) 为避免除零的小值。奖励低行为随机性下的高成功率（但允许困难任务合理探索）。
- **综合评分**：可配置的**Entropic Agent Score (EAS)**，将上述指标加权整合，用于在特定基准上提供紧凑比较信号。
- **鲁棒性熵**：通过比较多次运行的轨迹熵与结果熵，理想模式为中等轨迹熵、低结果熵、高成功率、控制成本。

### 算法流程（文字说明）
1. 从智能体框架获取标准化运行记录（AgentRun），包含任务ID、事件序列、成功标志、成本、延迟、可选的前后假设分布。
2. 对记录中的事件类型、动作名称、工具名称、轨迹字符串等计算频率分布。
3. 根据上述公式分别计算各熵指标。
4. 可选生成综合EAS得分。

## 3. 实验设计：数据集/场景、基准、对比方法

### 实验阶段一：受控基准 (Control Benchmark)
- **场景**：合成生成6个任务（涵盖事实问答、多步推理、编码/调试），预先设计四种参考智能体模式：
  - 直接LLM智能体 (direct-llm)
  - 基于搜索的ReAct智能体 (react-search)
  - 搜索+代码ReAct智能体 (react-search-code)
  - 规划-执行智能体 (planner-executor)
- **运行**：每个智能体对每个任务运行3次，共72条标准化轨迹。
- **对比**：统计各智能体的成功率、成本、延迟、步骤数、动作熵、轨迹熵、工具熵、信息增益、EAS，展示传统指标无法区分的差异。

### 实验阶段二：学习路线规划智能体 (Learning Roadmap Agent)
- **场景**：自动生成个性化的学习路线图（包含概述、先修知识、周计划、项目、资源、评估检查点）。
- **任务集**：3个路线请求（LLM智能体生产、Python数据分析仪表板、AI辅助前端应用）。
- **实现框架**：LangChain（使用OpenAI模型）和Google ADK（使用Gemini 2.5 Flash），使用相同的规划辅助、提示词、评分规则和EEA追踪格式。
- **运行**：每个框架对每个任务运行1次，共6条轨迹。

## 6. 论文的主要结论与发现

- EEA能够成功捕捉传统指标遗漏的行为特征：例如direct-llm轨迹熵最低（0.000 bit）但成功率仅0.389；planner-executor成功率最高（0.778）且信息增益最大（0.430 bit），但成本与动作熵也高；react-search/react-search-code使用不同工具集导致工具熵差异（0.000 vs 0.722）。这些差异仅靠成功率无法区分。

- **核心主张**：熵提供了一种有用的行为视角，与成功率、成本、延迟、质量结合，能帮助更好地理解智能体如何探索、适应和失败。

## 7. 优点：方法或实验设计上的亮点

- **轻量框架中立**：不依赖特定智能体运行时，仅需要标准化追踪格式，支持LangChain、Google ADK、自定义循环、可观测性日志等多种输入，易于集成。
- **指标可解释性强**：熵在信息论中直观易懂，能描述多样性与不确定性，且可分解为多个维度（动作、轨迹、工具、信息增益）。
- **开源实现**：提供Python包（`entropy-agent-eval`），包括核心指标、评估器、CLI、基准测试工具、可视化等，方便社区复现与扩展。
- **设计思路务实**：明确不宣称自己是智能的通用定义，而是作为补充工具；鼓励与人类判断、领域特定评估结合使用。

## 8. 不足与局限
