## 摘要
用户依赖执行轨迹来观察代理行为、诊断故障并确保问责。这些轨迹包含丰富的过程细节，包括工具调用、中间决策和错误恢复逻辑。然而，这些细节可能暴露私有的过程技能，使得下游方法能够在没有访问模型权重或技能文件的情况下恢复关键公式、阈值和策略。为了量化这种风险并评估保护措施，我们构建了\textsc{CapTraceBench}基准，包含75个专业的长视野任务和跨七个领域的154个精选技能。我们还介绍了\textsc{RedAct}（https://github.com/XuShuwenn/RedAct），一个受保护轨迹发布框架，该框架定位受保护的关键信息，在保留验证者关键证据的同时重写轨迹，并嵌入行为水印以便进行下游溯源分析。在代表性的轨迹重用方法中，\textsc{RedAct}将原始轨迹上的归一化技能迁移（NST）从44.7-67.1%降低到低于无技能基线的水平，同时保留审计证据。其独立的行为水印在最多1.9%的误报率下达到93.6-100.0%的真实检测率。这些结果将公共代理轨迹视为安全接口，并表明选择性编辑可以在不删除审计证据的情况下减少过程能力泄露。

## Abstract
Users rely on execution traces to observe agent behavior, diagnose failures, and ensure accountability. These traces contain rich procedural detail, including tool invocations, intermediate decisions, and error-recovery logic. Yet this detail can expose private procedural skills, allowing downstream methods to recover key formulas, thresholds, and strategies without access to model weights or skill files. To quantify this risk and evaluate protection, we construct \textsc{CapTraceBench}, a benchmark of 75 specialized long-horizon tasks and 154 curated skills across seven domains. We also introduce \textsc{RedAct} https://github.com/XuShuwenn/RedAct, a protected trace release framework that localizes protected key information, rewrites traces while preserving verifier-critical evidence, and embeds behavioral watermarks for downstream provenance analysis. Across representative trace reuse methods, \textsc{RedAct} reduces normalized skill transfer (NST) from 44.7--67.1\% on raw traces to below the no-skill baseline, while preserving audit evidence. Its standalone behavioral watermarks reach 93.6--100.0\% true detection with a false alarm rate of at most 1.9\%. These results frame public agent traces as security interfaces and show that selective redaction can reduce procedural capability leakage without removing audit evidence.

### 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：工具使用智能体的执行轨迹包含丰富的程序性细节（如工具调用、中间决策、错误恢复逻辑），这些细节在用于透明度、调试和问责的同时，也可能泄露私有的**过程技能**（procedural skills）——即可复用的工作流、公式、阈值、工具选择、验证脚本等。下游攻击者仅通过观察公开发布的轨迹，就能恢复这些技能，而无需访问模型权重或私有技能文件。
- **研究背景**：随着智能体系统在生产环境中部署，技能学习和演化研究表明，从智能体经验中蒸馏可复用的技能可以显著提升下游任务性能。这使得轨迹发布成为一个**程序性资产保护问题**：如何在保留审计所需证据的同时，防止受保护技能的泄露？作者提出这是一个“审计能力与技能保护之间的根本张力”，并将此问题形式化为**黑盒轨迹披露（black-box trace disclosure）**。
- **整体含义**：将公开代理轨迹视为安全接口，表明选择性编辑（redaction）可以在不破坏审计的情况下减少程序性能力泄露。

### 2. 提出的方法论

#### 核心思想
提出**RedAct**框架（Redacting Agent Capability Traces），由两个互补层组成：
1. **重写层（Rewriting Layer）**：定位并抽象受保护的程序性信息（如公式、阈值、API选择、私有启发式），同时保留验证者关键证据（final answers, tool names, verifier paths, schema fields）。
2. **水印层（Watermarking Layer）**：向公开轨迹中嵌入行为水印，用于下游溯源追踪，且不影响任务语义。

#### 关键技术细节

- **关键信息引导重写**：
  - 假设技能所有者能访问受保护技能包（skill package）。
  - 第一步：LLM-based Key-Item Locator \(g_\eta(d, s) \to K_{\text{prot}}\) 从任务指令 \(d\) 和技能包 \(s\) 中提取受保护的流程项目（公式、常量、阈值、专业工具选择、验证例程）。
  - 第二步：LLM-based Rewriter \(\rho_\theta(d, \tau, K_{\text{prot}})\) 构建受保护轨迹 \(z_{\text{RA}}\)：抽象中间轮次中揭示受保护项目的部分，保留最终输出、工具使用证据和执行顺序。
  - 轻量人工审核用于去重。

- **行为水印**：
  - 定义钩子家族（hook family）\(h\)，每个家族包含三个组件：\(W_{\text{check}}^h\)（选择有效插入点）、\(W_{\text{inject}}^h\)（插入无害的动作-观察模式和激活短语 \(\kappa_h\)）、\(W_{\text{detect}}^h\)（在学生轨迹中检测对应行为）。
  - 实例化四种钩子家族：
    - **Standalone**：Ritual Marker（任务开始/结束固定动作）、Env Check（良性环境探测）
    - **Contextual**：Error Anchoring（错误后恢复短语）、Cross Check（工具结果后验证）
  - 水印比例 \(r \in \{0.1, 0.2, 0.3\}\)，检测时使用激活关键词查询学生模型。

- **算法流程（文字说明）**：
  - 保护发布：对每条轨迹，提取关键信息 \(K\) → 分割用户/助手回合 → 对助手批处理应用重写（检查可解析性和对齐）→ 交错回合 → 验证最终答案不变。
  - 水印注入：选择有资格轨迹 → 按比例采样 → 插入激活短语和行为 → 验证不影响验证器结果。

### 3. 实验设计

#### 数据集 / 场景

- **基准**：作者构建**CapTraceBench**，包含75个长视野专业任务（long-horizon tasks），来自7个领域（生命科学、化学、地球与气候、工程、金融与经济、媒体与网络、软件与安全），23个任务家族，共154个精选技能。
- 难度分为Easy（20任务/31技能）、Medium（38任务/73技能）、Hard（17任务/50技能）。
- 每个任务包含：人工编写的指令、可执行环境、自动验证器、1-3个特定技能。

#### 对比方法

- **下游轨迹重用方法**（四种）：
  1. 单智能体技能提取（Extracted Skills）：从轨迹中合成结构化SKILL.md文档和可重用脚本。
  2. 多智能体技能演化（Evolved Skills）：通过多智能体精炼从成功和失败轨迹中提取标准化技能。
  3. 工作流检索重用（Retrieval Reuse）：索引轨迹并作为上下文示例注入。
  4. 轨迹微调（Trajectory Fine-tuning）：在公开轨迹上微调开源学生模型（用于溯源检测）。

- **披露条件**：
  - No Skills（无技能基线）、Oracle Skills（直接提供技能）、Raw Traces（原始轨迹）、Protected Traces（RedAct保护后轨迹）。

#### 评估指标

- **Success Rate (SR)**：任务通过率。
- **Step Success Rate (SSR)**：步骤级平均通过率（部分完成得分）。
- **Normalized Skill Transfer (NST)**：相对于No Skills的SSR增益，标准化到Oracle Skills增益。越低表示技能迁移越少。
- **Recovered Protected Information (RPI)**：下游产物中恢复的受保护关键信息比例。越低越好。
- 水印检测：True Detection (TD), False Alarm (FA)。

### 6. 主要结论与发现

- **原始轨迹是脆弱的攻击面**：在No Skills基线SR=45.2%, SSR=68.0%下，原始轨迹重用方法几乎能达到Oracle Skills性能（SSR 73.5-73.7%），说明轨迹泄露了可操作知识（工具选择、中间检查、恢复例程），而非仅最终答案。
- **RedAct有效抑制重用优势**：三种重用方法在受保护轨迹上的SSR降至67.5%、66.3%、65.5%，均不高于No Skills基线（68.0%）。NST由44.7-67.1%降至负值（-5.9%到-29.4%），RPI下降37-48%。
- **困难任务泄露更严重**：原始轨迹在硬任务上SSR增益6.5-10.5%，保护后压缩至最多4.8%，甚至低于基线。
- **行为水印有效**：独立水印（Env Check, Ritual Marker）TD 93.6-100%，FA≤1.9%；上下文水印检测率较低（16-32%），但FA=0%，适合选择性检测。
- **保留审计证据**：保护后最终答案、工具名、验证路径、模式字段保留91-96.6%，工具调用数和轨迹长度在90-110%间，移除了70.6%的关键项目。
- **预算影响有限**：增加检索预算(k从1到8)，保护轨迹仍比原始轨迹低4.2-5.3% SR、4.6-5.1% SSR，差距稳定。

### 7. 优点

- **问题新颖且重要**：首次形式化“黑盒轨迹披露”问题，将轨迹发布提升为安全与隐私议题。
- **构建了高质量基准**：CapTraceBench覆盖多领域、多难度，技能经过精心筛选，支持自动验证器，便于复现。
- **方法设计精巧**：双重保护——显式关键信息定位+重写，以及隐式行为水印溯源，兼顾保护与审计需求。
- **实验充分**：多种下游重用方法、多种模型、多种水印比例、消融实验、人类评估，结果具有统计意义。
- **开源释放**：公开基准、提示词、检测器，推动社区研究。

### 8. 不足与局限（论文自述及分析）

- **受控环境**：75个任务均为自动验证器控制的专业化任务；真实部署轨迹可能包含更多噪声、不断变化的工具环境、更弱的验证反馈，保护效果可能不同。
- **水印上下文依赖**：上下文水印（Cross Check, Error Anchoring）在学习过程中较难保留（检测率低），而独立水印更易被简单移除（如删除固定标记）。作者指出强上下文触发器和大规模开源学生上的评估是未来工作。
- **安全假设限制**：假设技能所有者能访问完整技能包；若攻击者同时获取多源辅助信息，保护可能被削弱。
- **未覆盖更高级攻击**：如自适应攻击（主动去除水印）、模型逆蒸馏等。作者承认水印是统计证据而非密码学证明。
- **适用性**：仅针对同一任务环境（same-task, in-distribution）的评估，跨任务迁移的安全性未测试。
