## 摘要
基于视觉语言模型的GUI智能体在自动化数字任务方面展现出巨大潜力，但由于缺乏步骤级验证，在长时间跨度场景中常出现失败。现有过程奖励模型仅通过文本推理验证动作，忽略了GUI状态变化的视觉特性。我们提出VisCritic，一种视觉过程奖励框架，通过直接比较动作前后的截图在视觉特征空间中来验证智能体动作。VisCritic采用孪生视觉Transformer提取变化感知表示，并结合动作感知评论头，该评论头联合评估动作成功性、任务进度和错误类型。一个评论训练数据构建流程从现有轨迹中生成弱监督样本，无需额外人工标签用于评论训练。在五个基准上的实验和离线分析表明，VisCritic作为即插即用增强模块适用于多种GUI智能体，在提供视觉诊断线索的同时普遍改善了基准指标。

## Abstract
GUI agents powered by vision-language models show strong potential for automating digital tasks, yet frequently fail in long-horizon scenarios due to the absence of step-level verification. Existing process reward models verify actions through textual reasoning alone, missing the visual nature of GUI state changes. We introduce VisCritic, a visual process reward framework that verifies agent actions by directly comparing pre-action and post-action screenshots in visual feature space. VisCritic employs a Siamese vision transformer to extract change-aware representations, coupled with an Action-Aware Critic Head that jointly evaluates action success, task progress, and error type. A critic-training data construction pipeline generates weakly supervised samples from existing trajectories without additional human labels for critic training. Experiments and offline analyses across five benchmarks demonstrate that VisCritic serves as a plug-and-play enhancement for diverse GUI agents, generally improving benchmark metrics while providing visual diagnostic cues.

- **核心问题**：基于视觉语言模型（VLM）的GUI智能体在长程多步任务中经常失败，因为缺乏可靠的步骤级验证机制。单个错误动作会级联导致整个任务失败。
- **现有方法不足**：现有过程奖励模型（PRM）仅通过文本推理（如状态描述、工具调用、结构化检查表）来验证动作，忽略了GUI状态变化的视觉本质（像素级变化如按钮颜色改变、弹窗出现、页面滚动）。
- **研究动机**：直接比较动作前后的截图在视觉特征空间中的差异，可以作为更自然、更有效的过程奖励信号。
- **整体含义**：提出VisCritic框架，通过视觉状态比较来实现动作验证，作为即插即用的推理时增强模块，提升多种GUI智能体的性能。

## 2. 方法论

### 核心思想
- 使用孪生视觉Transformer（Siamese ViT）编码动作前后的截图，提取语义级别的差异特征。
- 引入变化区域注意力（Change Region Attention），聚焦发生有意义UI变化的区域。
- 设计一个动作感知评论头（Action-Aware Critic Head），联合预测三个任务：动作成功概率（y_suc）、任务进度分数（y_prog）、错误类型（y_err）。
- 训练数据通过弱监督流程从现有轨迹自动构造，无需额外人工标注。

### 关键技术细节
1. **视觉差异编码器（VDE）**：
   - 共享权重的ViT编码器提取前后截图的patch特征：F_t, F_{t+1}。
   - 计算差分特征 ΔF = F_{t+1} - F_t，以及变化幅度图 M_i = ||ΔF_i||_2。
   - 通过softmax得到变化区域注意力权重 α_i = exp(M_i/β) / Σ exp(M_j/β)。
   - 全局注意力差异向量 v_Δ = Σ α_i * ΔF_i。
2. **动作感知评论头**：
   - 编码动作a_t和任务指令l的文本特征h_a, h_l。
   - 通过交叉注意力融合视觉差分特征与文本上下文，结合空间自注意力。
   - 输出三个头：动作成功概率（sigmoid）、进度分数（tanh）、错误类型（softmax）。
3. **训练数据构造**：
   - 正样本：从成功轨迹中抽取 (s_t, a_t, s_{t+1})，y_suc=1。
   - 负样本：四种扰动策略——动作不匹配、状态不匹配、无操作检测、失败轨迹采样。
   - 错误类型标签由扰动策略自动推导（与人工标注一致性达81.4%）。
   - 进度分数：成功步骤为(t+1)/T，失败轨迹发散点后为负值。
4. **训练流程**：
   - 阶段1：对比预训练（InfoNCE损失 + 定位辅助损失）。
   - 阶段2：多任务微调（三个损失加权求和）。

### 关键公式（文字说明）
- 差分特征：逐patch相减。
- 变化幅度：L2范数。
- 注意力权重：带温度参数β的softmax。

1. VisCritic作为即插即用模块，在大多数测试的交互设置中提升了所有基础智能体的性能（离线AITW除外），优于文本PRM基线。

6. 注意力图提供可视化诊断线索，可指示成功或错误类型。

## 7. 优点

- **创新性**：首次将视觉状态比较作为GUI智能体的过程奖励信号，弥补了文本验证与视觉变化之间的模态鸿沟。
