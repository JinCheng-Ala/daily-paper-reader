## 摘要
随着自主智能体处理日益复杂的多步、多智能体任务，其执行轨迹规模已超出最大上下文窗口的限制。当前有效诊断智能体故障的方法通常将完整轨迹加载到LLM的上下文窗口中，这会导致注意力稀释，并在智能体痕迹不可避免地超出上下文限制时失效。为解决此问题，我们提出SAFARI（通过主动调查扩展长视界智能体故障归因）框架，该框架用工具增强的诊断循环替代线性上下文加载。通过为LLM配备专门的工具箱以读取和搜索轨迹片段，并结合持久短期记忆（STM）进行跨轮推理，SAFARI有效地将诊断准确性与架构上下文限制解耦。实验表明，在Who&When数据集上以1M token预算，SAFARI比当前最优结果提升20%；在TRAIL GAIA子集上以25K token预算提升19%。最重要的是，当目标故障位于模型原生上下文窗口5倍之外时（传统评估器完全失效的场景），SAFARI仍能保持0.58的精度。

## Abstract
As autonomous agents tackle increasingly complex multi-step, multi-agent tasks, their execution trajectories have scaled beyond the constraints of even the largest context windows. Current methods for effectively diagnosing agent failures load the full trajectory into an LLM's context window, which suffers from attention dilution and fails when agentic traces inevitably exceed context limits. To address this, we introduce SAFARI (Scaling long-horizon Agentic Fault AttRibution via active Investigation), a framework that replaces linear context loading with a tool-augmented diagnostic loop. By equipping LLMs with a specialized toolbox to read and search trajectory segments alongside a persistent Short-Term Memory (STM) for cross-turn reasoning, SAFARI effectively decouples diagnostic accuracy from architectural context limits. Our experiments demonstrate that SAFARI outperforms state-of-the-art results by 20% on the Who&When dataset within a 1M token budget, and by 19% on TRAIL GAIA subset on a 25K token budget. Most significantly, SAFARI maintains a 0.58 precision even when the target fault resides 5x beyond the model's native context window, a scenario where traditional evaluators fail entirely.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：随着大语言模型（LLM）驱动的自主智能体系统执行越来越复杂的多步、多智能体任务，它们的执行轨迹（trajectories）长度已远远超过了当前最强LLM的上下文窗口限制。传统的故障归因（Fault Attribution）方法，即定位智能体系统中错误发生的具体步骤，通常依赖于将完整轨迹加载到LLM的上下文窗口中进行一次性分析。这种方法存在两个根本性缺陷：
    1.  **注意力稀释**：即使轨迹勉强能够放入上下文窗口，LLM也会因为处理过长的信息而注意力稀释，性能下降。
    2.  **上下文超限**：当轨迹长度不可避免地超出上下文窗口时，这种方法会直接失败，无法进行诊断。
- **整体含义**：论文旨在解决长时域、超长轨迹下的智能体故障归因这一关键挑战。SAFARI框架通过引入**主动调查**的范式，将诊断过程从被动的“全文阅读”转变为主动的“证据搜寻”，从而在根本上解耦了诊断精度与LLM架构的上下文限制，为构建更可靠、可扩展的自主智能体系统提供了关键技术支持。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：SAFARI用**工具增强的诊断循环**替代了传统的线性上下文加载。它设计了一个“调查代理”（Investigator Agent），该代理能够通过迭代地调用专用工具，主动、选择性地查询和推理轨迹片段，从而像人类调查员一样逐步收集证据并定位故障，而不是一次性“阅读”整个轨迹。

- **关键技术细节**：
    1.  **调查代理**：一个LLM，负责主持整个调查过程。它接收系统提示和目标轨迹的元信息，但不直接读取轨迹全文。
    2.  **专用工具箱**：配备两个核心工具，用于在巨大的轨迹搜索空间中有效导航：
        - `read(offset, limit)`：读取轨迹中指定偏移位置和长度的连续片段。模拟工程师“滚动”查看日志。
        - `search(pattern)`：在序列化的轨迹上执行不区分大小写的正则表达式搜索，允许代理瞬间“跳转”到特定步骤ID、变量值或错误模式，无需消耗完整上下文窗口。
    3.  **短期记忆**：一个持久化的、在每次LLM调用时都会被追加到对话历史末尾的结构化信息块。包含四个核心字段：底层任务目标、有关失败步骤的假设与证据、未完成的调查计划以及过去的工具调用记录。它确保关键证据和逻辑意图在上下文窗口限制下不会被意外丢失。
    4.  **原子推理评估器**：调查代理在得出最终结论前，必须通过一个验证阶段。它将自身的假设分解为最多`E=3`个原子声明，每个声明都是一个关于特定步骤的可证伪陈述。这些声明被并行发送给一个独立的“推理评估器”LLM，该评估器**无权访问原始轨迹**，仅根据调查代理提供的推理和引用的证据来验证声明是否正确，并给出置信度分数。只有当平均置信度超过阈值（默认70）时，调查代理才能最终提交结论。
    5.  **调查循环**：调查代理在第`k`轮进行推理，生成假设，调用工具获取观察结果。如果评估未通过，本次的工具调用和观察结果会被追加到历史记录中，进入下一轮迭代。如果达到最大迭代次数仍未通过，则执行“强制终结论证”程序。

### 3. 实验设计：数据集、Benchmark、对比方法

- **数据集**：使用了两个主流的长时域智能体故障归因基准测试：
    - **Who&When**：包含来自Magentic-One的算法生成子集和CaptainAgent的手工制作子集。手工制作子集轨迹更长、复杂度更高。
    - **TRAIL**：包含GAIA和SWE-bench两个子集。GAIA子集有最长的轨迹（高达122个步骤），SWE-bench子集则采用密集令牌，步骤少但每个步骤信息量大，有些轨迹超过200万令牌。
- **Benchmark**：采用**决定性故障归因**（Decisive Fault Attribution）这一标准任务，并报告**步级准确率**、**容忍准确率**（允许一定步数误差）、**精确率**和**严格精确率**（针对TRAIL，因该数据集未标注唯一决定性故障）。
- **对比方法**：
    - **一次性提示**：直接将完整轨迹加载进LLM上下文窗口进行分析的基线方法。
    - **单步扫描**：逐步骤扫描轨迹直到定位到故障，以暴力方式绕过上下文长度限制的方法。
    - **RAFFLES**：当前最先进的迭代推理基线方法，通过多轮推理来细化归因，但仍受限于模型的原生上下文窗口。

### 6. 论文的主要结论与发现

1.  **卓越的长序列性能**：在资源受限环境下，SAFARI显著优于所有基线。在Who&When数据集（1M预算）上提升20%，在TRAIL GAIA子集（25K预算）上提升19%。
2.  **打破上下文窗口限制**：SAFARI能够在目标故障位于模型原生上下文窗口**5倍之外**时，仍能保持**0.58的精确率**。这是传统评估器完全无法做到的，突显了“主动调查”范式的巨大优势。
3.  **短期记忆的有效性**：消融实验证实，短期记忆模块不仅是简单的摘要，更是维持调查意图和逻辑连贯性的“逻辑锚点”，是SAFARI成功的关键。在高预算下，STM带来的信息损失有时会超过其优势，导致性能略降。
4.  **领域适应性**：SAFARI在处理通用智能体轨迹（GAIA）时表现优秀，但在处理令牌密集的代码型轨迹（SWE-bench）时遇到更大挑战，因为将其压缩进短期记忆更加困难。

### 7. 优点

- **方法论创新**：提出的“主动调查”范式是对传统“全量加载”范式的根本性变革，为处理超长文本序列提供了一种全新的、有效且可解释的思路。
- **目标明确，解决痛点**：直接针对当前LLM应用中的关键瓶颈——上下文长度限制——进行攻关，具有很强的实用价值。
- **设计精巧，组件有效**：工具箱设计（`read`+`search`）和验证机制（原子推理评估器）确保了调查的效率和精度；短期记忆模块的设计既解决了记忆丢失问题，又保持了轻量和结构化。
- **评估全面，方法公平**：不仅评估了标准场景，还专门设计了“固定缩放因子”这种新的评估指标，用于客观衡量模型在上下文窗口外的表现。实验设计客观且具有说服力。

### 8. 不足与局限

- **高预算下的性能倒退**：当轨迹长度完全在上下文窗口内时，SAFARI性能略低于一些全量加载的基线方法，这主要归因于短期记忆模块带来的信息损失。这表明方法在特定场景下存在优化空间。
- **对数据密度的敏感性**：在处理如SWE-bench这样令牌密集的轨迹时，性能下降明显。未来工作需要探索如何更好地压缩和表征此类数据。
- **依赖于大型骨干模型**：当前研究依赖于Claude-Opus-4.6等最强大的商业模型。其“主动调查”能力的泛化性在更小型或不同架构的模型上尚未得到验证。同时，这些模型内部的知识被先验地用于推理（如论文附录中提到的“FunkMonk”记忆），这可能混淆了基准测试的难度。
- **数据记忆风险**：论文坦诚地指出了其使用的公开数据集可能已被LLM在训练阶段记忆，这会影响评估的公平性。这是当前学术研究中的普遍问题，但对该方法在私有或非公开数据上的表现提出了疑问。
- **延迟与成本**：在低预算场景下，SAFARI通过多次工具迭代来实现高精度，这带来了显著更高的延迟和推理成本。其效率与性能之间的权衡是部署时需要考虑的因素。
