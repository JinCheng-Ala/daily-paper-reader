## 摘要
针对LLM代理的运行时监督通常被框定为标量风险预测：估计失败可能性、置信度或不确定性，一旦得分超过阈值则进行干预。我们认为这种框架针对了错误的控制对象。相关的问题不是代理如果继续下去失败的可能性有多大，而是可用的干预是否会改善结果。两个轨迹前缀可能具有相同的风险估计，但需要不同的行动，因为一个仍可恢复而另一个则不能。我们将这种不匹配形式化为目标误差，并确定干预优势——即干预而非继续所期望的效用增益——作为监督的决策对象。为了衡量这种不匹配，我们引入了前缀分支，这是一种相同前缀的反事实协议，从相同的轨迹状态执行候选行动。在四个基准测试中，行动条件控制相对于标量路由产生了依赖于机制的性能提升。在校准分解中，重新校准相同的标量得分改善了预测指标，但控制遗憾保持不变，表明仅靠校准不能修复目标误差。一个简单的前缀仅行动条件控制器在最强交互机制下显著减少了遗憾，在ALFWorld上从0.506降至0.110。当干预较弱或标量路由已经保留了干预相关信息时，增益会缩小。这些结果表明，LLM-Agent监督应从校准风险评分转向行动条件价值估计。

## Abstract
Runtime oversight for LLM agents is commonly framed as scalar risk prediction: estimate failure likelihood, confidence, or uncertainty, then intervene once the score crosses a threshold. We argue that this framing targets the wrong object for control. The relevant question is not how likely the agent is to fail if it continues, but whether an available intervention would improve the outcome. Two trajectory prefixes can have the same risk estimate while requiring different actions, because one remains recoverable and the other does not. We formalize this mismatch as target error and identify intervention advantage, the expected utility gain from intervening rather than continuing, as the decision object for oversight. To measure this mismatch, we introduce prefix branching, a same-prefix counterfactual protocol that executes candidate actions from identical trajectory states. Across four benchmarks, action-conditioned control yields regime-dependent gains over scalar routing. In a calibration decomposition, recalibrating the same scalar score improves prediction metrics but leaves control regret unchanged, showing that calibration alone does not repair target error. A simple prefix-only action-conditioned controller substantially reduces regret in the strongest interactive regime, from 0.506 to 0.110 on ALFWorld. Gains shrink when interventions are weak or when scalar routing already preserves intervention-relevant information. These results suggest that LLM-agent oversight should move from calibrated risk scoring toward action-conditioned value estimation.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有 LLM 代理运行时监督普遍采用**标量风险预测**（如失败概率、不确定性或置信度）并在此基础上设置阈值进行干预。论文指出这种做法瞄准了错误的控制目标：真正关键的不是“代理如果继续下去失败的可能性有多大”，而是“是否存在一种可用的干预能比继续执行带来更好的结果”。两个轨迹前缀可能具有相同的风险评分，但一个可恢复、一个不可恢复，需要不同的干预决策。
- **整体含义**：标量风险评分（尤其是延续风险）虽然可以校准，但无法捕捉干预是否真正有价值。因此，监督系统应从事后风险评分转向**行动条件价值估计**，即基于每个候选行动（继续、介入、停止）的预期效用做出决策，而不是仅靠一个分数触发干预。论文将这种不匹配形式化为**目标误差**，并引入**干预优势**作为控制目标。

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：用**干预优势**（\( \tau(h) = \mathbb{E}[U_{intervene}(h)] - \mathbb{E}[U_{continue}(h)] \)）代替延续风险作为控制决策的依据。只有当干预的期望效用大于继续时，才应进行干预。
- **关键技术**：
  - **前缀分支协议**：在离线阶段，对同一前缀状态 \( h \) 分别执行继续、干预、停止等所有候选行动，记录各自的最终效用。通过这种方式获得反事实数据，从而能比较同一状态下不同行动的价值。
  - **保守行动条件控制器**：基于随机森林（提供预测方差），对每个行动预测成功概率 \( \hat{p}_a(h) \) 并转化为期望效用，再使用下置信界（LCB）进行保守选择：\( \hat{p}^{LCB}_a(h) = \hat{p}_a(h) - \beta \cdot \hat{\sigma}_a(h) \)。最终选择 \( \arg\max_a \hat{p}^{LCB}_a(h) \cdot r^+_a + (1-\hat{p}^{LCB}_a(h)) \cdot r^-_a \)。
- **公式化损失**：定义了标量抽象损失 \( \text{Gap}(g) = V^* - V_g \)，其中 \( V^* \) 是全信息最优控制器的价值，\( V_g \) 是仅依赖标量 \( g \) 的最优控制器价值。损失大小反映了压缩为标量后丢失的控制相关信息。

### 6. 论文的主要结论与发现

- **标量路由存在目标误差**：一个校准良好的失败评分仍然可能隐藏恢复性差异，导致在可恢复与不可恢复的高风险状态做出相同错误决策。
- **行动条件控制可显著降低遗憾**：在最强交互机制（ALFWorld）上，简单前缀仅行动条件控制器将遗憾从 0.506 降至 0.110；ScienceWorld 从 0.245 降至 0.169；GSM8K 小幅改善（0.423→0.394）；HotpotQA 几乎无改善（0.436→0.417）。
- **校准无法修复控制后悔**：对同一标量进行 Platt 缩放或等渗回归后，预测指标（ECE、Brier 评分）大幅改善，但控制遗憾完全不发生变化，验证了目标误差独立于校准。
- **增益出现的边界条件**：行动条件控制有效需同时满足两个条件：① 干预本身具有足够下游价值（能改变结果）；② 标量评分丢弃了干预相关的信息（隐藏恢复性）。Oracle 干预探针进一步揭示 GSM8K 的弱干预掩盖了潜在差距（增益从 0.028 升至 0.127），而 HotpotQA 的标量已接近充分。
- **可开发性诊断**：基于前缀特征超过标量预测的 oracle 行动准确率提升（exploitability），与部署增益呈中等正相关（整体 \( r=0.716 \)），可用于开发阶段识别值得构建行动条件控制器的场景。

### 7. 优点：方法或实验设计上的亮点

- **理论清晰**：形式化了控制充分性、标量抽象损失、干预优势，将监督问题明确定位为决策问题而非预测问题。
- **反事实评估**：前缀分支协议允许在**相同前缀状态**下比较不同行动的后果，避免了离线策略评估中的分布偏移问题。
- **保守的控制器设计**：使用下置信界（LCB）避免过度乐观，使得增益归因于决策目标改变而非过拟合。
- **系统性的机制分析**：通过干预强度调节、Oracle 干预探针、成本敏感性扫描、跨模型迁移等实验，清晰划定了“何时行动条件控制有益”的边界，具有较强的诊断价值。
- **公平比较**：控制器使用相同的信息量（前缀仅特征，无未来信息），避免了信息优势带来的不公平。

### 8. 不足与局限：包括实验覆盖、偏差风险、应用限制

- **基准覆盖有限**：主要结论依赖于四个基准，且最强正面证据来自 ALFWorld。在推理基准（GSM8K、HotpotQA）上增益很小，在 ScienceWorld 上增益适中。尚需在更多交互环境和真实部署场景中验证。
- **控制器设计简单**：论文明确说明其见证控制器是“conservative and prefix-only”，增益应视为决策目标重要性的证据，而非最优架构。未来需要更强大的多动作选择和在线策略。
- **离线评估局限**：前缀分支是离线单次评估协议，重复分支仅在小规模浅层子集上验证了稳定性。在更深前缀上可能存在采样噪声（文中承认深层前缀依赖单次 rollout）。
- **干预种类简化**：仅考虑二元“继续/干预”抽象，未处理多种结构化干预的选择问题（如选择专家、工具、重试等）。论文将多动作选择留作未来工作。
- **未涉及在线学习**：所有控制器均基于离线数据训练，未考虑部署时自适应调整。在线行动条件价值学习可能带来额外收益，但也需面对更复杂的探索-利用平衡。
- **伦理风险**：更可靠的监督技术可能被滥用（如增强有害代理的可靠性），论文仅通过将贡献定位为离线评估协议而非在线代理能力来缓解此类风险，但实际影响需进一步关注。
