## 摘要
能否用一张图表示每种LLM智能体的运行过程？轨迹记录了每一步做了什么，但从未记录它依赖于什么、读取了哪些状态、复用了哪些结果。GRADE恢复了缺失的层次：它将任何运行过程建模为一张图，包含步骤节点和两种边层——执行边（以何种顺序运行了什么）可从轨迹中免费读取，依赖边（每一步依赖什么）很少被记录，因此每条依赖边根据其已知、观测、声明或推断的方式来分级。一个表示，每一层各得其所。在涵盖工具使用、编码和网页的六个LLM智能体语料库中，依赖层可以在运行规模失效时预测失败，并且在留一语料库迁移下，依赖层在每个保留类别上仍高于随机水平，而运行规模则失效。同时，执行层可定位失败的多智能体运行中的故障步骤。本文还深入分析了为何通用图神经网络可能误读依赖层（与我们基于特征的替代方法不同）。相同的图表示可进一步用于从单次运行的故障诊断到大规模效率和鲁棒性优化等多种用途。

## Abstract
Can one graph represent every kind of LLM agent's run? A trace records what each step did, never what it relied on, the state it read, and the results it reused. GRADE recovers that missing layer: it models any run as one graph over its step nodes with two edge layers, execution edges (what ran in what order) read from the trace for free, and dependency edges (what each step relied on) rarely logged, so each is graded by how it is known, observed, declared, or inferred. One representation, and each layer earns its place. Across six corpora of LLM agents spanning tool use, coding, and the web, the dependency layer can predict failure where run size is weak and, under leave-one-corpus-out transfer, stays above chance on every held-out class while run size fails. Meanwhile, the execution layer localizes the faulting step in a failed multi-agent run. This work also provides a more in-depth analysis of why generic graph neural networks may misread the dependency layer, unlike our feature-based alternative. The same graph representation opens further uses, carrying from failure diagnosis in a single run to efficiency and robustness optimization at scale.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：当前LLM智能体的运行轨迹仅记录“做了什么”及“执行顺序”，但完全缺失“每一步依赖了什么”——即每一步读取了哪些状态、复用了哪些中间结果。这导致一种重要失效模式被系统性忽视：**依赖失效**（reliance failure），例如一个读取的价格随后变陈旧，但每一步仍正确执行，整个运行却因过时状态而失败。这类失效在现有方法中无法被 trace 捕捉，因而无法分析。

- **背景**：现有表示方法（如 ReAct 轨迹、MetaGPT 编排图、GPTSwarm 图等）都是对完整图的**投影**——它们只保留了执行层（控制流），而丢弃了依赖层。也没有任何方法为依赖边标注其来源的置信度（观测/声明/推断）。因此，分析被迫按智能体类别（工具使用、编码、网页导航）分别重建，且重现了相同的盲点。

- **整体含义**：本文提出 **GRADE**，统一表示所有LLM智能体的一次运行，将执行层和依赖层同时显式建模，并引入**依赖边置信度层级**（observed, declared, inferred），从而支持跨类别的统一分析。这一表示本身即核心贡献，在此基础上实现了两个重要功能：依赖层预测运行失败、执行层定位故障步骤。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- 一次智能体运行建模为有向、带类型、时序的多重图 \( G = (V, E_X, E_D, \tau, t, \sigma) \)，其中：
  - 节点类型四类：agent（智能体）、decision（决策）、tool call（工具调用）、external resource（外部资源，如数据库行、源代码文件）。
  - **执行边 \( E_X \)**：记录控制流（agent发出决策和工具节点、控制权在步骤间转交）。从 trace 直接解析，零成本获得。
  - **依赖边 \( E_D \)**：记录每一步依赖的前序节点（读/写资源）。trace 本身不记录依赖，需要从 trace 内容中恢复（观测）、通过仪器化声明（declared）、或在无信息时基于假设推断（inferred）。
  - **依赖边置信度标注 \( \sigma : E_D \to \{observed, declared, inferred\} \) **：每条依赖边都标注其如何获得。这是所有先前工作缺失的关键信息。

### 2.2 关键技术细节
- **执行层的获取**：确定性读取 trace——哪个 agent 在何时执行、控制如何传递。
- **依赖层的获取**：
  - **观测 (observed)**：当 trace 已经记录了读/写事件（例如编码步骤编辑了某个文件、数据库读取某行），则直接提取依赖边。
  - **声明 (declared)**：通过额外仪器化（添加日志记录读和写事件）来获得。
  - **推断 (inferred)**：当无访问信息时，在命名假设下推断（最弱假设为全历史假设A0：每一步依赖所有前序步骤）。
- **边际提升估计 (Marginal-Lift Estimand)**：为了量化依赖层的价值，定义 \( \Delta_{\text{dep}} = \text{AUC}(\text{flat} + \text{dependency}) - \text{AUC}(\text{flat}) \)，其中 flat 只使用免费的运行规模特征（步数、工具调用数、决策数、agent数）。只有显著正值的 \(\Delta_{\text{dep}}\) 才表明依赖层提供了运行规模之外的新信息。
- **退化机制 (Degenerate Regime)**：当依赖层使用全历史假设A0推断时，边数 \( |E_D| = \binom{n}{2} \) 成为步数 \(n\) 的确定函数，依赖结构退化为运行规模的伪像。本文提出**饱和比 \( \rho = |E_D| / \binom{n}{2} \)** 来量化退化程度，\(\rho \approx 1\) 时依赖层无结构信息，\(\rho \ll 1\) 时依赖层有价值。
- **特征选择**：失败预测使用四个规模归一化的依赖形状特征（相对依赖链深度、最大 blast hub 份额、最大 audit hub 份额、以及依赖边密度）。为了公平比较，所有依赖特征都按运行长度归一化，以避免重新编码运行规模。

## 6. 主要结论与发现

### 6.1 依赖层预测失败
- **核心发现**：依赖层在运行规模是弱预测器时显著提升失败预测 AUC（如 SWE-Gym 提升 +0.142）。在运行规模已是强预测器时，依赖层贡献为零或负，说明重合信号而非新增信息。
- **跨类迁移**：依赖层（dep-only）在全部6个 held-out 类上 AUC > 0.5，从不反转（范围 0.551~0.662）；而运行规模（flat）在 tau-bench 和 SWE-Gym 上反转至低于随机（0.468、0.350）。证明依赖形状是跨类不变信号，而运行规模是类依赖的。
- **退化机制验证**：推断（全历史）依赖层在迁移时反复反转（3个 held-out 类低于随机），证实其只编码运行规模；观测依赖层保持正信号。

### 6.2 执行层定位故障
- **核心发现**：在执行图上的 agent 中心性和手递位置排名，在 Who&When 上的 top-3 准确率达 0.614，超越位置先验的 0.516 和随机的 0.346。所有三个指标（top-1、 top-3、 MRR）的结构超出先验都具有统计显著性的95%置信区间。

### 6.3 图神经网络失效原因
- 通用图网络（GIN、R-GCN、HGT）在转移时全面落后于源感知特征：源感知特征最差为 0.569，而 GIN 最低 0.531、R-GCN 0.409、HGT 0.235。原因：依赖层可能退化，但图网络缺乏读取依赖边置信度层级的通道，导致将推断边当作观测边处理，从而学到运行规模这一非可迁移信号。

## 7. 优点

### 7.1 方法上的亮点
- **统一表示**：首次将执行层和依赖层显式分离并置于同一图内，且为依赖边标注置信度层级。这使得跨类分析成为可能，消除了原有按智能体类别分别构建方案的碎片化。
- **边际提升估计量**：要求依赖层必须自证价值，而非仅报告自身 AUC —— 这是对 costly 依赖层贡献的严格度量。
- **退化机制与饱和比**：识别并量化解构了依赖层的一种特殊问题（全历史假设下结构消失），提供了操作性的判断条件（\(\rho\)）。
- **源感知特征替代图网络**：证明简单的逻辑回归 + 规模归一化的结构特征，在跨类迁移上远优于通用图网络，揭示了图网络缺少的归纳偏置。

### 7.2 实验设计亮点
- **留一语料库迁移**（pooled leave-one-corpus-out）比 pairwise 设计更严谨，测试了信号的共性而非两两相似性。
- **双轴线验证**：依赖层预测失败（§4），执行层定位故障（§6），两个层分别对应两个失效模式。
- **声明依赖的独立验证**：在 FLORA-Bench 上验证了声明依赖同样稀疏（\(\rho \approx 0.07\)），且当声明结构与失败耦合时可预测失败。

## 8. 不足与局限

### 8.1 实验覆盖
- 数据集偏向编码和数据库任务（5/6），仅一个网页导航语料，缺乏具身/机器人、多模态等更丰富的智能体类型。
- 无同时包含观测依赖和步骤级故障标签的语料库：无法验证依赖层定位故障的能力。
- Who&When 的依赖层退化（全历史推断），导致无法用真实依赖验证定位信号。

### 8.2 偏差风险
- **观测依赖的挑战性**：观测依赖的获取基于“将每个重要写操作连接到可能的先前读操作”这一启发性原则，缺乏地面真实依赖标签（agent 的 true reliance），是所有依赖恢复工作共有的限制。
- **图网络比较仅覆盖现成模型**：可能更复杂的、能编码置信度层级的图网络（如直接输入边类型张量或使用注意力区别）尚未测试。论文只定位了缺失归纳偏置，未排除所有可能的架构。

### 8.3 应用限制
- 声明依赖需要额外仪器化（日志记录），在已有系统中可能难以部署。
- 依赖层预测失败是关联性的，不能直接引导修复。离线的候选剪枝在验证前不可用于实际运行。
- 大规模监控下的无监督异常检测尝试弱（结果无效），仍为开放方向。

### 8.4 可重复性
- 未提供训练图网络的超参数设置细节（如层数、学习率等），仅表示“默认参数”，可能影响图网络实验的可复现性。但特征方法完全透明且可重复。
