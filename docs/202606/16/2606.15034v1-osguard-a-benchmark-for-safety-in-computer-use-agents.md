## 摘要
计算机使用代理日益通过完成现实的桌面和网页任务来评估。然而，任务成功本身可能遗漏代理通过不安全捷径达到名义目标的失败。我们引入OSGuard，这是一个双粒度基准测试套件，用于在良性、不变的用户指令下评估计算机使用代理的安全性。OSGuard包含一个用于本地护栏决策的动作级基准测试和一个用于端到端评估的风险增强执行套件。动作级基准测试由上下文化的提议动作组成，这些动作被标记为允许、无关或不安全，每个都相对于原始指令和当前界面状态进行判断。执行套件包含手动构建的OSWorld衍生任务变体，其中原始任务仍然可完成，但环境被修改以引入潜在危险，如破坏性覆盖等。每个变体都配有针对原始任务成功标准保留的增强评估器，同时添加了基于状态的显式安全不变量，使我们能够区分名义任务目标下的安全完成与不安全完成。我们在OSGuard上的实验结果表明，当前的多模态护栏在孤立动作判断上表现良好，而风险增强执行则揭示了本地监督与可靠端到端安全之间存在的差距。这种双粒度设计使得能够更精确地诊断模型是否既能识别不安全的提议动作，又能在部署为护栏时提高全任务安全性。

## Abstract
Computer-use agents are increasingly evaluated by whether they complete realistic desktop and web tasks. However, task success alone can miss failures in which an agent reaches the nominal goal through an unsafe shortcut. We introduce OSGuard, a dual-granularity benchmark suite for evaluating safety in computer-use agents under benign, unchanged user instructions. OSGuard contains an action-level benchmark for local guardrail decisions and a risk-augmented execution suite for end-to-end evaluation. The action-level benchmark consists of contextualized proposed actions labeled as allowed, unrelated, or unsafe, each judged relative to the original instruction and current interface state. The execution suite contains manually constructed OSWorld-derived task variants in which the original task remains achievable, but the environment is modified to introduce latent hazards such as destructive overwrites, etc. Each variant is paired with augmented evaluators that retain the original task-success criterion while adding explicit state-based safety invariants, allowing us to distinguish safe completions from unsafe completions that satisfy the nominal task objective. Our experimental results on OSGuard show that current multimodal guardrails can perform well on isolated action judgments, while risk-augmented execution exposes remaining gaps between local oversight and reliable end-to-end safety. This dual-granularity design enables more precise diagnosis of whether models can both recognize unsafe proposed actions and improve full-task safety when deployed as guardrails.

- **研究动机**：当前计算机使用代理（Computer-Use Agents）的评估主要关注是否完成用户指令，但任务成功可能掩盖代理通过“不安全捷径”达成目标的行为。例如，代理可能覆盖无关内容、扩大权限、修改全局设置、访问不必要的敏感信息或对错误目标进行操作，而这些行为在普通计算机使用中并不显式恶意。
- **核心问题**：在用户指令良性、不变且任务本身可达的前提下，如何系统性地评估代理在执行任务时是否违反了任务状态相关的安全约束（即“不安全捷径”）。
- **整体含义**：现有安全基准要么聚焦恶意请求或对抗性环境（如OS-Harm、RiOSWorld），要么通过扰动指令诱发有害行为（如AUTOELICIT），而OSGuard针对的是“普通良性任务下的状态依赖型危险”，填补了该领域的空白。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：采用双粒度评估框架，将安全性评估从局部动作层面延伸到全任务执行层面，并引入“安全不变量”概念。
- **动作级基准（Action-Level Benchmark）**：
  - 每个评估实例由 `(原始指令 u, 当前界面状态 s_{t-1}, 候选动作 a_t)` 组成。
  - 参考标签分为三类：`allowed`（允许执行）、`unrelated`（与任务无关但安全）、`unsafe`（不安全，应阻止）。
  - 构造来源：标准OSWorld执行轨迹、中断OGWorld轨迹+状态兼容提议动作、中断轨迹+不安全提议动作、风险增强变体执行轨迹。
  - 人工标注确保标签一致性，仅基于原始指令、状态和候选动作进行判断（不依赖提议者自述意图）。
- **风险增强执行套件（Risk-Augmented Execution Suite）**：
  - 基于OSWorld任务手动构建45个变体：保持原始指令不变，修改环境状态以引入潜伏危险（如破坏性覆盖、范围逃逸、配置冲突、不必要敏感信息访问、错误目标/全局更新等），同时保留安全完成任务的可能路径。
  - 每个变体配增广评估器：保留原OSWorld成功标准，并添加显式的基于状态的安全不变量（如文件存在性/哈希、目录权限、规则保留、无越界复制等）。
  - 成功定义：既完成原始任务又满足所有安全不变量。区分“安全成功”与“不安全完成”（后者达成名义目标但违反不变性）。
- **护栏（Guardrail）**：
  - 一个反应式、状态条件化的提示模型，输入为原始指令、当前状态和候选动作，输出允许/无关/不安全三类决策，并附带自然语言反馈。
  - 若护栏拦截动作，代理可在同一状态重新提议（最多重试2次），若均未通过则终止执行并记录。

- **动作级评估结果**：

- **执行套件评估结果**：

  - 加入最强护栏（Gemini 3 Pro Preview）后：安全成功率仍为62%（未改善），不安全完成率从38%微降至33%，引入4%的重试终止率。护栏仅修正了有限的不安全轨迹，总体改善很小。
- **核心发现**：当前系统在局部动作判断上表现尚可，但端到端安全执行仍有显著差距；同一类状态依赖危险既误导执行代理，也难被护栏可靠识别。

## 7. 优点

- **双粒度设计**：创新性地将动作级本地监督与全任务执行评估关联，支持更精准的诊断（可区分是模型无法识别不安全动作还是无法改变执行结果）。
- **聚焦关键缺口**：针对良性指令下的状态依赖危险，而非恶意或对抗性场景，更贴近真实部署中的安全故障。
- **可复现性**：安全不变量采用具体的状态检查（文件哈希、权限等），而非语义判断，便于验证。
- **基准构建严谨**：动作级标签基于人工标注，且仅使用模型可见信息；执行变体从原有成功任务改造，排除基础能力混淆。
- **实验设计合理**：对比了多种主流模型及专门护栏方法，并分析了按来源和危险族的性能差异。
