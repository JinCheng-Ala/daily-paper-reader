## 摘要
工具增强的LLM智能体依赖运行时过滤来决定每个步骤应暴露哪些工具。因果最小工具过滤（CMTF）通过仅暴露下一个因果必要的工具前沿来减少工具选择混淆，但它假设用户请求已映射到符号目标状态。实际上，诸如“处理我的预约”或“处理这封邮件”之类的请求可能对应多个可能目标。这导致错误目标执行，即智能体为意外目标遵循有效的因果工具路径。我们提出GIST-CMTF，一种目标状态推断层，它使用与CMTF相同的状态转换词汇预测候选符号目标，估计模糊性，并应用CMTF或将澄清作为产生缺失目标或状态变量的因果动作暴露。我们在七个模型后端、六种过滤方法和120个受控工具使用任务上评估GIST-CMTF。GIST-CMTF实现了97.0%的任务成功率，而顶部目标CMTF为80.1%，语义目标CMTF为82.9%。它将错误目标执行从顶部目标CMTF下的19.4%降低到2.5%，同时保留了因果过滤的单工具暴露，并且使用的令牌数远少于全工具暴露。这些结果表明，可靠的工具增强智能体在暴露外部动作之前应验证目标状态，而不仅仅是工具相关性。

## Abstract
Tool-augmented LLM agents rely on runtime filtering to decide which tools should be visible at each step. Causal Minimal Tool Filtering (CMTF) reduces tool-choice confusion by exposing only the next causally necessary tool frontier, but it assumes that the user request has already been mapped to a symbolic goal state. In practice, requests such as "handle my appointment" or "take care of this email" may correspond to multiple possible goals. This creates wrong-goal execution, where an agent follows a valid causal tool path for an unintended objective. We introduce GIST-CMTF, a goal-state inference layer that predicts candidate symbolic goals over the same state-transition vocabulary used by CMTF, estimates ambiguity, and either applies CMTF or exposes clarification as a causal action that produces missing goal or state variables. We evaluate GIST-CMTF across seven model backends, six filtering methods, and 120 controlled tool-use tasks. GIST-CMTF achieves 97.0% task success, compared with 80.1% for top-goal CMTF and 82.9% for semantic-goal CMTF. It reduces wrong-goal execution from 19.4% under top-goal CMTF to 2.5%, while preserving the one-tool exposure of causal filtering and using substantially fewer tokens than all-tools exposure. These results suggest that reliable tool-augmented agents should validate goal state, not only tool relevance, before exposing external actions.

## 1. 核心问题与整体含义（研究动机与背景）
- **背景**：工具增强的 LLM 智能体需要在运行时决定哪些工具对当前步骤可见（工具暴露问题）。Causal Minimal Tool Filtering（CMTF）通过因果状态转移方法，仅暴露当前状态到目标所必需的下一步工具，有效减少了工具选择混淆。但 CMTF 存在一个上游假设：用户的自然语言请求已经被映射到一个确定的、符号化的目标状态（goal state）。
- **问题**：实际用户请求常是模糊的、不明确的，例如“处理我的预约”可能对应查找、总结、重新安排、取消等多个目标。若直接使用 CMTF，系统可能为一个错误的推断目标执行一条因果上合理但用户并不想要的工具路径——即**错误目标执行（wrong-goal execution）**。这是不同于普通工具选择错误的独特失败模式。
- **论文整体含义**：提出 GIST-CMTF，在 CMTF 之前增加一个目标状态推断层，先验证目标是否充分明确，再决定是应用 CMTF 还是触发澄清动作。从而将可靠工具使用的关键从“选对工具”扩展到“选对目标”。

## 2. 方法论：核心思想、关键技术细节
### 核心思想
- 将目标推断建模为与 CMTF 共享同一符号状态转移词汇的结构化预测任务。
- 当推断出的目标模糊或缺失必要变量时，不暴露下游工具，而是将**澄清作为一个因果动作**，与普通工具一样具有前置条件和效果，融入同一框架。
- 这样，系统在暴露外部 API 之前，先决策“是否应该暴露任何工具路径”。

### 关键技术细节
- **候选目标生成**：给定用户请求 `q`、当前状态 `st`、工具库 `T`，推断出候选目标集 `Gq = {(g1,p1), (g2,p2), ...}`，每个 `gi` 是符号目标状态子集，`pi` 是置信度。
- **模糊性检测**：根据置信度阈值 `τ` 和最大置信度与次大置信度的差值 `δ` 判断是否模糊；也可结合缺失变量、风险规则等。
- **澄清作为因果动作**：例如 `a_goal: {ambiguous_goal} → {goal_specified}`，`a_entity: {missing_entity} → {entity_identified}` 等。这些动作具有与普通工具相同的 precondition-effect 表示。
- **策略**：
  - 若模糊（`A=1`），暴露澄清动作；
  - 否则，将最高置信目标 `g*` 传递给 CMTF，得到下一步最小因果前沿。
- **算法流程**（Algorithm 1）：
  1. `Gq ← H(q, st, T)`

## 6. 主要结论与发现
- **GIST-CMTF 大幅提升任务成功率**：整体 97.0%，远超 All tools（53.5%）、State-aware（56.4%）、Top-goal CMTF（80.1%）、Semantic-goal CMTF（82.9%），接近 gold-goal CMTF（99.5%）。
