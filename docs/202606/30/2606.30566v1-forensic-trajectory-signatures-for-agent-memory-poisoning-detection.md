## 摘要
我们发现在持续记忆投毒条件下，LLM智能体存在一种行为不变性：在通过可观察的记忆工具调用检索路由信息的架构中，成功的攻击需要在email_send_email之前调用memory_recall_fact，而这一转换在非窃密会话中很少出现。在该评估架构下，这一不变性源于攻击的信息检索依赖性，而非仅仅是一种经验相关性，抑制它会导致攻击失败。仅利用这一不变性的简单规则实现了AUC=0.9563。基于19个轨迹特征的随机森林分类器将AUC提升至0.9904（BCa 95%置信区间[0.987, 0.993]，N=10000次重采样），表明攻击在多个独立行为通道上留下了印记。该特征具有过确定性：移除所有与回忆相关的特征（特征集的一半）后，AUC仍保持在0.990不变，证实了记忆投毒会引发分布式的轨迹特征，而非单一的可观察异常。对9个模型（7B-120B参数）的跨模型留出测试确认，在6/9的留出分区上AUC=1.000，所有三个例外均得到了机制性解释。该不变性可泛化至前沿模型（GPT-4.1, GPT-4o）而无需重新训练。仅使用严格前缀的变体实现了AUC=0.934，表明实时阻断在适度性能下降下是可行的。该边界具有取证价值：绕过记忆的提示注入攻击会产生不同的轨迹（得分=0.541），使得事件响应人员仅凭工具调用日志即可区分记忆通道攻击与提示注入攻击。

## Abstract
We discover a behavioral invariant in LLM agents under persistent memory poisoning: in architectures where routing information is retrieved through observable memory-tool invocations, successful attacks require calling memory_recall_fact before email_send_email, a transition that non-exfiltrating sessions rarely exhibit. Under the evaluated architecture, this invariant follows from the attack's information-retrieval dependency rather than being merely an empirical correlation, and suppressing it breaks the attack. A simple rule exploiting this invariant alone achieves AUC = 0.9563. A Random Forest classifier over 19 trajectory features refines it to AUC = 0.9904 (BCa 95% CI [0.987, 0.993], N=10,000 resamples), demonstrating that the attack imprints on multiple independent behavioral channels. The signature is overdetermined: removing all recall-related features (half the feature set) leaves AUC unchanged at 0.990, confirming that memory poisoning induces a distributed trajectory signature rather than a single observable anomaly. Cross-model hold-out on 9 models (7B-120B parameters) confirms AUC = 1.000 on 6/9 hold-out splits, with all three exceptions mechanistically explained. The invariant generalizes to frontier models (GPT-4.1, GPT-4o) without retraining. A strictly prefix-only variant achieves AUC = 0.934, suggesting that real-time blocking is feasible with moderate degradation. The boundary is forensically useful: prompt-injection attacks that bypass memory produce a distinct trajectory (score = 0.541), enabling incident responders to distinguish memory-channel attacks from prompt-injection attacks using tool-call logs alone.

1. **行为不变性发现**：在记忆通道投毒攻击中，`memory_recall_fact` 必定在 `email_send_email` 之前调用，这是由攻击的信息检索依赖强制决定的，不是经验巧合。
2. **简单规则高效**：仅利用该不变性的规则（`recall_before_send`）实现 AUC=0.9563，召回率 0.9792。
3. **全特征分类器**：随机森林在 19 特征上达到 AUC=0.9904（BCa 95% CI [0.987, 0.993]），召回率 0.9838，假阴性仅 31 例（集中于 Memory Sandbox 条件下的 qwq:32b 隐式绕过）。
4. **过确定性**：移除所有与 recall 相关的 9 个特征后 AUC 不变，说明攻击在多维行为通道留下印记，难以通过单点规避。
5. **跨模型泛化**：6/9 开源模型留出测试 AUC=1.000；前沿模型 GPT-4.1/4o 上无需重新训练即达 Recall=1.000。
6. **前缀实时阻断可行**：严格前缀变体 AUC=0.934，适合在线拦截；全会话版本 AUC=0.990 适合取证审计。
7. **攻击向量区分**：提示注入攻击的轨迹得分均值 0.541，远低于记忆通道攻击（均值 1.000），可从工具调用日志中区分攻击来源。

## 7. 优点

- **机制性基础**：检测依据并非单纯统计相关，而是攻击的强制信息依赖，因此对自适应敌手具有结构鲁棒性（抑制 recall 调用则攻击失败）。
- **低部署门槛**：仅需工具调用日志（主流 agent 框架已原生生成），无需模型权重、激活值或内存存储访问。
- **特征过确定**：攻击在多维度留下踪迹，难以通过简单扰动掩盖。
- **跨模型与前沿泛化**：在多种参数规模和训练组织的模型上表现一致，且零样本迁移到 GPT-4.1/4o。
- **双模式部署**：支持前缀实时阻断和全会话取证，适应不同时延要求。
- **攻击向量归因**：能通过轨迹差异区分记忆投毒与提示注入，辅助事件响应。

## 8. 不足与局限

- **假阴性集中于特定模型**：qwq:32b 在 Memory Sandbox 下通过 RAG 回退隐式绕过，不调用 `memory_recall_fact`，导致 31 次漏检（所有 FN）。该攻击属于不同攻击类，不在检测范围。
- **无真实良性基线**：报告的假阳性率（10.3%）仅针对“中毒但防御成功”的会话，未评估普通无攻击流量下的误报率。后续需补充无攻击工作负载。
- **前沿模型假阳性率高**：在 GPT-4.1/4o/o3-mini/o4-mini 的非攻击会话上，因推理模型频繁调用 `recall_fact` 进行自验证，FPR 升至 23.0%（而训练集 FPR 为 10.3%）。可能需要每模型阈值校准。
- **训练数据范围有限**：仅基于开源模型（Ollama），前沿模型验证样本量小（N=20）。更大规模的前沿训练集可改善决策边界。
- **仅针对单一攻击类型**：论文仅覆盖 DTA（延迟触发攻击，通过 RAG 注入 + 持久记忆 + 后续窃密），未验证其他记忆投毒变体（如直接写入、跨会话串通等）。
- **自适应敌手场景**：讨论了三种可能逃避策略，但未做完整的对抗性实验验证。例如，敌手可尝试使用 RAG 重新检索而非记忆工具来获取地址（违反架构假设）。
