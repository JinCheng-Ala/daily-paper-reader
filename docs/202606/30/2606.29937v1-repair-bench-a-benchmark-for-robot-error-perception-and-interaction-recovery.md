## 摘要
理解用户如何感知和应对机器人故障对于构建稳健且可信赖的机器人系统至关重要。然而，以往的研究（i）通常将故障视为独立事件，（ii）侧重于二元故障检测，（iii）采用基于规则的恢复建模。我们提出了REPAIR-Bench，该基准基于41名参与者的214次交互试验，涵盖四种诱发故障类型，并提供同步的面部动作单元、头部姿态、语音转录文本以及交互后的情感和恢复报告。该基准包含三项新颖的评估任务，共同捕获人机交互（HRI）中故障的生命周期：（i）在相互依赖的交互会话中进行故障检测，模拟用户在反复故障下的纵向适应；（ii）超越二元成功/故障公式的视觉故障类型分类；（iii）以用户为中心的恢复预测，从交互上下文中推断用户偏好的恢复策略，而非依赖手动设计或基于规则的策略。在基线实验中，层次递归建模相比单会话模型（严格F1：0.80 vs. 0.68）提高了故障检测性能，故障定位平均符号误差为-0.51秒，中位绝对误差为2.97秒；在恢复预测方面，经QLoRA微调的Mistral-7B达到了Hit@5=0.76和F1@5=0.32。REPAIR-Bench为HRI和医学HRI社区提供了一个标准化框架，用于（1）评估机器人故障和（2）构建透明、自适应且可信赖的恢复系统。

## Abstract
Understanding how users perceive and respond to robot failures is essential for building robust and trustworthy robot systems. Prior work, however, (i) often treats failures as independent events, (ii) emphasizes binary failure detection, (iii) with rule-based recovery modeling. We present REPAIR-Bench, built on 214 interaction trials from 41 participants, the benchmark spans four induced failure types and provides synchronized facial action units, head pose, speech transcripts, and post-interaction affect and recovery reports. The benchmark spans three novel evaluation tasks that jointly capture the lifecycle of failure in human-robot interaction (HRI): (i) failure detection over inter-dependent interaction sessions, modeling longitudinal user adaptation across repeated failures; (ii) visual failure-type classification beyond binary success/failure formulations; and (iii) user-centered recovery prediction, inferring users' preferred recovery strategies from interaction context rather than relying on manually designed or rule-based strategies. In baseline experiments, hierarchical recurrent modeling improved failure detection over a single-session model (strict F1: 0.80 vs. 0.68), achieved a failure localization mean signed error of -0.51 s, median absolute error of 2.97 s and, for recovery prediction, a QLoRA-tuned Mistral-7B reached Hit@5=0.76 and F1@5=0.32. REPAIR-Bench provides both the HRI and Medical HRI communities with a standardized framework for (1) evaluating robot failures and (2) building transparent, adaptive, and trustworthy recovery systems.

- **研究动机**：机器人（尤其医疗场景）在实际部署中常因感知错误、时间压力、用户意图误解等出现故障。有效感知和恢复故障对任务完成、信任建立和长期接受至关重要。

  - 恢复策略往往由研究者手动设计或基于规则，而非从用户行为中预测；

- **本文目标**：提出REPAIR-Bench——首个联合解决故障时序检测、故障类型分类和恢复策略预测的端到端多模态基准，为HRI和医疗HRI社区提供标准化评估框架。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将机器人故障生命周期拆解为三个相互关联的任务，利用用户视觉行为信号（检测和分类）和对话文本（恢复预测），并引入跨会话历史建模以捕获用户适应过程。
- **数据来源与预处理**：
  - 基于RFM-HRI数据集（214次试验，41名参与者，4种诱发故障+成功条件）。
  - 排除30个重复/信息不足样本，最终184个有效试验。
  - 使用MediaPipe提取面部动作单元(AU)、3D头部姿态、注视等视觉特征；Whisper ASR提取转录文本。
  - 视觉特征经6Hz Butterworth滤波、范围归一化；转录仅保留自然语言对话。
  - 故障时序标注：故障偏移（机器人说完错误语句的时刻）作为过渡点。
- **任务1：跨会话故障检测**
  - 输入：滑动窗口提取的视觉特征统计量（均值、最小、最大、标准差），每个窗口21通道（18 AU + 3 头部姿态）。
  - 输出：每个窗口的二进制标签（故障前/后）和过渡时刻（秒）。
  - 架构：两层层次递归网络（HRNN）。底层GRU处理当前试验窗口序列，顶层GRU维护跨试验历史状态，通过桥接层初始化底层隐藏状态。
  - 损失：窗口交叉熵（带类别权重和标签平滑）+ Huber定位损失。
  - 训练：按参与者拆分（75%/25%），参与者内按时间顺序迭代，状态跨试验传递但不跨参与者。
  - 处理挑战：高维噪声特征、视觉信号微弱，采用平滑窗口和统计特征降噪。
- **任务2：故障类型分类**
  - 输入：帧级视觉特征（AU、面部landmarks、注视、头部姿态）。
  - 输出：5类（4种故障+成功）。
  - 架构：两层BiLSTM编码帧序列，经注意力聚合为试验嵌入；上层GRU按时间顺序处理同一参与者的试验嵌入，结合当前试验嵌入和上下文输出分类。
  - 训练：参与者级拆分，试验按时间顺序组织，保证跨试验依赖。
- **任务3：用户偏好恢复预测**
  - 输入：对话转录+故障标签+候选恢复策略集合。
  - 输出：逗号分隔的top-k恢复策略（有序集合）。
  - 建模方式：约束生成排序问题，禁止额外解释。
  - 模型：基座Mistral-7B、Gemma-9B，以及经QLoRA微调的变体。
  - 微调：4-bit NF4量化 + 双重量化 + bfloat16，LoRA适配器注入注意力和MLP层，仅更新适配器参数。
  - 训练prompt：角色定义、输出格式约束、转录、故障标签、候选策略。
  - 评估：Hit@k、Precision@k、Recall@k、F1@k。

## 3. 实验设计

- **数据集**：RFM-HRI（医疗场景下移动急救车引导物品取回任务），包含实验室和医院两个采集环境，184个有效样本（排除后）。四种故障类型：语音故障、时序故障、理解故障、搜索故障；外加成功条件。
- **基准任务**：
  - Task 1：故障检测（窗口级二分类+过渡时刻定位）。
  - Task 2：故障类型分类（5类）。
  - Task 3：恢复策略预测（top-k输出）。
- **对比方法**：
  - Task 1：随机森林(RF)、GRU、BiLSTM（含/不含历史特征）、三层HRNN、两层HRNN（3秒窗、5帧窗）。
  - Task 2：BiLSTM-GRU vs 两层HRNN（参数量相同）。
  - Task 3：Mistral-7B base vs QLoRA、Gemma-9B base vs QLoRA。
- **评估指标**：
  - Task 1：窗口级准确率、平衡准确率、精度、召回率、F1、AUC；定位误差(MAE、MedAE、均值符号误差)；时间IoU阈值(0.1,0.25,0.5,0.75)下的F1。
  - Task 2：macro P、R、F1；每类P、R、F1。
  - Task 3：Hit@k、Precision@k、Recall@k、F1@k (k=1~5)。

## 6. 主要结论与发现

- **Task 1（故障检测）**：
  - 两层HRNN（3秒窗口）取得最佳性能：窗口级严格F1=0.80，AUC=0.87；过渡定位中位绝对误差2.97秒，平均符号误差-0.51秒（略偏早预测），mIoU=0.78。
  - 跨会话历史建模（层次结构）显著优于单会话模型（F1提升0.12）。
  - 纯视觉信号（无语音）足以实现有效检测，但噪声较大。
- **Task 2（故障分类）**：
  - HRNN取得最佳macro F1=0.44，优于BiLSTM-GRU的0.38。
  - 成功类分类极好（F1=0.93），但理解（0.22）和时序（0.29）类困难；自报告情绪与故障类型有弱关联（Cramér’s V=0.386）。
- **Task 3（恢复预测）**：
  - 基座Mistral-7B优于Gemma-9B（后者近乎随机）。
  - QLoRA微调后性能持续提升，Mistral-7B QLoRA在Hit@5=0.76、F1@5=0.32。
  - 微调主要提升高k值下的召回和Hit，表明模型能捕捉多种用户偏好策略。
- **总体**：REPAIR-Bench为系统研究机器人故障提供了标准化框架，层次/跨会话建模和微调LLM对自适应恢复至关重要。

## 7. 优点

1. **任务整合创新**：首次将故障检测、分类和恢复预测统一在一个基准中，反映故障全生命周期。
2. **跨会话历史建模**：突破独立事件假设，通过HRNN捕获用户对重复故障的行为适应，更贴近真实场景。
3. **用户中心恢复**：从交互数据直接学习用户偏好的恢复策略，避免人工规则的主观性。
4. **多模态与黑箱信号利用**：任务1/2仅用视觉信号（面部AU、头部姿态），证明非语言早期指标的有效性；任务3结合文本，覆盖互补模态。
5. **标准化评估框架**：提供明确的评估协议、公开模型基线、公平对比（参数数量、拆分方式等），便于后续研究复现和比较。
6. **医学场景价值**：基于医疗急救车交互数据，直接面向高安全需求应用。

## 8. 不足与局限

1. **数据规模有限**：仅184个有效样本（41参与者），可能限制模型泛化性和统计可靠性；恢复预测任务更是小样本学习。
2. **恢复策略集合预设**：实验中用户仅从给定策略列表中选择，未涵盖开放式或自由表达偏好，存在生态效度问题。
3. **仅依赖视觉信号任务1/2的挑战**：面部表情噪声大且故障反应细微，人类观察者也常难以区分；排除语音信号虽属有意设计，但可能丢失强早期线索。
4. **场景单一**：仅基于特定医疗急救车取回任务、特定机器人平台，结果泛化到其他HRI场景（如社交机器人、服务机器人）需验证。
5. **分类性能不均衡**：理解、时序故障分类F1低（<0.30），模型难以区分某些故障类型，可能因表情差异小或标签噪声。
6. **未充分分析与偏见**：未报告不同人口统计属性、实验室vs医院环境的效果差异；未讨论时序标注的共识可靠性（仅提到100%共识，未报告Kappa系数）。
