## 摘要
在知识密集型问答中，LLM智能体进行检索和推理行动时，对其当前答案是否不确定、缺乏支持或已经完整知之甚少。这导致两种失败模式：一是给出自信但缺乏支持的答案，损害准确性；二是在已有足够证据时过度检索，造成计算浪费。为了给智能体提供其操作状态空间的更完整视图，我们引入了校准验证器遥测（CalVerT），它通过额外的遥测信息增强智能体状态：校准的自信心分数和基础验证器分数。我们证明，CalVerT可以在无训练和基于训练的设置中改进智能体。在四个问答基准上，我们发现CalVerT通过触发对智能体过度依赖参数知识的案例进行检索来提高F1值，同时在智能体有足够上下文回答的案例中减少冗余检索。我们表明，CalVerT可以在无需训练的情况下增强现有问答框架。此外，CalVerT还改进了训练系统：只需用遥测信息增强智能体状态，与经过相同训练但没有CalVerT遥测的智能体相比，我们在强化学习后观察到改进。

## Abstract
LLM agents in knowledge intensive question answering take retrieval and reasoning actions with incomplete knowledge about whether their current answer is uncertain, unsupported, or already complete. This produces two failure modes: committing to confident but unsupported answers, which hurts accuracy, and over-retrieving when the evidence in hand already suffices, resulting in wasted compute. To give agents a more complete picture of the state space they are operating in, we introduce calibrated verifier telemetry (CalVerT), which augments the agent's state with additional telemetry: a calibrated self-confidence score and a grounding verifier score. We show that CalVerT can improve agents in both training-free and training-based settings. On four QA benchmarks, we find that CalVerT raises F1 by triggering retrieval in cases where agents over-rely on parametric knowledge, while cutting redundant retrieval in cases where agents have sufficient context to answer. We show that CalVerT can augment existing QA frameworks without training. Moreover, CalVerT also improves trained systems: by simply augmenting an agent's state with telemetry, we observe improvements after reinforcement learning, as compared to an agent with identical training but no CalVerT telemetry.

## 1. 核心问题与整体含义（研究动机和背景）

*   **问题**：大规模语言模型（LLM）智能体在知识密集型问答（如多跳检索、工具使用）中，需要反复执行**检索、反思、修订**等动作。然而，智能体在执行这些动作时，**缺乏对其当前答案状态（是否确定、是否有证据支撑、是否已经完整）的清晰认知**，导致了两种典型失败模式：
    *   **参数过度信任（Parametric Over-trust）**：智能体过于依赖自身可能有误的参数化知识，贸然给出自信但无依据的答案，损害准确性。
    *   **过度检索（Over-retrieval）**：当手中已有足够证据时，智能体仍继续检索，浪费计算资源（也可能因噪音降低答案质量）。
*   **整体含义**：现有智能体方法（如ReAct, Self-RAG, Search-R1 等）通常缺乏对当前答案质量的**测量能力**，难以在“是否已够好”与“是否需要更多信息”之间取得平衡。因此，论文提出通过在智能体状态空间中引入**校准的遥测信号**，让智能体拥有更完整的“仪表盘”，从而做出更好的行动决策。

## 2. 方法论

### 核心思想
在智能体的**每轮状态表示**中，额外添加两类遥测信号，帮助其判断何时应执行 {提交(commit), 检索(retrieve), 优化(refine), 分解(decompose)} 四种动作之一。信号不加硬编码阈值，而是作为自然语言描述插入提示，让智能体自主推理。

### 关键技术细节
1.  **遥测信号来源**：
    *   **自信信号（Self-Confidence）**：使用 **DINC O** 方法（Wang & Stengel-Eskin, 2026）从模型本身获取：
        *   `NVC`（归一化口头自信）：通过提示模型“这个答案正确吗？是/否”并归一化得到。
        *   `SC`（自一致性率）：多个独立采样中与主要答案一致的样本比例。
        *   最终使用 `final_conf` = 0.5*(NVC + SC)，范围 [0,1]。
    *   **基础信号（Grounding）**：使用 **MiniCheck-7B**（Tang et al., 2024）验证器，将候选答案分解为声明（claims），并评估每个声明相对于当前检索证据池的蕴含程度：
        *   `g_mean`：所有声明的平均蕴含概率。
        *   `g_min`：最薄弱声明的蕴含概率（突出最弱环节）。
2.  **智能体循环**：
    *   采用 ReAct 风格循环。规划器将问题分解为子目标 DAG。
    *   每轮：当前子目标上，智能体基于已有证据和遥测信号的文本描述，选择动作。
    *   自信信号在首次访问子目标时计算并缓存；基础信号每轮根据当前候选答案和证据重新计算。
    *   动作定义：`commit`（提交答案并存储支持事实），`retrieve`（检索新证据），`refine`（基于现有证据重生成答案），`decompose`（放弃当前子问题并拆解）。
3.  **两种使用方式**：
    *   **无训练（Prompt-only）**：将四个标量及其自然语言含义直接插入每轮提示。用于冻结的生成器，对比有无遥测。
    *   **基于训练（GRPO）**：遵循 Search-R1 (Jin et al., 2025)，使用 GRPO 强化学习训练智能体的 LoRA 适配器。奖励函数为最终答案质量（F1+EM奖励）减去动作成本（检索、优化等额外惩罚）。遥测信号作为状态的一部分参与决策。

## 6. 主要结论与发现

1.  **CalVerT 提升无训练智能体性能**：给冻结的生成器暴露遥测信号后，在多数基准上 F1 提升，且动作次数减少（表明减少无效检索）。例如 Qwen3-32B 在 2Wiki 上 F1 提升 +3.7，WiTQA 上 +4.7。
2.  **框架可移植性**：替换五种现有框架的门控信号后，四种 F1 提升（最高 TARG +15.4 F1），一种保持相同但检索成本下降 34 次。
3.  **遥测信号相互补充**：单独使用自信信号或基础信号均不如两者结合（表10）。自信信号引导是否检索，基础信号引导是否提交。
4.  **训练受益更大**：GRPO 训练下，带遥测信号的智能体远优于无遥测训练（Qwen3-8B: +5.9% F1；30B-A3B: +3.3% F1），而无遥测训练仅产生微小改进。
5.  **信号效果随难度增长**：在多跳基准上，增益随跳数增加而增大；在单跳基准上，增益集中在罕见实体上（而非常见实体）。
6.  **小模型需要训练**：Qwen3-8B 在无训练 prompt 下无法有效利用遥测（性能下降），但通过 GRPO 训练可恢复并超越基线。

## 7. 优点

*   **简单且框架无关**：无需修改基础架构或重新训练，可直接作为内置信号插入任何 ReAct 风格智能体的提示。
*   **双信号正交互补**：同时提供“模型内部自信”和“证据外部支撑”两个正交维度，克服单信号（如仅自信或仅基础）的局限性。
*   **无需硬编码阈值**：将数字信号以自然语言描述形式呈现，让智能体自主推理，增加了灵活性。
*   **促进科学公平：实验设计严谨**：严格保持消融实验的变量控制（如相同规划器、检索器、合成器），确认遥测是唯一变化因素。
*   **实用性验证深入**：不仅验证了 prompt-only，还验证了框架替换和训练场景，覆盖了当前主流用法。
*   **可视化分析详尽**：展示了动作分布、难度分层、计算开销等详细分析，增强了结果的可解释性。

## 8. 不足与局限

1.  **动作词汇以提交为中心**：当前设计假定每个子问题只需一个答案（适用于标准多跳 QA），但**不适用于需要多个正确答案的召回任务**（如 “列出所有…”）。这是一个信号无关的扩展点，留待未来工作。
2.  **训练实验泛化性有限**：仅使用 HotpotQA distractor 的 hard 子集（1600 训练 + 200 评估），未在其他多跳数据集或单跳任务上训练验证，可能不能直接推广到其他领域。
3.  **计算开销较大**：无训练设置下遥测计算（DINC O + MiniCheck）带来约 200-247% 的额外 TFLOPs（表7），这在延迟敏感或预算严格的应用中可能不可接受。
4.  **闭源模型评估受限**：仅测试了 gpt-4o 和 gpt-4.1（均较旧版本），因为新模型不暴露 logprobs 限制了 DINC O 的使用。因此结论可能不适用于最新闭源模型。
5.  **小模型的prompt-only受限**：Qwen3-8B 在无训练下无法有效利用遥测，表明该方法在较小模型上需要额外训练才能发挥效用。
6.  **未探索其他检索策略**：所有实验使用固定 BM25 + 重排序器，未测试与基于嵌入的检索、迭代检索等其他策略的交互。
7.  **未评估长程对话或多工具场景**：论文聚焦于知识密集型 QA，但现代智能体大量出现在网页任务、编程、工具编排等场景；CalVerT 在这些场景的适用性未验证。
