## 摘要
面向LLM智能体的技能自进化方法旨在将执行轨迹转化为可复用的技能文档，但当前流程通常每个任务仅学习一条轨迹，在检查候选技能补丁前进行合并，并在推理前加载完整技能语料库。我们提出SkillCAT，一个无需训练框架，将该过程分为三个阶段。对比因果提取（CCE）为每个任务采样多条轨迹，并比较同任务成功/失败对，以识别解释结果差异的证据。评估增强进化（AAE）在源任务克隆上回放每个候选补丁，并在层级技能补丁合并前仅保留改进或保持任务结果的补丁。拓扑感知任务执行（TTE）将进化后的技能编译为可路由的子技能拓扑，因此推理时仅加载与任务相关的能力节点。我们在常见智能体基准测试（包括SpreadsheetBench、WikiTableQuestions和DocVQA）上评估SkillCAT，并进一步测试跨模型和分布外泛化能力。在这些设置下，SkillCAT将基线平均得分提升高达40.40%，展示了无需模型训练的可靠技能进化。

## Abstract
Skill self-evolution methods for LLM agents aim to turn execution trajectories into reusable skill documents, but current pipelines typically learn from one trajectory per task, merge candidate skill patches before checking them, and load the full skill corpus before inference. We propose SkillCAT, a training-free framework that separates this process into three stages. Contrastive Causal Extraction (CCE) samples multiple trajectories for each task and compares same-task success/failure pairs to identify evidence that explains outcome differences. Assessment-Augmented Evolution (AAE) replays each candidate patch on source-task clones and keeps only patches that improve or preserve task outcomes before hierarchical skill patch merging. Topology-Aware Task Execution (TTE) compiles the evolved skills into a routable sub-skill topology, so inference loads only the capability nodes relevant to the task. We evaluate SkillCAT on common agent benchmarks, including SpreadsheetBench, WikiTableQuestions, and DocVQA, and further test cross-model and out-of-distribution generalization. Across these settings, SkillCAT raises the average score over baselines by up to 40.40%, demonstrating reliable skill evolution without model training.

# SkillCAT 论文详细总结

## 1. 论文的核心问题与整体含义（研究动机与背景）

- **背景**：LLM 智能体在执行任务时通常依赖外部技能文档（结构化指令、工具使用规则、任务经验等），这些技能可以通过从执行轨迹中提取与精炼来自动进化，称为“技能自进化”（skill self-evolution）。
- **现有方法的局限**：
  - **单轨迹偏差**：每任务仅用一条轨迹提取技能，成功轨迹可能反映偶然策略，失败轨迹难以定位根本原因。
  - **未验证合并**：候选技能补丁在没有独立验证其对源任务效果的情况下直接合并，低质量或有害补丁会污染技能库。
  - **上下文过载**：进化后的技能持续增长，推理时加载全部技能导致提示过长，干扰任务相关内容的利用。
- **本文目标**：提出 SkillCAT，一个免训练的框架，通过分离技能生命周期为三个可观察阶段来解决上述问题，实现可靠的技能进化。

## 2. 论文提出的方法论

### 2.1 核心思想
将技能自进化解耦为三个阶段：**对比因果提取（CCE）**、**评估增强进化（AAE）** 和 **拓扑感知任务执行（TTE）**。前两阶段离线进行技能学习，第三阶段在线部署。

### 2.2 关键技术细节

#### (a) 对比因果提取（CCE）
- **多种子采样**：对每个进化任务使用多个随机种子生成多条轨迹。
- **构建对比对**：对同一任务的成功与失败轨迹组成对比对（`τ⁺, τ⁻`），消除任务难度和输入差异的影响。
- **因果分水岭识别**：定位成功与失败轨迹动作序列首次出现差异的时间步 `w_i`。
- **记录提取**：围绕分水岭提取候选经验记录 `r_i`，包括局部证据、失败原因和可编辑的经验教训，而非总结整个轨迹。
- **备选机制**：若某任务所有轨迹全部成功或全部失败，则使用单轨迹提取器或跳过该任务。

#### (b) 评估增强进化（AAE）
- **源任务回放**：将每个候选补丁 `p_j` 作为临时技能在其源任务克隆上重新执行，记录基线结果 `y_j` 与回放结果 `ŷ_j`。
- **得分校准**：根据结果转移定义四级评分：
  - 失败→成功：3.0（最强证据）
  - 成功→成功：2.0（行为保持）
  - 失败→失败：1.0（低效）
  - 成功→失败：0.0（拒绝）
- **阈值选择**：仅保留得分 ≥ 阈值 θ（实验中设为 2.0）的补丁。
- **分层合并**：将保留补丁按得分层级从低到高排序，使用类似 Trace2Skill 的合并算子逐步融入基础技能 `S_0`，得到最终进化技能 `S*`。

#### (c) 拓扑感知任务执行（TTE）
- **拓扑构建**：离线将 `S*` 的 Markdown 层级解析为核心技能 `S°` 和可路由节点集合 `V`（二级章节），每个节点保留正文和路由元数据（标题、关键词、摘要、依赖）。
- **路由与组装**：推理时，LLM 路由器根据任务描述 `c` 从拓扑摘要中选择最相关节点（预算 `k=7`），经过确定性扩展（验证标识符、添加依赖边和基础节点）后，将核心技能与选定节点正文拼接成路由技能 `S_c`，仅加载任务相关上下文。

### 2.3 算法流程简示
1. 对每个任务运行多种子 → 收集轨迹集 `T_i`。
2. 若混合结果存在，构建对比对并提取对比记录；否则启用备选机制。
3. 将记录转为候选补丁，进行源任务回放评分。
4. 阈值筛选后分层合并更新技能。
5. 编译技能拓扑；推理时路由选择并组装上下文，供智能体执行。

## 6. 论文的主要结论与发现

- **性能大幅提升**：在 SpreadsheetBench-Verified 上，SkillCAT 相较 Trace2Skill +Combined 提升最高达 40.40% 的平均得分（如 Qwen3.5-35B-A3B Deepening 设置下 Vrf 提升 25.83 百分点）。
- **各模块均有贡献**：消融实验表明，去除任意模块（CCE、AAE、TTE）均导致性能下降，其中去除 AAE 后性能低于 Trace2Skill，说明回放验证至关重要。
- **跨模型可迁移**：进化后的技能可以泛化到未见过的用户模型（gemma-4、gpt-5.4-mini），但收益因模型能力而异。

- **拓扑路由有效减负**：TTE 在保持高性能的同时减少约 41.6% 上下文长度。
