## 摘要
通过从专家轨迹中进行模仿学习来训练交互式网络代理已成为一种高效的方法。然而，确定专家干预的最佳时机在此背景下提出了关键挑战。延迟干预通常会导致早期错误的积累，将页面状态推入不可恢复的境地。相反，过早或过度干预则会使代理过度依赖专家策略，将模型困在以单一、僵化轨迹为特征的局部最优中。我们提出了推测性回滚修正（SRC），一种针对可重置代理环境的分支级模仿框架。SRC并非在每个访问状态都请求教师标签，也不仅在完成轨迹后才进行修正，而是采用固定视野分支审查：学生执行一小段推测性片段，然后由教师审查，仅在局部进展中断时定位首个有害偏离。回滚保留了有用的前缀，同时成功的展开由硬验证器过滤，并保留在轻量级的质量多样性存档中。生成的数据支持对定位修正和验证器通过的轨迹进行下一动作监督微调。在WebArena-Infinity上，SRC收集了977条验证器通过的轨迹和9183个下一动作示例；固定视野审查在保留验证器通过解变体的同时，优化了恢复与查询的权衡。代码地址：https://github.com/LongkunHao/SRC_gui_agent。

## Abstract
Training interactive web agents through imitation learning from expert trajectories has emerged as a highly effective approach. However, determining the optimal timing for expert intervention presents a critical challenge in this context. Delayed intervention often leads to the accumulation of early-stage errors, pushing the page state into an irrecoverable regime. Conversely, premature or excessive intervention causes the agent to become overly reliant on expert policies, trapping the model in local optima characterized by a single, rigid trajectory. We propose Speculative Rollback Correction (SRC), a branch-level imitation framework for resettable agent environments. Instead of requesting teacher labels at every visited state or correcting only after a completed trajectory, SRC uses fixed-horizon branch review: the student executes a short speculative segment before teacher review, and the teacher localizes the first harmful deviation only when local progress breaks. Rollback preserves useful prefixes, while successful rollouts are filtered by a hard verifier and retained in a lightweight quality-diversity archive. The resulting data supports next-action supervised fine-tuning on both localized corrections and verifier-passing trajectories. On WebArena-Infinity, SRC collects 977 verifier-passing trajectories and 9,183 next-action examples; fixed-horizon review improves the recovery-versus-query tradeoff over step-level review while retaining verifier-passing solution variants. Code is available at https://github.com/LongkunHao/SRC_gui_agent.

- **问题背景**：通过模仿学习训练网页/GUI智能体时，标准行为克隆存在**分布漂移**问题：学生策略的动作会导致自身状态偏离专家轨迹，早期错误累积后使后续状态不可恢复。同时，同一任务往往存在多条可行的成功路径，但单一专家示范会压制这些多样性，将有用探索误判为错误。
- **关键挑战**：专家干预的时机难以平衡——延迟干预导致错误无法挽回，过早/过度干预则使学生过度依赖专家，陷入单一路径的局部最优。
- **现有方法不足**：DAgger 要求每步查询专家；后验修正（如 LEAP）在错误污染后续状态后才介入；随机切换（如 OEC）浪费修正机会。

## 2. 方法论：核心思想、关键技术细节

### 核心思想：推测性回滚修正（SRC）

- 将专家监督从“每步查询”或“完整轨迹后修正”转向**短分支级别审查**：学生先执行一个固定长度 K 的动作分支，教师判断分支是否保持了局部进展；仅当进展中断时，定位首个有害动作索引并回滚，保留有用前缀，然后由教师在该状态生成修正动作。
- 分离三种角色：
  - **教师审查器**：判断局部进展（是否保持可恢复、任务导向）
  - **硬验证器**：判断最终成功（轨迹是否通过任务验证）
  - **质量多样性存档**：保留多种高效且通过验证的轨迹，按行为描述子（路径长度桶、主导动作类型、干预次数桶）分类，过滤低质量轨迹（过长、重复、过多干预）

### 关键技术细节

- **固定视野分支**：学生执行最多 K 个动作，然后教师返回 ACCEPT 或首个有害索引 j。若接受，则所有动作提交；若拒绝，保留前缀 0..j-1，环境重置回滚到 j-1 状态，教师生成修正动作，然后恢复学生继续执行。
- **多叶收集**：可选保留被拒绝的学生分支作为独立“叶子”，完成并验证，从而在单次收集中获得混合轨迹（学生、教师修正、混合）。
- **训练数据**：回滚修正标签（D_corr）+ 存档轨迹中提取的动作标签（D_arc），使用下一动作监督微调，无奖励建模或偏好优化。

## 6. 主要结论与发现

- SRC 在三个基准上均优于 Expert SFT：WebArena-Infinity +9.7 SR，WebArena-Lite +3.5 SR，OSWorld 子集 +12.9 SR。
- 中等分支视野 K=3 取得最佳权衡（51.9% SR，教师查询数 2529），优于步级修正（K=1）和长分支（K=7）。
- 训练数据中仅 14.2% 来自教师干预，85.8% 来自学生接受的分支，表明 SRC 不主导收集成本。
- 存档覆盖范围从 147 个描述子桶增长到 259，验证了多样性保持。
- SRC 缩小了开源模型与闭源模型（Kimi K2.5, Qwen3.5 Plus）之间的差距。

## 7. 优点

- **分支级审查**创新性地解决了干预时机困境：延迟但定位精确，保留有用探索。
- **质量多样性存档**保证了训练数据不坍缩到单一教师路径，保留多种有效解法。
- **角色分离**（局部进展判断、全局成功验证、多样性保留）设计清晰，避免了概念混淆。
- **模型和模态无关**：适用于文本、DOM、截图等多种输入；代码开源。
- **算法简洁**：无需奖励建模或对抗训练，仅用下一动作监督微调。

## 8. 不足与局限

- **环境假设**：要求训练环境可重置回放，不适用于不可重置的在线场景、隐含状态变化的网站或不可逆工作流。
- **固定审查视野 K**：单任务可能不是最优，未来需要自适应分支长度（如基于子目标分解）。
- **域外验证不足**：仅在网页和桌面 GUI 上评估，未涉及 Android 等移动端。
