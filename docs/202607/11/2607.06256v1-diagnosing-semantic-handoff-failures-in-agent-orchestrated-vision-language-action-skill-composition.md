## 摘要
长期家庭任务要求机器人组合多种语言条件技能，但连续技能之间的边界很少明确。一个技能可能满足其自身后置条件，同时使机器人、物体或相机视图处于下一个技能无法可靠启动的状态。我们在BEHAVIOR-1K中通过智能体编排的视觉-语言-动作执行框架研究这一语义交接问题。该框架调用基于π₀.5的技能检查点，这些检查点使用清洗后的BEHAVIOR-1K演示训练，为每个技能分配类型化参数和步骤预算，并使用多视图视觉-语言模型验证来决定执行应前进、重试还是重新规划。为分离孤立技能能力与长期组合鲁棒性，我们在两种初始状态分布下评估相同的检查点：清洁的技能边界快照和先前技能产生的链式终端状态。在人工审核验证下，选定的导航、抓取、放置和开门技能从清洁快照中实现了77-100%的成功率，然而组合的执行过程仍频繁因链式状态而停滞。执行轨迹将这些失败归因于下一技能就绪性、目标接地和底层控制执行，揭示了单技能成功与可靠长期任务完成之间的显著差距。这些发现将近乎为零的端到端任务成功转化为可操作的诊断，表明未来的VLA技能库必须学习对混乱的链式状态分布的鲁棒性，而清洁演示系统性地低估了这种分布。

## Abstract
Long-horizon household tasks require robots to compose many language-conditioned skills, yet the boundary between consecutive skills is rarely explicit. A skill may satisfy its own postcondition while leaving the robot, objects, or camera views in a state from which the next skill cannot reliably start. We study this semantic handoff problem in BEHAVIOR-1K through an agent-orchestrated vision-language-action execution harness. The harness invokes $π_{0.5}$-based skill checkpoints trained from cleaned BEHAVIOR-1K demonstrations, assigns each skill typed arguments and a step budget, and uses multi-view vision-language model verification to decide whether execution should advance, retry, or replan. To separate isolated skill competence from long-horizon compositional robustness, we evaluate the same checkpoints under two initial-state distributions: clean skill-boundary snapshots and chained terminal states produced by previous skills. Selected navigation, grasping, placement, and door-opening skills achieve 77--100% success from clean snapshots under human-reviewed verification, yet composed rollouts still frequently stall from chained states. Execution traces attribute these failures to next-skill readiness, target grounding, and low-level control execution, revealing a substantial gap between single-skill success and reliable long-horizon task completion. These findings turn near-zero end-to-end task success into actionable diagnostics, showing that future VLA skill libraries must learn robustness to the messy chained-state distribution that clean demonstrations systematically underrepresent.

## 一、核心问题与整体含义（研究动机与背景）

- **研究动机**：长期家庭任务需要机器人组合多个语言条件技能（如导航、抓取、放置等），但连续技能之间的语义边界并不明确。一个技能可能完成了自己的后置条件（如“移动到微波炉”），却使机器人、物体或相机视角处于下一个技能无法可靠启动的状态——这被定义为**语义交接失败**。

## 二、方法论：核心思想、关键技术、算法流程

### 核心思想
- 设计一个**智能体编排的VLA执行框架**，将技能调用视为具有语义契约的工具调用，通过**计划-行动-验证-重规划**循环进行任务分解、状态追踪、验证和恢复，从而诊断技能组合中的失败原因。

### 关键技术细节
1. **技能合约（Semantic Skill Contracts）**：每个技能调用 `s_t` 包含名称、参数、语言指令、步骤预算 `B_s`、验证间隔 `K_s` 和后置条件 `φ_t`。技能库包括 `move_to`、`pick_up_from`、`place_on`、`open_door` 等9种类型。
2. **VLA执行器**：使用基于**π₀.5**（PaliGemma 2B视觉-语言骨干 + 300M参数流匹配动作专家）的视觉-语言-动作策略，处理头部和两个手腕的三个224×224 RGB视图及本体感觉状态，预测多步动作块。
3. **多视图VLM验证器**：每 `K=200` 个模拟步，使用Gemini 2.5 Flash对头部、左腕、右腕视图进行多视图验证，判断技能后置条件是否满足（**置信阈值0.6**）。对于 `move_to`，额外要求目标“在手臂可及范围内”。
4. **诊断性轨迹记录**：每次执行都产出结构化的轨迹文件（含技能调度、验证决策、重规划、最终任务谓词状态等），可解析出每次失败尝试的原因归属。

### 算法流程
1. 离线阶段：从清洗后的BEHAVIOR-1K演示中提取技能片段，经过全技能中期训练（50k步）和技能组后训练（专门化检查点）。
2. 在线阶段：智能体层维护任务脚本、计划和技能合约，调度步骤级VLA技能；每 `K` 步触发多视图VLM验证；根据验证结果决定**前进/重试/重规划**；在技能预算内或全局预算内循环，直到任务谓词满足或预算耗尽。

## 六、主要结论与发现

1. **单技能从清洁快照表现良好**：`place_on`、`open_door`、`pick_up_from` 等达到77-100%成功率，技能库并非全局薄弱。
2. **组合执行崩溃**：端到端成功率近乎为零（平均进度19.5%），技能在链式初始状态下频繁停滞。例如，导航后抓取失败是因为物体/视角/姿态偏移。
3. **失败原因分类**（130次失败尝试）：
   - **控制执行（58次）**：抓取但未能闭合/保持、触到开关但未触发、放置位置不对。
   - **目标接地（37次）**：目标物体不可见或实例错误，源于导航/开窗后视角变化。
   - **下一技能就绪性（35次）**：导航“到区域”但未到“手臂可及”位置，后续技能（如抓取）无法启动。
4. **验证标准消融**：将`move_to`从“到达区域”收紧为“手臂可及”，额外发现12个就绪性失败、触发25次重导航，并恢复了“开启收音机”任务（其开关仅在机器人被迫重导航至可及姿势后成功）。
5. **验证器审计**：双注释者盲审显示，验证器标记的失败中20/21为真实失败（过严格率0.05），验证器的失败判定可靠，但失败类别标签仍是验证器导出的，可能带有偏差。

## 七、优点

1. **问题定义清晰**：首次系统界定并命名了技能组合中的“语义交接失败”，区分了“后置条件满足”与“下一技能可启动”两种不同需求。
2. **诊断视角创新**：将重点从“提升端到端成功率”转向“从失败中提取可操作诊断”，揭示了单技能成功与组合鲁棒性之间的本质差距，为改进提供了具体方向（链式状态微调、目标接地监督）。
3. **框架设计模块化**：技能合约、验证器、恢复、轨迹记录分离，便于独立改进和诊断。
4. **多视图验证**：利用头部和手腕多视角图像，更全面判断环境状态，减少单视角盲区。
5. **验证审计**：进行了人工审计验证器决策，增加了诊断结果的可靠性。
6. **消融设计**：改变单个验证条件（手臂可及标准）即产生可衡量的任务恢复效果，表明框架对验证灵敏度敏感，方向具有可操作性。

## 八、不足与局限

1. **实验覆盖有限**：
   - 端到端实验仅30次rollout（10任务×3实例），统计规模小，不能视为总体分布估计。
   - 失败原因分类基于一次代表性轮次的人工解析（196次验证调用），可能受抽样偏差影响。
2. **验证器引入偏差**：
   - 失败类别标签是验证器导出的，未经验证器类别标签本身的盲审；虽然二元决策可靠（过严格率0.05），但类别归属可能不准确（如将“控制执行”误判为“目标接地”）。
   - 验证器为专有VLM（Gemini 2.5 Flash），其决策黑箱且可能对特定场景（物体放置、抓取可见性、铰接物体状态）有偏见。
3. **对象名称模糊**：任务脚本使用了部分未充分指定的对象名（如`door`或`storage box`），增加了目标接地失败的风险。
4. **恢复机制局限**：恢复仅由智能体在预算内重规划（通常是重导航后重试），不包括与预算无关的智能恢复、“或无恢复”与重规划的消融未做，无法分离VLA技能脆弱性与恢复策略限制。
5. **未比较基线方法**：未与Code as Policies、SayCan等现有长期任务分解方法进行端到端成功率比较，无法量化同框架的相对优势。
6. **计算资源细节缺失**：未报告训练所需的GPU型号、数量及时间，降低了结果的可重现性参考价值。
7. **备选计划不评分**：进度分数基于教师提示的参考技能序列，备选有效计划不被计分，导致诊断性分数比实际能力更低（单注释者判断也引入主观性）。
