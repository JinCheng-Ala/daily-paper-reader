## 摘要
代理安全正从内容审核转向在使用工具的代理行动前预防操作失败。我们提出Janus，一种面向长期代理安全的前瞻性框架，它训练守卫从部分轨迹中预见到延迟的风险。Janus通过多智能体模拟合成多样的代理轨迹，并学习一个包含两个耦合任务的共享策略：预测与安全相关的未来状态的预测任务，以及基于观察到的前缀和预测的未来决定安全性的裁决任务。这两个任务通过CoAA-RL联合优化，该奖励机制根据预测对下游安全判断的实用性进行奖励。由此产生的守卫模型Vanguard在执行前阻止不安全行动。在四个代理安全基准测试中，Vanguard相比基线守卫将平均保护率提高了15.9个百分点，同时将良性任务完成率提高了5.1个百分点。

## Abstract
Agent safety is moving from content moderation toward preventing operational failures before tool-using agents act. We propose Janus, a foresight-oriented framework for long-horizon agent safety that trains guards to anticipate delayed risks from partial trajectories. Janus synthesizes diverse agent trajectories via multi-agent simulation and learns a shared policy with two coupled tasks: an anticipation task that forecasts safety-relevant futures and an adjudication task that decides safety from both the observed prefix and anticipated future. The two tasks are jointly optimized with CoAA-RL, which rewards forecasts by their utility for downstream safety judgment. The resulting guard model, Vanguard, blocks unsafe actions before execution. Across four agent-safety benchmarks, Vanguard improves average protection by 15.9 percentage points over baseline guards while increasing benign task completion by 5.1 percentage points.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：随着工具型大语言模型代理（如 OpenClaw、Codex、Claude Code）的部署，安全风险从文本生成扩展到**操作失败**（如删除文件、泄露数据、破坏环境）。现有安全机制多为**反应式**，仅在有害动作发生或之后才检测，对于长期任务（long-horizon）中早期信号逐渐演变为危害的情况无能为力。
- **核心问题**：如何**在有害动作执行前**，从部分轨迹中预测延迟出现的风险，并提前阻断不安全行为。
- **整体含义**：提出**前瞻性守卫**概念，将安全判断从“当前状态评估”升级为“结合未来预测的决策”，是代理安全从内容审核向操作预防转变的关键一步。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
Janus 框架通过**多智能体模拟**生成多样化的长期代理轨迹，并训练一个共享策略 $\pi_\theta$ 同时完成两个耦合任务：
- **预测任务**：从部分轨迹 $\tau_{\text{pre}}$ 预测安全相关的未来总结 $s$。
- **裁决任务**：根据用户指令、观察到的前缀和预测的未来总结，判定当前状态的安全性（safe / unsafe / potential_unsafe）。

### 关键技术细节

#### (1) 数据构建
- **风险分类**：三种风险来源：用户源头（恶意指令）、环境源头（注入攻击）、代理源头（自身规划失败）。
- **模拟管道**：多智能体系统（Manager、Instructor、ToolDesigner、Grader、EnvInjector、Executor、Simulator、Reviewer）生成轨迹。不使用真实工具，而是模拟器返回结构化观察。
- **质量控制**：Reviewer 进行拒绝采样，保留满足任务一致性、风险覆盖和完整性的样本。
- **步骤级标注**：使用 GPT-5.5 对每个关键决策点标注安全标签和未来摘要。共生成 75,180 个训练样本（34,100 Safe, 18,665 Unsafe, 22,415 Potential Unsafe）。

#### (2) Coupled Anticipation and Adjudication Reinforcement Learning (CoAA-RL)
- **预测任务**：从 $(x, \tau_{\text{pre}}, m=A)$ 采样 $G_A$ 个未来摘要 $S_k$，每个摘要应与真实未来摘要 $s$ 语义一致（使用双向 NLI 相似度 Sim）。
- **裁决任务**：对每个摘要 $S_k$ 采样 $G_D$ 个裁决输出 $J_{k,j}$，解析为标签 $\hat{y}_{k,j}$ 和理由。奖励 $r_{k,j}^D = \lambda_D \ell_{k,j} + (1-\lambda_D)\phi_{k,j}$（标签正确性+格式正确性）。
- **耦合奖励**：摘要 $S_k$ 的效用 $u_k = \frac{1}{G_D}\sum_j r_{k,j}^D$；预测奖励 $r_k^A = (1-\lambda_A)\operatorname{Sim}(S_k, s) + \lambda_A u_k$。既奖励与真实未来一致，也奖励对下游裁决有用。
- **组相对优化**：基于 GRPO，分别计算预测和裁决的组内优势，联合优化目标 $\theta^\star = \arg\min_\theta [\mathcal{L}^A_{\text{GRPO}}(\theta) + \eta \mathcal{L}^D_{\text{GRPO}}(\theta)]$。

#### (3) 推理
- **两阶段安全预测**：对每个当前步骤，先预测未来摘要 $\hat{s}$，再基于它生成裁决 $\hat{J}$，输出标签和理由。若标签为 “unsafe” 则阻断执行。

- **Vanguard 显著提升保护率**：在四个基准上平均 ASR 为 0.071，基线守卫平均为 0.230，提升 15.9 个百分点。

- **预测任务的关键作用**：消融实验显示，移除预测任务导致 ASR 上升 100% 以上，Utility 下降；使用 oracle 未来摘要可进一步降低 ASR，表明准确预测对安全裁决至关重要。
- **耦合奖励优于单一奖励**：仅相似度或仅效用奖励均显著降低性能，解耦训练在高 ASR 基准上失败。
- **早期预测能力**：Vanguard 在仅 25% 前缀时已取得最低 ASR，随着前缀增长优势更明显，表明其能提前捕捉风险。

## 7. 优点

1. **前瞻性安全范式**：首次明确提出“预测性守卫”概念，将风险识别从事后/实时前移至预见阶段。
2. **创新的 CoAA-RL 框架**：耦合预测与裁决任务，通过奖励设计使预测直接服务于下游安全判断，实现端到端优化。
3. **风险分类系统全面**：覆盖用户、环境、代理三类风险来源，细粒度子类（如规划失败、指令遵循失败等），使训练数据多样化。
4. **模拟数据管道高效安全**：多智能体模拟生成高风险轨迹，无需真实工具执行，避免实际危害。
5. **强实验验证**：在四个长/短期基准上一致提升，消融实验系统，超参数分析完整，前缀预算实验展示了实际部署潜力。
