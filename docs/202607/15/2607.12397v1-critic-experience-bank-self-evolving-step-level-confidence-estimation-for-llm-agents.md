## 摘要
LLM智能体在外部环境中行动，每个动作都会改变后续决策所依赖的状态，并且单个错误步骤可能浪费交互预算或在最终失败被观察到之前很久就引发不可逆的副作用。因此，可靠部署需要步骤级置信度估计：在执行动作之前，对每个提议动作是否富有成效的校准概率。现有的LLM置信度估计器旨在根据给定提示对响应进行评分，但智能体的置信度还取决于执行后果：在环境响应后，类似情境中的类似动作是否实际推进了任务。我们引入了方法（方法简称），一个自我进化的批评者框架，其中LLM批评者从其自身过去的判断及其观察到的后果中积累证据。在每个轨迹之后，一个看到完整执行反馈的后见LLM投票决定每个步骤是否富有成效。由此产生的伪标签填充了一个记忆库，当类似步骤再次出现时，从中检索相关的富有成效和不富有成效的经验并将其注入批评者的提示中。方法简称无需训练，也不使用任何真实步骤标签。在三个智能体基准测试和三个批评者骨干网络上，方法简称在每一个数据集-批评者组合中都达到了最佳校准（ECE和Brier）和排序（AUC），相对于最强的无训练基线，ECE降低了高达54%。

## Abstract
LLM agents act in external environments where each action changes the state that later decisions condition on, and where a single wrong step can waste interaction budget or trigger irreversible side effects long before the final failure is observed. Reliable deployment therefore requires \emph{step-level confidence estimation}: a calibrated probability that each proposed action is productive, available \emph{before} the action is executed. Existing LLM confidence estimators are designed to score a response from the given prompt, but agent confidence also depends on execution consequences: whether similar actions in similar situations actually advanced the task after the environment responded. We introduce the \method (\methodshort), a self-evolving critic framework in which an LLM critic accumulates evidence from its own past judgments and their observed consequences. After each trajectory, a hindsight LLM that sees the full execution feedback votes on whether each step was productive. The resulting pseudo-labels populate a memory bank from which related productive and unproductive experiences are retrieved into the critic's prompt whenever a similar step recurs. \methodshort requires no training and uses no ground truth step labels. Across three agent benchmarks and three critic backbones, \methodshort attains the best calibration (ECE and Brier) and ranking (AUC) in every dataset--critic combination, reducing ECE by up to $54\%$ relative to the strongest training-free baseline.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：LLM 智能体在外部环境（如网页、移动 GUI、Shell）中执行多步骤任务时，每步的决策会影响后续状态，单个错误操作可能造成交互预算浪费或不可逆的副作用（如误提交、删除文件）。传统的最终任务成功率无法提供执行前的可靠性信号，因此需要**步骤级置信度估计**：在执行动作之前，评估该动作是否具有“生产力”（productive）的概率。
- **现有方法局限**：
  - 传统 LLM 置信度估计（如 Token 不确定性、Verbalized Confidence、CoT 等）针对的是静态提示下的响应，无法利用执行后果。
  - 训练型过程奖励模型（PRM）需要带标签的过程数据和可训练模型，不适合部署时使用固定 API 模型且无步骤标签的场景。
- **本文目标**：在推理阶段（inference stage），无需训练或真实步骤标签，仅利用智能体自身的执行反馈，实现自演化的步骤级置信度估计。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- **Critic Experience Bank (CEB)**：一个自我进化的批评者框架，核心是利用智能体过去的执行经验来校准当前步骤的置信度。
- 在每个完整轨迹结束后，一个**后见 LLM**（hindsight LLM）根据完整的执行反馈投票判断每一步是否“productive”，生成伪标签存入记忆库。当未来遇到相似步骤时，检索相关经验（对比正例与负例）注入批评者的提示，从而调整置信度。

### 关键技术细节
- **轨迹表示**：`τ = (s1, a1, o1), ..., (sT, aT, oT)`，其中 s = state, a = action, o = observation。
- **估计目标**：`ẑ_t = f(q, s_t, h_{<t}, a_t)`，其中 q 是任务，h 是历史。
- **后见标签生成**：
  - 使用后见 LLM g_φ 进行 V 次采样（温度 > 0），通过多数投票得到伪标签 ŷ_t（0/1）。
  - 投票阈值：`ŷ_t = 1[?t ≥ 1/2]`，其中 `?t` 是平均投票比例。
- **经验记录格式**：`e = (q', s', a', õ', ŷ', ẑ', χ')`，其中 χ' 是批评者原始判断与事后结果的一致性标志（`χ' = 1 当 [ẑ' ≥ 1/2] = ŷ'`）。
- **检索与提示**：
  - 复合查询键：`κ_t = task: q ∥ state: s_t ∥ action: a_t`。
  - 使用 TF-IDF 嵌入计算余弦相似度，从银行中分别检索 top-k 正例和负例，构成对比提示。
  - 附加**重复标志** `ρ_t`，如果当前动作与最近三步中的动作前缀匹配，则标记为可能循环。
- **置信度输出**：批评者返回 JSON 中的 `score` 字段，经 clip 到 [0,1] 得到 ẑ_t。

### 算法流程（文字描述）
1. 初始化经验银行 B 为空。
2. 对于每个任务 q 在任务流中：
   - 初始化空轨迹列表和分数列表。
   - 对于智能体提议的每一步 a_t：
     - 提取当前状态摘要 s_t。
     - 构建复合查询键 κ_t。
     - 从 B 中检索 top-k 正例 R₊ 和负例 R₋（按类）。
     - 调用批评者 f_θ（包含 q, s_t, a_t, h_{<t}, R₊, R₋, ρ_t）得到置信度 ẑ_t。
     - 保存分数，执行 a_t 并观察 o_t，加入轨迹。
   - 后见 LLM g_φ 对完整轨迹进行 V 次投票，得到每步伪标签 ŷ_t。
   - 将每步的 (q, s_t, a_t, o_t, ŷ_t, ẑ_t) 构建成记录并追加到银行 B 中。

## 3. 实验设计

- **步骤标签**：
  - Mind2Web、AMEX：基于参考动作匹配（严格匹配）。

## 6. 主要结论与发现

- **主要结论**：CEB 在无需训练、无需真实步骤标签的条件下，在所有数据集和骨干网络组合上均取得最佳校准（ECE、Brier）与排序（AUC）指标，ECE 相对最强无训练基线降低最高达 54%。
- **自我演化特性**：随着银行中经验增加，ECE 持续下降（图 3），而静态批评者保持不变，证实了经验积累的有效性。
- **组件重要性**：正负例对比检索是最关键的消融项（+0.065 ECE），其次是 TF-IDF 检索（+0.042），投票去噪和重复信号提供较小增益（+0.028、+0.017）。
- **伪标签噪声不大**：使用 GT 标签的银行仅比 CEB 伪标签银行多降低 0.007（Mind2Web）和 0.029（AMEX）ECE，说明多数投票已提取大部分校准信号。
- **下游收益**：在选择性执行和重生成场景中，CEB 能更准确地识别错误步骤，提升最终任务成功率（表 5、12）。

## 7. 优点

- **训练无关**：完全依靠推理时 LLM 调用和检索，无需额外训练或调整模型权重，兼容封闭 API。
- **自演化**：银行随任务流自动增长，批评者效果随时间提升，无需人工干预。
- **对比式检索**：同时检索正负例，让批评者看到成功与失败模式，有效纠正高估（overconfidence）。
- **检索设计合理**：复合键（任务+状态+动作）使用 TF-IDF，在结构相似任务中效果优于稠密嵌入（附录 F），且无需额外 API 依赖。
- **实验充分公正**：对比了多种无训练基线，在多个数据集和模型上验证，并做了宽松标签评估和上界分析，结论可靠。

## 8. 不足与局限

- **批评者与后见标签器同模型**：实验中使用同一 LLM 作为批评者和后见标签器，未测试独立标签器的影响（可能引入自我确认偏差）。
- **步骤标签依赖**：虽然无需人工标签，但后见标签仍依赖 LLM 的判断，可能存在噪声；在 InterCode-Bash 中基于奖励的标签也可能有噪音。
- **扩展性问题**：银行无去除或压缩机制，当任务流极大时，存储和检索成本可能成为瓶颈。
- **线性任务流假设**：所有任务按序处理，未考虑并行或动态重排，实际部署可能更复杂。
- **仅评估无闭环干预**：选择性执行和重生成实验是离线模拟，未在真实环境中实现闭环的 deferred执行或人类审查。
- **超参数固定**：检索深度 k 和后见投票次数 V 在实验中固定为 2 和 5，未在所有场景下验证最优性（附录 C 仅在一个子集上扫描）。
