## 摘要
基于大语言模型（LLM）的多智能体系统通过协同推理与行动实现复杂问题的求解，但其分布式结构也为系统级故障的诊断带来了新挑战。当执行失败时，由于长程交互与智能体行为的紧密耦合，很难确定哪个智能体应承担责任，以及轨迹在哪个节点首次出现不可逆的误判。本文研究基于LLM的多智能体系统中的故障定位问题，提出AgentLocate框架，将故障归因于特定智能体及其最早的关键步骤。AgentLocate结合基于LLM的评判机制与独立评估者的多视角验证，通过置信度感知策略聚合评估结果。所得反馈进一步用于对评判器进行轻量级微调，提升归因质量。我们在两个互补的基准上评估AgentLocate，覆盖多样化任务、智能体配置与轨迹长度。实验结果表明，AgentLocate在识别责任智能体与故障步骤两方面均持续优于现有故障定位方法，同时在令牌消耗与运行时间方面保持高效。

## Abstract
Large language model (LLM) based multi-agent systems enable complex problem solving through coordinated reasoning and action, but their distributed structure also introduces new challenges in diagnosing system-level failures. When an execution fails, identifying which agent is responsible and at what point the trajectory first becomes irreversibly misdirected is difficult due to long-horizon interactions and tightly coupled agent behaviors. In this paper, we study the problem of failure localization in LLM-based multi-agent systems and present AgentLocate, a framework that attributes failures to both a specific agent and the earliest decisive step. AgentLocate combines an LLM-based judging mechanism with multi-perspective verification by independent evaluators, whose assessments are aggregated using a confidence-aware strategy. The resulting feedback is further used to adapt the judge through lightweight fine-tuning, improving attribution quality. We evaluate AgentLocate on two complementary benchmarks covering diverse tasks, agent configurations, and trajectory lengths. Experimental results show that AgentLocate consistently outperforms existing failure localization methods in identifying both responsible agents and failure steps, while remaining efficient in terms of token usage and running time.

## 2. 论文提出的方法论：核心思想与关键技术细节

### 核心思想
将故障定位视为一个**可验证、可改进的过程**，而非一次性决策。通过「假设生成 → 多视角验证 → 自适应改进」的迭代循环提升归因质量。

### 技术细节

1. **Decisive Failure Hypothesis Generation（关键故障假设生成）**
   - 使用 **LLM 作为 Judge**，根据完整轨迹 `τ` 和用户查询 `q` 推理出责任智能体 `ˆi` 和最早关键步骤 `ˆt`。
   - 支持两种模式：**一次性（all-at-once）** 一次性分析全轨迹；**逐步（step-by-step）** 逐步读取前缀，一旦发现满足反事实逆转条件的步骤即输出。

2. **Evaluator Verification（评估者验证）**
   - 实例化 **H 个独立 Evaluator**，每个使用不同的提示风格（基础、简洁、证据导向）重新分析轨迹，并输出自己的预测 `(ih, th)`、理由 `rh` 和置信度 `ch`。
   - 通过**置信度加权投票**聚合评估结果：
     ```
     A(i,t) = Σ_{h∈H} ch · 1{(ih,th)=(i,t)}
     (ieval,teval) = argmax_{i,t} A(i,t)
     ```
   - 该聚合结果作为最终验证估计，降低单一模型或提示的偏差。

3. **Adaptive Judge Improvement（自适应 Judge 改进）**
   - 将 Judge 的初始预测、各 Evaluator 的输出以及聚合结果打包为训练实例 `zq`。
   - 利用 **LoRA 参数高效微调**对 Judge 进行更新，使其更好地对齐因果定义的关键故障标准。
   - 微调后的 Judge `Judge_ft` 用于后续推理，形成反馈闭环（可重复多轮）。

## 8. 不足与局限

- **实验覆盖的局限**：
  - 仅使用两个公开基准，且规模有限（Who&When 共 184 个失败案例），可能无法代表真实世界中更广泛、更复杂的多智能体系统。
  - 未在工业级系统（如 AutoGen、MetaGPT 等大规模实例）上进行测试。
- **偏差风险**：
  - 论文默认使用同一 LLM 作为 Judge 和 Evaluator，未充分探索不同能力级模型组合的影响（虽有表9但仅改变Evaluator）。
  - 对步骤级定位的严格评估可能未考虑“近似关键步骤”的实用性，虽然提供了容差分析，但现实中宽容差是否合理需进一步论证。
- **应用限制**：
  - AgentLocate 依赖于轨迹日志的完整记录，若系统不提供完整中间状态或动作细节，则无法应用。
  - 假设存在“理想动作”的反事实标准，但在真实非齐次环境中难以定义；原文通过 Judge 近似，但近似质量可能随任务复杂而下降。
- **可扩展性**：未讨论在超大规模智能体系统（如数百个智能体）或超长轨迹（数千步）下的性能退化趋势。
