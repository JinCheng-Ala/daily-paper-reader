## 摘要
生产级大语言模型智能体（如Claude Code和Codex）操作于不受信任的内容、文件、命令和工作区状态，使得安全失效可直接被利用。因此，红队测试必须跟上不断演进的模型和工具。现有方法主要优化攻击成功率并保留工件（如基准测试、载荷或攻击程序），这些工件记录了攻击成功的位置，但未揭示不安全智能体行为背后的启用条件。我们研究了对生产级大语言模型智能体的自动化红队测试，利用一个智能体研究环境发现关于另一个智能体的可复用漏洞知识。我们提出AHA，一个可证伪的发现循环：提出漏洞假设，构造证伪器，实例化有效攻击，在沙箱环境中执行，反思轨迹，并将确认的发现提升为漏洞概念图（VCG）。每个概念通过声明、启用条件、证伪器、迁移预测和支撑证据，将攻击者面对的表面与不安全轨迹关联起来。在涵盖直接攻击和间接攻击的三个场景中，针对Claude Code和Codex，发现的概念揭示了跨模型和智能体的可复用漏洞核心。冻结的VCG无需进一步搜索，在相同的单次协议下比最强的冻结发现基线高出14.2个百分点，同时可跨场景和攻击通道迁移。生成的VCG为生产安全团队提供可审计的工件，用于检查漏洞、验证补丁并积累可复用的安全知识。我们的代码见https://github.com/henrymao2004/Auto-research-red-teaming-in-sleep。

## Abstract
Production LLM agents such as Claude Code and Codex operate over untrusted content, files, commands, and workspace state, making safety failures directly actionable. Red-teaming must therefore keep pace with evolving models and tools. Existing approaches mainly optimize attack success and preserve artifacts such as benchmarks, payloads, or attack programs, which record where attacks succeed but not the enabling conditions behind unsafe agent behavior. We study automated red-teaming for production LLM agents using one agentic research environment to discover reusable vulnerability knowledge about another. We present AHA, a falsifiable discovery loop that proposes a vulnerability hypothesis, constructs a falsifier, instantiates a valid attack, executes it in a sandboxed harness, reflects on the trajectory, and promotes confirmed findings into a Vulnerability Concept Graph (VCG). Each concept links an attacker-facing surface to an unsafe trajectory through a claim, enabling condition, falsifier, transfer prediction, and supporting evidence. Across Claude Code and Codex on three scenarios covering direct and indirect attacks, the discovered concepts reveal a reusable vulnerability core across models and agents. A frozen VCG requires no further search and outperforms the strongest frozen discovery baseline by 14.2 percentage points under the same single-shot protocol, while transferring across scenarios and attack channels. The resulting VCG provides an auditable artifact for production safety teams to inspect vulnerabilities, validate patches, and accumulate reusable safety knowledge. Our code is available at https://github.com/henrymao2004/Auto-research-red-teaming-in-sleep.

## 1. 论文的核心问题与整体含义

- **研究动机**：生产级大语言模型智能体（如 Claude Code、Codex）直接操作文件、命令、工作区等真实环境，一旦安全失效即为真实动作（写入文件、窃取数据等）。现有红队测试方法主要优化攻击成功率并保留载荷、策略等工件，但这些工件只记录了攻击“落地”的位置，而未记录导致轨迹不安全的“使能条件”（enabling condition），因此难以审计、打补丁或在环境变化后复用。
- **核心问题**：如何将生产智能体红队测试从一次性的攻击探索转变为**可积累、可审计、可复用的漏洞知识发现**，使得发现结果在模型升级、工具更新后仍然有效。
- **整体含义**：论文将红队测试重构为**自动化研究（autoresearch）** 过程，输出一个**漏洞概念图（Vulnerability Concept Graph, VCG）**，每个概念封装了攻击面→不安全轨迹的机制及其证据，支持后续审计、补丁验证和跨场景迁移。

## 2. 方法论

- **核心思想**：提出 AHA（Agent Hacks Agent）框架，通过一个研究智能体对另一个生产智能体进行可证伪的发现循环。每个迭代包含：提出漏洞假设并提交证伪器→实例化为场景有效攻击→在沙箱中执行→反思轨迹并分类假设状态→定期审计防止奖励黑客。只有满足证据规则的概念才被提升到 VCG。
- **关键技术细节**：
  - **四个角色隔离的子智能体**：
    - **Hypothesizer**：在攻击编写前提交漏洞声明及显式证伪器（什么观察结果会反驳该假设）。
    - **Attack Designer**：将假设实例化为符合场景合约的有效载荷，不修改假设。
    - **Reflector**：对比轨迹与证伪器，输出假设状态（confirmed/falsified/inconclusive），并可选输出新概念元组。
    - **Critic**：每 10 次迭代，从全新上下文审计奖励黑客、模板重复、过专业化、覆盖空白。
  - **搜索模式**：explore（新机制）、exploit（深化已有概念）、transfer（测试跨类别迁移）、consolidate（重测低置信度概念），由 VCG 计数器决定。
  - **VCG 结构**：每个概念包含五个自然语言字段（漏洞声明、使能条件、攻击模板、失败结果、迁移预测）及三个计数器（n_obs, n_conf, n_fals）。**提升规则**：n_conf ≥ 3 且置信度 ≥ 0.6 且至少一次有效 break（judge 确认且假设未被证伪）。
  - **执行隔离**：每个攻击在独立 Docker 容器中执行（2 CPU, 4 GB 内存，600 秒超时），裁判在容器外运行。
- **无显式公式**；规则基于计数器和阈值。

## 6. 主要结论与发现

- **RQ1（机制共享性）**：发现的概念形成跨模型/跨智能体的共享机制核。Claimed authorization 系列在 18 个设置中出现 16 次；Task‑goal hijack 在间接攻击场景中跨模型共享。
- **RQ2（可重用性）**：冻结 VCG 在单发协议下平均 ASR 比最强冻结基线高 14.2 百分点（全局）和 13.5 百分点（Claude Code 子集）；在所有场景的 Claude Code 智能体上均领先。
- **RQ3（消融）**：移除任一个安全部件（证伪器、跨实例记忆、定期批评）后发现 ASR 几乎不变，但有效概念数降低，说明这些部件是**使发现结果可重复的关键**，而非仅提高发现数量。
- **RQ4（迁移性）**：概念可跨场景（直接↔间接通道）和跨受害者模型迁移，且迁移后的 ASR 接近甚至超过目标的原生 ASR，验证了机制级知识而非载荷模式。
- **整体结论**：AHA 将红队测试转化为可积累、可审计的漏洞知识生产，VCG 比传统载荷更具持久性和可操作性，有助于生产安全团队的策略补丁和持续监控。

## 7. 优点

- **方法学创新**：从“优化攻击载荷”转向“发现漏洞机制”，输出结构化、可审计的 VCG；引入证伪器防止事后解释，提升结果的可靠性。
- **实验设计亮点**：
  - 冻结‑单发评估严格分离发现与部署，直接测量工件的重用性。
  - 消融实验设计精良，揭示了使发现转化为可重复结果的必要组件。
  - 迁移实验覆盖跨场景和跨模型，提供机制级泛化的强证据。
- **可扩展性**：支持“构建场景”和“导入场景”工作流，可将新的安全关注点或现有基准快速接入同一循环（附录 D‑F）。
- **实际应用价值**：VCG 可供安全团队直接检查使能条件并执行补丁，无需求助于重新搜索；每个概念包含迁移预测，支持在不同产品间复用。

## 8. 不足与局限

- **实验覆盖范围有限**：仅测试了 Claude Code 和 Codex 两个智能体，且受害者模型限于三款（Minimax、Kimi、Deepseek）；未覆盖浏览器代理、移动代理或多模态智能体，也未测试其他大模型提供商（如 Gemini、Llama）。

- **双重用途风险**：AHA 能自动发现漏洞，可被恶意方滥用；论文指出机制级工件更偏向防御性，但未提供具体防护措施。
