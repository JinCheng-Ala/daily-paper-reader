## 摘要
大型语言模型代理通过生成交织规划、工具调用和中间结果的轨迹来完成任务。当前的评估指标要么将这种轨迹简化为一个二元的成功标志，要么通过与参考轨迹进行精确匹配来比较。成功标志无法区分一个合理的解决方案和一个侥幸成功的方案，也无法说明失败的运行为何出错。精确匹配会惩罚那些有效但顺序或分解方式与参考不同的规划。我们将轨迹评估重新定义为代理执行图与一组有效解决方案图之间的距离，并通过在属性依赖图上求解非平衡融合Gromov-Wasserstein传输问题来实现。由此得到的分数，称为\otap{}（面向代理规划的最优传输），是一个伪度量，可证明对保留依赖关系的重排序具有不变性，对冗余步骤具有有界敏感性。其非平衡边界允许缺失或幻觉步骤而无需强制匹配，其软耦合适应规划粒度的变化。在控制扰动和三个公开基准上，\otap{}能够区分有效和无效的轨迹，而单纯的语义指标得分低于随机水平。当依赖图被精确恢复时，其准确性最高；仅当依赖图是从自由文本轨迹中启发式推断时，准确性才会下降。

## Abstract
Large language model agents solve tasks by generating trajectories that interleave planning, tool calls, and intermediate results. Current evaluation metrics reduce such a trajectory to a binary success flag or compare it against a reference by exact matching. A success flag cannot distinguish a sound solution from one that succeeds by luck, and says nothing about why a failed run went wrong. Exact matching penalizes plans that are valid but reordered or decomposed differently from the reference. We reframe trajectory evaluation as a distance between the agent's execution graph and a set of valid solution graphs, and instantiate it via an unbalanced fused Gromov-Wasserstein transport problem over attributed dependency graphs. The resulting score, termed \otap{} (Optimal Transport for Agentic Planning), is a pseudo-metric that is provably invariant to dependency-preserving reorderings and has bounded sensitivity to redundant steps. Its unbalanced marginals handle missing or hallucinated steps without forcing a match, and its soft coupling accommodates variation in plan granularity. On controlled perturbations and three public benchmarks, \otap{} separates valid from invalid trajectories in a regime where semantics-only metrics score below chance. Its accuracy is highest when the dependency graph is recovered exactly, and drops only when the graph is inferred heuristically from free-text traces.

    - 轨迹图定义：每个节点包含（动作文本、工具ID、参数、执行效果）；有向边代表数据/因果依赖。

    - UFGW目标函数（公式3）：
        \[
        \min_{T\ge0} (1-\theta)\sum C_{ij}T_{ij} + \theta\sum |D^P_{ik}-D^R_{jl}|^2 T_{ij}T_{kl} + \lambda_1 KL(T\mathbf{1}\|\mu) + \lambda_2 KL(T^\top\mathbf{1}\|\nu) - \varepsilon H(T)

## 6. 论文的主要结论与发现

1. **Otap在区分有效/无效轨迹方面显著优于基线**：在人工任务上AUROC最高（0.820），基线最高仅0.674；在PlanBench（精确图）和WebArena（显式图）上同样领先，AUROC分别为0.804和0.859。
2. **依赖图质量至关重要**：当图精确（PlanBench和WebArena）时，Otap几乎完美识别有效重排序（P2=100%）；当图启发式推断（GAIA）时，AUROC降至0.698但仍为最佳，但排名准确率和相关系数下降。
3. **纯语义指标在因果反转下失效**：因果反转不改变文本，导致嵌入基线AUROC低于随机（0.462-0.468），而Otap通过结构项保持性能。
4. **奥威尔研究负结果**：在预测形式化规划有效性时，Otap略逊于BLEU（79.2% vs 86.7%），因为Otap有意容忍偏差（非平衡边界、软匹配），而在严格有效性判断中这些偏差应被惩罚。结论：**当形式验证器存在时，应使用验证器**。
5. **消融确认关键组件**：去掉结构项（θ=0）导致AUROC从82.0降至54.5；去掉工具项导致P5（工具不兼容）性能骤降；多参考聚合增加16.9 AUROC点；签名/签名距离差异不大。

## 8. 不足与局限

- **依赖图提取的瓶颈**：在自由文本迹线（如GAIA）中启发式追踪边可能引入噪声，降低Otap的有效性。论文承认这是未来工作。
- **长轨迹的可扩展性**：Gromov-Wasserstein项复杂度为O(n²m+nm²)，论文中GAIA较长轨迹（>50步）运行时达23.9秒，需要稀疏化或早期停止缓解。
- **粒度变化的基本困难**：合并/拆分步骤的扰动（P3）仍会产生较好的评分，但论文指出“原则性的粒度不变性”仍是开放问题。
- **实验覆盖范围有限**：仅评估了Blocksworld、Web导航和GAIA（通用QA）三类任务，未涉及工具调用密集、多轮交互的典型代理场景（如τ-bench、ToolBench）。
- **奥威尔负结果的局限性**：Otap在存在形式验证器时不推荐使用，但现实中许多任务缺乏验证器，其适用范围受限于此。
- **主观性**：人工任务中的质量阶梯是作者构建的，虽然基于确定性扰动，但缺乏人类判定作为参照。论文注明了这一点。
- **未探索深度反馈**：Otap尽管可微，但论文未演示其在训练循环中作为过程奖励的实际使用，仅提出潜力。
