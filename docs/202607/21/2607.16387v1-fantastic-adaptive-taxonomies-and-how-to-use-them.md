## 摘要
智能体系统的执行轨迹记录了其失败的方式，而改进此类系统而不改变模型权重的过程（轨迹选择、提示词与工作流优化、运行时监控）会读取这些轨迹以获取反馈。然而，原始轨迹并不适合积累这种反馈：它们冗长、针对特定实例，并且缺乏针对重复失败现象的稳定词汇。我们认为，智能体系统应该维护一个关于其失败方式的显式表示，该表示从其自身行为中归纳得出，并在需要失败反馈的任何地方可重用。AdaMAST通过将目标系统的轨迹转换为紧凑且基于证据的失败分类法来构建这种表示：命名失败代码沿着三个固定轴（系统级、角色特定和领域特定）组织，每个名称、定义和证据模式都是从轨迹中归纳得出的；没有手动编写的代码，也没有人工标注的轨迹。该分类法不仅仅是事后诊断工具，更是一个共享的反馈接口，以三种方式改进智能体。在智能体系统搜索中，基于分类法编码的失败候选诊断在我们测试的所有五个基准上都优于自由形式反思。在运行时，分类法反馈将SWE-agent在SWE-bench Verified Mini上的解决率从使用自由文本反思的60%提升到70%，并将Claude Code作为运行时技能从64.0%提升到70.7%。在轨迹选择中，AdaMAST-Judge（一个基于归纳代码构建的验证器）将Terminal-Bench 2.0上的最佳5选1准确率比Pass@1提高了8-15个百分点。该词汇本身紧凑（数量级压缩同时保留轨迹区分度）、符合人类判断（比手工参考词汇更接近专家失败标注），并且具有自适应性（为不同领域归纳的分类法共享的代码极少）。自适应失败分类法闭环了智能体产生的轨迹与改进它们的流程之间的联系。

## Abstract
An agent system's execution traces record how it fails, and procedures that improve such a system without changing model weights (trajectory selection, prompt and workflow optimization, runtime monitoring) read these traces for feedback. Yet raw traces are a poor medium for accumulating that feedback: long, instance-specific, and lacking a stable vocabulary for recurring failures. We argue that an agent system should instead maintain an explicit representation of how it fails, induced from its own behavior and reusable wherever failure feedback is needed. AdaMAST builds this representation by converting a target system's traces into a compact, evidence-grounded failure taxonomy: named failure codes organized along three fixed axes (system-level, role-specific, and domain-specific), with every name, definition, and evidence pattern induced from the traces; no code is hand-authored, no trace human-annotated. The taxonomy is not merely a post-hoc diagnostic but a shared feedback interface, improving agents in three ways. In agent-system search, taxonomy-coded diagnoses of failed candidates outperform free-form reflection on all five benchmarks we test. At runtime, taxonomy feedback raises SWE-agent's resolution on SWE-bench Verified Mini from 60% with free-text reflection to 70%, and improves Claude Code from 64.0% to 70.7% as a runtime skill. In trajectory selection, AdaMAST-Judge, a verifier built on the induced codes, improves best-of-5 accuracy on Terminal-Bench 2.0 by 8-15 points over Pass@1. The vocabulary itself is compact (an order-of-magnitude compression that preserves trace distinctions), human-faithful (matching expert failure annotations more closely than a hand-crafted reference vocabulary), and adaptive (taxonomies induced for different domains share few codes). Adaptive failure taxonomies close the loop between the traces agents produce and the procedures that improve them.

## 1. 核心问题与整体含义（研究动机和背景）
**问题**：大语言模型（LLM）代理系统的执行轨迹记录了其失败过程，用于改进系统的各种过程（如轨迹选择、提示词优化、运行时监控）都需要从轨迹中提取反馈。然而，原始执行轨迹存在严重缺陷：
- **冗长且实例特定**：每个轨迹很长，且只描述特定运行实例。
- **缺乏稳定词汇**：同一类失败在不同轨迹中被重复诊断，以自由文本形式每次重新推导，无法积累和复用。

**核心主张**：代理系统不应依赖原始轨迹作为反馈界面，而应维护一个显式的、紧凑的、基于证据的**失败分类法**（taxonomy），该分类法从系统自身的轨迹中归纳得出，可被所有改进过程共享使用。

**整体含义**：该工作提出了**AdaMAST**（Adaptive Multi-Agent System Failure Taxonomies），自动将目标系统的执行轨迹转换为沿三个固定轴（系统级、角色特指、领域特指）组织的命名失败代码集合，无需人工编写代码、无需人工标注轨迹。该分类法作为共享反馈接口，在搜索、运行时监控和轨迹选择三个应用场景中均显著提升了性能。

## 2. 论文提出的方法论

### 核心思想
将代理系统本身的失败模式显式化：从系统产生的执行轨迹中**归纳**（induce，而非推理）出紧凑的失败代码集。每个代码具有名称、定义、证据模式。代码按三个固定轴组织：
- **系统级（Axis A）**：修复框架或编排层面的问题。
- **角色特指（Axis B）**：修复发现的角色（如求解器、检查器、协调器）问题。
- **领域特指（Axis C）**：注入任务知识。

### 关键技术细节：生成管道
管道包含四个阶段，共8次LLM调用（一次部署后摊销）：
1. **分析（Analysis）**：提取领域上下文、发现的角色、重复行为信号。
2. **策展（Curation）**：针对每个轴分别提出候选代码。
3. **整合（Consolidation）**：去重、修复命名、拒绝无支持代码，确保覆盖。
4. **交叉标注者一致性（Inter-Annotator Agreement）**：四个LLM标注者独立标注保留轨迹，使用五阶段协商协议；当平均成对Cohen's κ ≥ 0.75且覆盖率≥0.70时接受分类法。不满足时触发合并、添加、重标记编辑。

### 在线精炼
系统演化时，可从当前轨迹池触发精炼回合，使用与一致性门相同的编辑操作，使分类法与当前系统保持同步。

### 三个消费者
- **搜索**：在AdaEvolve搜索循环中，突变提示接收分类法编码的诊断（代码+引用证据）。
- **运行时监控**：分类法作为运行时技能嵌入代理上下文，在检查点触发自我检查。
- **轨迹选择**：AdaMAST-Judge将每个代码转化为验证标准，通过特征选择器组合，用于选择最佳轨迹。

- **充分**：实验覆盖了三个截然不同的下游应用场景，多个基准，多个种子，消融实验较多。
- **公平**：在相同搜索骨架、相同架构种子、相同迭代次数下比较；运行时臂共享相同求解器、实例集和评估协议；轨迹选择使用相同轨迹池。变更的仅是对比方法中的反馈源。
- **不足**：部分小基准种子数较少（如MMLU-Pro仅2种子），但主表的多种子复制提供了可靠性估计。搜索部分单次运行（Frontier-CS, OlympiadBench）未报告方差，但机制分析补充了证据。总体上实验设计是客观的。

## 6. 主要结论与发现

1. **搜索提升**：分类法引导的突变在所有5个基准上优于自由文本反思（+3.4到+7.5个百分点）。OlympiadBench上AdaMAST达到91.9%，比固定MAST分类法高2.4 pp，比自由文本反思高4.0 pp。
2. **运行时监控提升**：SWE-agent上AdaMAST从Base的50%提升到70%（+20 pp），超过Reflexion（60%）和MAST（68%）。Claude Code上从64.0%提升到70.7%（+6.7 pp），超过MAST（67.3%）。
3. **轨迹选择提升**：AdaMAST-Judge在Terminal-Bench上比Pass@1提升8.1-14.9 pp，比固定MAST分类法在非饱和框架上提升3.4-4.5 pp。
4. **分类法本身属性**：紧凑（～18倍压缩，89%轨迹保留唯一代码签名）、与人类专家标注一致（TRAIL上area-κ=0.682 vs 手工分类法0.516）、自适应（跨域Jaccard仅0.14）。

## 7. 优点

- **方法创新**：首次提出从系统自身轨迹自动归纳自适应失败分类法，作为持久反馈界面，而非一次性诊断。
- **无需人工标注**：管道完全由LLM驱动，仅一致性门控使用LLM标注者，大幅降低成本。
- **可复用性**：分类法一旦生成，可在多个下游流程中复用（搜索、运行时、选择），而自由文本反思每次丢弃。
- **一致性门控**：通过交叉标注者一致性确保分类法可靠，减少假阳性代码。
- **在线精炼**：支持分类法随系统演化而更新，保持当前性。
- **实验广泛**：覆盖三种类型应用、多个基准、多种模型、多种消融，验证了泛化能力。
- **与专家一致**：在TRAIL数据集上与人类专家标注的一致性优于手工分类法，说明归纳代码具有语义有效性。

## 8. 不足与局限

- **依赖LLM生成**：分类法由LLM生成，可能继承LLM的偏差（如幻觉、系统偏好）。论文使用跨模型面板部分缓解，但未完全避免。
- **人类验证有限**：一致性门控使用LLM标注者而非真人类专家（除TRAIL外）。TRAIL验证仅使用单数据集，规模较小（117条轨迹）。
- **算力开销未透明**：未报告GPU型号、训练时长等详细信息，使可复现性和比较受限。
- **样本效率不单调**：小规模搜索基准上增加归纳轨迹数并未单调提升性能，表明可能需特定抽样策略。
- **选择应用场景的混合结果**：在Terminal-Bench上，分类法文本替换为其他领域分类法时性能变化不大，说明该场景中特征选择器和启发式特征贡献较大，分类法文本的边际效应较小。
- **未在真实生产系统验证**：实验基于学术基准，未评估在长期运行、高负载、多轮交互的生产环境中的表现。
- **分类法轴设计固定**：三个轴是人工设计的，可能不适用于某些新型架构或非传统代理系统。
- **与更复杂方法的比较缺失**：论文未与基于训练或微调的方法（如生成式验证器）直接比较，仅与零样本方法对比。
