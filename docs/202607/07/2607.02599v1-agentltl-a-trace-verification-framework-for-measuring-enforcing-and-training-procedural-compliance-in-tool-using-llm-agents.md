## 摘要
使用工具的LLM代理通常通过最终答案正确性或LLM评判者来评估。两者都不捕获答案是如何产生的。在安全关键场景中，程序本身是正确性的一部分。在本文中，我们引入了AgentLTL，这是一种源自一阶线性时序逻辑（FO-LTL）的语言，用于表达代理轨迹上的程序性规则。它产生一个确定性的、无需评判者的合规性分数。在该框架中，单个规范驱动两种用途。第一种是约束：该分数对完成的轨迹进行评分，或通过在线检查每个前缀在执行前门控工具调用。第二种是微调：该分数作为密集奖励。在一个涵盖排序、分支、迭代和基础操作的基准测试上，块-警告约束提高了七个模型中五个模型的合规性。使用相同奖励进行微调，在保留模式（包括未见过的工具名称别名）上，准确性和合规性分别提高了38和17.5个百分点。这些发现与模型获取程序结构而非记忆表面工具名称和程序这一观点一致。

## Abstract
Tool-using LLM agents are usually evaluated by final-answer correctness or LLM judges. Neither captures how an answer was produced. In safety-critical settings, the procedure itself is part of correctness. In this paper, we introduce AgentLTL, a language derived from First-Order Linear Temporal Logic (FO-LTL) that expresses procedural rules over agent traces. It yields a deterministic, judge-free compliance score. In this framework, a single specification drives two usages. The first is harnessing: the constraints score completed traces, or gate tool calls by checking each prefix online, before execution. The second is finetuning: the score serves as a dense reward. On a benchmark spanning ordering, branching, iteration, and grounding, block-and-warn harnessing improves compliance on five of seven models. Finetuning with the same reward yields +38 and +17.5 percentage point gains in accuracy and compliance on held-out patterns, including unseen tool-name aliases. These findings are consistent with the model acquiring procedural structure rather than memorizing surface tool names and procedures.

### 论文提出的方法论

- **核心思想**：基于一阶线性时序逻辑（FO-LTL）定义语言 **AgentLTL**，用于表达对智能体执行轨迹（trace）的程序性、语义和接地约束。同一个约束集可同时用于：① 离线评估完成的轨迹；② 在线强制（在工具调用执行前检查前缀并阻止违规模块）；③ 作为强化学习密集奖励。
- **关键技术细节**：
  - **轨迹模型**：`τ = <c0, ..., c_{n-1}>`，每个调用 `ci = (ni, ai, ri, i)`，包含工具名、参数、结果和位置。
  - **公式语言**：包含原子命题（如`CalledWith`、`InOrder`）、时序算子（G, F, X, U, W, R）和数据量词（∀/∃，范围从轨迹中提取的有限域）。变量赋值后，公式对轨迹返回布尔值。
  - **合规评分**：定义程序 `P = {(κ_i, w_i)}`，权重集。**合规分数** `C(τ, G_P) = 1 - Σ(violated weight) / Σ(total weight) ∈ [0,1]`，连续、确定、无裁判。
  - **分层约束系统（L1-L6）**：将约束组织成六层，分别捕获不同失败模式，避免单一算术错误导致分数崩溃。L1: 工具分支；L2: 文字参数匹配；L3: 全局序列顺序；L4: 实例/成对顺序；L5: 调用计数；L6: 精确参数匹配。
  - **在线强制实现**：将提议调用追加到前缀后，检查可强制执行子集。违反者根据严重程度执行：`HARD_STOP`（停止）、`SOFT_BLOCK`（阻止+允许纠正，多次后升级）、`BLOCK_AND_WARN`（阻止+警告，允许覆盖）、`TOLERATE`（仅记录）。
  - **训练奖励实现**：评分连续，可直接作为GRPO奖励项。配合答案正确性（整形）和轨迹编辑距离奖励，提供密集信用分配。

### 论文主要结论与发现

- 合规性与正确性**显著分离**：许多正确答案来自不合规轨迹（如DeepSeek-V4-Pro），或高合规但低正确（如Qwen3-Next-80B）。最终答案正确性掩盖了程序错误。
- 强制设置作为诊断工具：Block-and-warn提升5/7模型合规性，改善弱模型早期停止；Soft-block对多数模型最差（强制终止阻止恢复）。
- 序列顺序（L3/L4）是最普遍的故障层，占加权损失55-77%。

### 优点（方法或实验设计亮点）

- **统一规范语言**：同一约束同时支持评估、强制和训练，避免语义漂移。
- **确定性合规得分**：无裁判偏差，完全可重复。
- **分层权重**：隔离结构性错误和计算性错误，提供稠密奖励信号。
- **在线强制确定性**：严格基于FO-LTL可强制执行片段，无裁判不可靠。
- **分层次地分析**：按层报告通过率，精确定位故障类型（全局序列 vs. 分支 vs. 参数）。
- **转移学习验证**：通过零覆盖模板和工具名别名验证泛化能力，排除记忆假说。
- **接地研究设计**：流行度分层、提示变体、LLM裁判对比，多角度验证接地检测幻觉的有效性和局限。

### 不足与局限

1. **真空接地**：接地谓词在拒绝回答（无实体）时真空通过，大部分严格提示的接地增益来自拒绝而非正确接地答案。
2. **流行度混淆**：接地研究的流行度分层无法区分预训练曝光与仓库大小/复杂度，结论有混杂。
3. **合成工具环境**：使用确定性整数工具，无法直接推广到随机输出、部分故障、模糊参数的真实环境。
4. **单模型微调**：仅微调Qwen3-4B（LoRA），未跨尺度或模型族验证，对大模型（参数记忆更强）效果未知。
5. **约束编写成本**：需要领域专家编写FO-LTL约束，未考虑人工因素、误规范传播错误等。
6. **表达能力限制**：六层约束未覆盖定时、资源、隐私、智能体间协调；FO-LTL无法表达超性质（如并行调用的非干扰）。
7. **强制副作用**：soft-block强制终止可能造成系统不一致（如已触发副作用的工具调用）；未评价回滚策略。
8. **单轮约束**：约束在轨迹开始前固定，无法响应工具输出动态调整（如空结果跳过分支）。
9. **嵌套深层控制流**：`nested_loops+branching`模板在微调后仍为0%正确率，表明训练未覆盖深度嵌套模式。
