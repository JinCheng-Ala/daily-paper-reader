## 摘要
当可靠工具在持续会话中悄然发生变化时，LLM代理的工具选择会发生什么？我们借用认知心理学中的集转移概念，研究代理对隐藏可靠性变化的适应能力。我们的基准测试构建了具有冗余的工具技能库，其中多个工具可以解决同一任务，但隐藏可靠性不同。在我们的评估框架中，分支时间表在隐藏边界处切换可靠工具组，并将每次切换与无切换控制配对。我们发现，代理在默认情况下会在每个边界后的几轮交互中建立起一个小的重复例行程序，每次可靠性切换后，调用份额集中在少数几个离散值上。我们为每个代理轨迹计算集转移准确率：在每次切换后的窗口内路由到目标工具组的联合概率。我们在开源代理框架中测试了开放权重的LLM，发现了相同的例行程序中存在性质不同的失败模式。我们还发现，工具集呈现选项的方式——是竞争还是互补——会改变路由动态。

## Abstract
What happens to an LLM agent's tool choice when the reliable tool silently changes within an ongoing session? We borrow set-shifting from cognitive psychology to study how well agents adapt to hidden reliability shifts. Our benchmark mounts tool-skill libraries with redundancies, where many tools solve the same task but differ in hidden reliability. In our evaluation framework, a branched schedule shifts the reliable tool group at hidden boundaries and pairs every shift with a no-shift control. We find that agents, by default, settle on a small recurring routine within a few turns of each boundary, with call shares concentrating on a few discrete values after each reliability shift. We score the set-shifting accuracy for each agent trajectory: the joint probability of routing to the target tool group in every post-shift window. We test open-weight LLMs in an open-source agentic harness and find qualitatively distinct failure modes across the same set of routines. We also find that set framing, how the toolset presents the alternatives as competing or complementary, shifts the routing dynamics.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：大型语言模型（LLM）代理在持续会话中，当可靠工具悄然发生变化时，能否适应并调整工具选择？具体而言，代理是否表现出类似人类的认知僵化（perseveration），即固守已学得的规则而忽视环境变化？

- **研究动机**：受认知心理学中威斯康星卡片分类测验（WCST）和Einstellung效应的启发，作者将经典的“集转移”（set-shifting）范式引入LLM代理领域。现代LLM代理通常处于丰富的代理框架（agentic harness）中，挂载了多个功能等效但名称和描述不同的工具。当某个工具组的后端可靠性在会话过程中被静默切换时，代理能否放弃旧工具、转向新可靠工具？这直接关系到代理在实际部署中的鲁棒性和适应性。

- **整体含义**：理解代理的工具选择动态对于设计更可靠的自主代理至关重要，尤其在工具库存在冗余且可靠性可能动态变化的环境中（如后端服务升级、故障转移等）。本文首次系统性地评估LLM代理在工具可靠性隐蔽变化下的行为，揭示了代理默认会陷入重复性例行程序，且不同模型表现出定性的不同失败模式。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想

- 构建**冗余工具-技能集**（redundant tool-skill sets）：一个任务由多个功能等效的工具组解决，但各组的隐藏可靠性在不同阶段不同。在会话过程中，可靠工具组在隐蔽边界处静默切换，每次切换都配对一个无切换控制组。
- 借鉴WCST的**分支调度树**（branched schedule tree）：轨迹从一个共享前缀开始，在每层内部节点分支为三个子路径——逆转（回到先前使用过的组）、新颖切换（切换到之前未使用过的组）和无切换控制。总共三层边界（b1, b2, b3），产生9个端点。
- 定义**集转移准确率**（set-shifting accuracy, Φ）：代理在每次切换后的窗口内路由到目标工具组的联合概率，计算为各窗口目标调用份额的乘积。

### 2.2 关键技术细节

- **工具组结构**：每个工具组包含多个功能相同的工具（相同schema和操作，仅名称和描述不同）。一个技能（skill）对应一个工具组，描述何时以及如何调用该组工具。
- **隐藏可靠性切换**：系统提示和工具描述保持不变，只有二进制反馈（成功/失败）指示当前可靠组。代理只能通过工具调用的返回结果推测哪一组可靠。
- **确定性验证器**：一个无模型的状态机，根据当前阶段设置BENCHMARK_CURRENT_BEST_TOOLS和BENCHMARK_DISALLOWED_TOOLS，将不在最佳组内的调用返回失败，并阻止不允许的副作用工具。
- **路径分类**：每轮调用根据涉及的工具组分为四类：R1（仅目标组）、R2（目标组+另一组）、R3（目标组+另外两组）、R*（无目标组）。这些对应目标份额s∈{1, 0.5, 0.33, 0}。
- **评估指标**：除了Φ，还报告路径类比率、任务完成率F（每轮至少调用一个可靠组工具且无禁止副作用）。

### 2.3 公式

Φ = ∏_{k=1}^{K} s_k，其中s_k是第k个后切换窗口中调用落入目标工具组的比例。

## 6. 论文的主要结论与发现

1. **代理默认为重复性例行程序**：在每次可靠性切换后，代理在数轮内迅速收敛到一个小范围的固定模式，调用份额集中在少量离散值（1, 0.5, 0.33, 0）。这种例程在循环网络中表现为“吸引子”。
2. **不同模型有定性不同的失败模式**：
   - **mimo-v2.5**：倾向于部分固着（partial perseveration），依赖最近使用的工具组，产生中等混合模式（如R2），对从未见过的目标（CC）适应最差。
   - **deepseek-v4-pro**：倾向于前缀级固着（prefix-level lock-in），一次选择会在整个轨迹中持续，呈现双峰分布（要么几乎全部正确，要么几乎全部错误），对目标先验角色不敏感。
3. **历史和深度影响**：mimo受近期历史影响大，深度和路径显著改变结果；deepseek受首次承诺决定，额外深度和路径影响小。
4. **策略提示可以改变固着目标**：“适配性”提示（Adaptive）使deepseek在所有端点达到高目标份额（s≥0.86），但“博学”提示（Polymath）增加工具多样性而不提高正确调用。
5. **集框架改变路由动态**：竞争框架（各组描述为可互相替代）比互补框架（各组描述为补充）更可能引导代理定位到当前可靠组（显著方向一致，p=3e-4）。互补框架下代理倾向于每轮调用所有组（混合例程）。

## 7. 优点：方法或实验设计上的亮点

- **创新性基准**：首次将WCST的集转移范式系统性地引入LLM代理工具选择评估，构建了带冗余工具组的纵向可恢复会话环境，并设计了分支树结构控制历史效应。
- **细粒度描述性框架**：定义了路径类（R1-R3-R*）和目标份额，能捕捉代理在切换后的微观行为模式（如混合调用、完全忽略新目标），而不仅仅看最终成功与否。
- **双通道干预**：同时测试了指令性（策略提示）和结构性（集框架）两种干预，揭示了不同层面影响代理行为的途径。
- **统计严谨**：使用自举法置信区间、威尔逊区间、配对置换检验，避免依赖单一点估计，并报告了高/低模式轨迹数，揭示双峰分布。
- **可复现性**：开放源代码和数据集（https://github.com/zwycl/wcst-tool-bench），使用开源模型和框架。

## 8. 不足与局限

- **模型覆盖有限**：仅测试了两个开放权重模型，未能包含GPT-4、Claude等前沿封闭模型，因此对通用性结论保持谨慎。
- **模拟环境**：三个领域均为模拟，真实世界的工具反馈可能更嘈杂、包含更多信息（如错误消息），可能改变代理行为。
- **轨迹长度不足**：最大90轮，仅K=2层分支，可能无法观察更长轨迹中的恢复动态。
- **框架对比统计力弱**：每个单元格仅8个重复对，单个单元格的p值均不显著（p>0.016），结论依赖跨单元格的方向性一致性，需要更大样本量确认。
- **未量化框架连续谱**：互补-竞争并非二分连续体，可能存在中间状态，论文未研究其中边界。
- **策略提示效果未全面测试**：适配性提示仅在deepseek上验证，未测试mimo或其他模型，且只用在调度域，未推广到其他领域。
- **未研究记忆压缩影响**：长轨迹中压缩上下文可能影响固着模式，论文未探讨。
- **中立提示约束**：用户提示故意不含词汇线索，真实场景中用户可能提供指向性语言，这可能改变结果。
