## 摘要
对于在多步交互环境中的LLM智能体，一个关键挑战是如何有效利用积累的交互经验。现有工作通常将这种经验的两种用途分开：将其作为自然语言规则保留在模型外部以供后续提示，或者使用轨迹和反馈来更新模型参数。前者易于解释，但可能随着策略演变而不同步；后者更广泛地改进策略，但在稀疏奖励设置中仅对局部错误提供有限的纠正。我们提出了LLM智能体的经验规则与策略联合学习（JERP），该方法从相同的交互轨迹中更新长期经验规则池和策略。在决策时，JERP检索任务相关规则，并结合交互历史将其作为智能体的条件。每轮结束后，它使用收集的轨迹既优化策略，又通过对比当前 rollout 与参考成功轨迹来修订规则池。这种耦合使规则池与演变的策略保持一致，同时允许稳定有效的行为逐渐被吸收到模型本身中。在AlfWorld和WebShop上的实验表明，JERP在复杂交互任务的决策性能上取得了一致的提升。

## Abstract
For LLM agents in multi-step interactive environments, a key challenge is to make effective use of accumulated interaction experience. Existing work has typically separated two uses of such experience: keeping it outside the model as natural-language rules for later prompting, or using trajectories and feedback to update the model parameters. The former is easy to interpret but can fall out of sync with the evolving policy; the latter improves the policy more broadly but provides only limited correction for local mistakes in sparse-reward settings. We present Joint Learning of Experiential Rules and Policies for LLM Agents (JERP), which updates a long-term experiential-rule pool and the policy from the same interaction trajectories. At decision time, JERP retrieves task-relevant rules and conditions the agent on them together with the interaction history. After each episode, it uses the collected trajectories both to optimize the policy and to revise the rule pool by comparing current rollouts with reference successful trajectories. This coupling keeps the rule pool aligned with the evolving policy while allowing stable and effective behaviors to be gradually absorbed into the model itself. Experiments on AlfWorld and WebShop show that JERP yields consistent gains in decision performance for complex interactive tasks.

## 1. 论文的核心问题与整体含义（研究动机和背景）

论文聚焦于**大型语言模型（LLM）智能体在多步交互环境中的经验利用问题**。当前主流方法将交互经验分为两种用途：
- **外部规则存储**：将经验提炼为自然语言规则，用于后续提示（如 Reflexion、ExpeL、AutoGuide）。这类方法可解释性强，但规则会随着策略演化而变得过时甚至误导。
- **模型参数更新**：利用轨迹和反馈直接优化策略参数（如 RLHF、GRPO、DPO）。这类方法能全局改进行为，但在稀疏奖励环境下对局部错误的纠正能力有限。

二者的分离导致：规则池无法随策略演化而动态调整，而参数更新又缺乏对局部重复性错误的及时修正。论文提出**JERP（Joint Learning of Experiential Rules and Policies）**，旨在**联合学习经验规则和策略**，使同一交互轨迹同时服务于规则池更新和策略优化，实现两者的动态对齐。

## 2. 论文提出的方法论

### 核心思想
JERP 维护一个按任务划分的**长期经验规则池**（每个任务实例 d 对应一个规则池 K(d)），池中每条规则包含文本内容和效用分数。在每轮交互开始时，根据效用分数选取 top-k 规则构成**工作规则集** $\tilde{K}(d)$，与任务描述、历史观察一起构成输入提示 $x_t$。决策时，策略 $\pi_\theta$ 基于该提示生成动作。每轮结束后，收集的 N 条完整轨迹同时用于：
- **模型参数更新**：采用组相对优势估计（类似 GRPO）优化策略参数。
- **规则池更新**：通过对比当前轨迹与参考成功轨迹，生成五种规则编辑操作（ADD/EDIT/UPVOTE/DOWNVOTE/MERGE），并调整效用分数，超出容量时剪枝。

### 关键技术细节
1. **部分可观测马尔可夫决策过程（POMDP）建模**：状态 $s_t$ 不可观测，策略基于历史 $h_t = (o_0, a_0, ... , o_t)$ 和检索规则 $\tilde{K}(d)$。
2. **轨迹采样**：从旧策略 $\pi_{\theta_{old}}$ 采样 N 条轨迹，计算每条轨迹的结束奖励 $R(\tau_i)$（仅在终止时返回，稀疏奖励）。
3. **策略优化（Stage I）**：
   - 组内标准化奖励：$\mu_d, \sigma_d$ 为组内均值和标准差，优势 $A_i = (R(\tau_i) - \mu_d)/(\sigma_d + \delta)$。
   - 所有时间步共享同一优势 $A_{i,t} \equiv A_i$。
   - 优化目标 $L_{GRPO}$ 包含裁剪的重要性采样项和 KL 散度正则项（公式 (20)-(23)）。
4. **规则池更新（Stage II）**：
   - 输入当前规则池 $K(d)$、当前轨迹组 $T_d$、参考成功轨迹 $T^+(d)$。
   - 利用冻结参数的 LLM 生成结构化编辑操作：ADD（添加）、EDIT（修改文本）、UPVOTE/DOWNVOTE（增减效用分数）、MERGE（合并相似规则）。

5. **训练循环**（Algorithm 1）：每个训练步采样一个任务实例，构建工作规则集，采样 N 条轨迹，执行参数更新和规则更新，最后将成功的轨迹加入 $T^+(d)$。

## 3. 实验设计

### 数据集 / 场景
- **AlfWorld**：文本家庭操控环境，包含6种任务类型：Pick（拾取放置）、Clean（清洁后放置）、Heat（加热后放置）、Cool（冷却后放置）、Look（观察物体）、Pick2（取两个物体）。评价指标为任务成功率（%）。
- **WebShop**：在线购物交互环境，基于用户需求进行搜索、浏览、比较、购买。评价指标为平均分数（Score）和任务成功率（%）。

### 对比方法（Baselines）
- **Vanilla LLM**：直接提示基座模型，无推理或记忆。
- **ReAct**：推理与行动交替进行，但无跨回合经验。
- **Reflexion**：反思型记忆，将失败经验写为自然语言用于后续提示。
- **RLOO (+LoRA)**：REINFORCE 风格策略优化，使用留一基线估计优势。
- **GRPO (+LoRA)**：组相对策略优化，无独立价值模型，JERP 的参数更新部分与之类似。

所有强化学习方法均使用 LoRA 进行参数高效微调。

### 实验结果
- JERP 在 AlfWorld 总体成功率为 **61.5%**，高于 GRPO（57.8%）和 RLOO（48.7%），尤其在 Clean、Heat、Cool、Pick2 等长序列、多约束任务上提升明显。
- 在 WebShop 上，JERP 平均分数 **79.0**，成功率 **64.1%**，显著高于 RLOO（57.8%）和 GRPO（56.2%）。
- 消融实验：固定规则池（只初始更新一次，之后不更新）的变体表现明显低于完整 JERP，说明动态规则池更新是关键。

## 6. 论文的主要结论与发现

- **联合学习规则与策略能有效提升 LLM 智能体在多步交互任务中的表现**，尤其在需要长序列操作和满足多个中间约束的任务上。
- **动态规则池更新是性能提升的关键**：固定规则池的变体性能随训练停滞，而 JERP 持续改进。
- **规则池与策略的耦合**：规则提供了对局部重复性错误的及时纠正，策略参数则逐步吸收稳定有效的全局行为，两者相辅相成。
- JERP 在 AlfWorld 和 WebShop 上均优于仅进行参数更新的 GRPO 和 RLOO，也优于仅存储外部经验的 Reflexion。
