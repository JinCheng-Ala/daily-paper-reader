## 摘要
大型语言模型（LLM）代理正日益通过代理技能（Agent Skills）进行扩展，这些可复用工件将自然语言元数据、程序化指令和执行时资源打包以供运行时使用。随着开源技能市场的扩大，用户和代理越来越依赖简短的元数据来选择第三方技能，这使得难以检测技能描述与其真实行为之间的不一致性，我们称之为跨层失调问题。为解决此问题，我们提出了渐进式负载感知分层对比学习（PL-HCL），这是一个基于LLM的框架，通过建模代理技能的分层结构并学习跨层一致性来检测失调。利用超过264,000个开源技能的规范化语料库和一个经过人工验证的挑战集，PL-HCL将Macro-F1从大约0.45（未适应基线）提升至0.87-0.89（跨评估的LLM骨干网络）。该方法为用户和运营商提供了一种有效的筛选工具，并为检测分层数字工件中的不一致性提供了设计原则。

## Abstract
Large language model (LLM) agents are increasingly extended through Agent Skills, reusable artifacts that package natural-language metadata, procedural instructions, and execution-time resources for runtime use. As open-source skill marketplaces expand, users and agents increasingly rely on brief metadata to select third-party skills, making it difficult to detect inconsistencies between a skill's description and its true behavior, a problem we call cross-layer misalignment. To address this issue, we propose Progressive Loading-Aware Hierarchical Contrastive Learning (PL-HCL), an LLM-based framework that detects misalignment by modeling the layered structure of Agent Skills and learning cross-layer consistency. Using a normalized corpus of over 264,000 open-source skills and a human-verified challenge set, PL-HCL improves Macro-F1 from approximately 0.45 for unadapted baselines to 0.87-0.89 across evaluated LLM backbones. This approach offers an effective screening tool for users and operators, as well as design principles for detecting inconsistencies in layered digital artifacts.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：大语言模型（LLM）代理通过可复用的 **Agent Skills**（由元数据、指令、资源等组成的分层工件）扩展能力。用户和代理在安装或执行前通常仅依赖简短的元数据（如技能名称、描述）来选择技能，但这些**表面描述可能与底层指令或资源所体现的真实行为不一致**——这种不一致称为 **跨层失调**。
- **风险**：失调可能表现为恶意隐藏（如声称无害但实际含注入攻击）或良性夸大（如描述的能力未实现）。现有评估主要关注执行后任务性能或安全漏洞，缺乏**预执行阶段**对技能工件内部一致性的检查。
- **研究意义**：通过建模技能的分层结构并学习跨层一致性，可帮助用户、市场和安全分析师在加载或执行前筛选出不一致的技能，提升代理生态的信任度。

## 6. 主要结论与发现

1. **基础LLM检测失调能力差**：即使Accuracy接近0.81（对齐类多数基线），F1_m低于0.15，说明模型倾向于将技能预测为对齐，无法捕捉不一致。
2. **CPT提升有限**：CPT改善了零样本下的F1_m（从0.03升至0.13），但宏F1仍仅0.51左右，不足以作为可靠检测器。
3. **PL-HCL带来大幅提升**：
   - Llama-3.1-8B：Macro-F1从0.52提升至0.872（两样本），F1_m 从0.145升至0.788。
   - Foundation-Sec-8B：Macro-F1从0.522提升至0.889，F1_m 从0.150升至0.817。
4. **泛化到真实失调**：PL-HCL训练仅使用合成负样本，但挑战集上的增益表明学到的“描述-证据”一致性信号能迁移到自然失调样本。
5. **超越恶意关键词检测**：案例显示PL-HCL能检测善意的夸大描述（如声称TypeScript断言函数实际仅curl外部URL），而基线LLM误判为对齐。

## 7. 优点

- **问题定义新颖**：首次将Agent Skills的预执行跨层一致性作为独立评估目标，填补了现有评估（只关注运行时性能或单层安全）的空白。
- **数据构建扎实**：构建了大规模开源技能语料（264k+）和人工验证挑战集，为后续研究提供基础。
- **方法设计巧妙**：
  - 利用技能的分层结构设计课程式CPT和层次对比学习。
  - 仅用合成负样本（交换和扰动）训练，却能在真实失调数据上泛化，说明方法捕捉到了深层一致性信号。
- **实验分析深入**：不仅报告聚合指标，还通过个体案例揭示模型的决策模式，证明PL-HCL学会了“主张 vs 证据”的对比而非仅识别不良关键词。
- **实践应用明确**：讨论了为市场、用户、安全分析师三类利益相关者提供的具体功能（预发布筛查、客户端扫描、大规模检索）。

## 8. 不足与局限

- **仅预执行评估**：无法检测仅在运行时暴露的动态负载、环境副作用或后期变异，需要与运行时沙箱互补。
- **数据来源单一**：主要基于SkillsMP和GitHub开源技能，可能不适用于封闭生态系统、企业内部技能或动态生成技能，存在分布偏移风险。
- **模型规模限制**：实验仅在8B参数模型上（Llama-3.1-8B等），未评估更大（如70B）或更小模型的性能，可能无法完全推广。
- **挑战集规模有限**：1,444个样本中仅294个失调，可能不足以充分评估模型在罕见失调模式上的能力，存在过拟合或评估噪声。
- **合成负样本的潜在偏倚**：交换和扰动方式可能无法覆盖真实失调的所有模式（如指令层隐藏恶意但元数据声称合法），可能导致训练时的偏差。
- **未进行跨生态迁移实验**：未验证模型是否可以直接应用于其他类型的分层数字制品（如MCP服务器、浏览器扩展等），虽然文中讨论了原理性类似，但缺少实证。
