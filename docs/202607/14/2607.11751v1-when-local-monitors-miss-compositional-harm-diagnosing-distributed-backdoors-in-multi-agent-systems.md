## 摘要
随着多智能体、使用工具的LLM系统被部署，一个常见的安全网是运行时监控器，它独立检查每条消息、工具调用或步骤。我们证明这个网存在一个根本性漏洞。分布式后门将有害负载分割到多个智能体上，因此每个局部检查都通过，而组装后的对象却是攻击。监控器可能在每一步都是正确的，但仍然遗漏攻击。问题不在于分割本身：分割后的片段仍然可能泄露可疑的标记或溯源边。困难的情况是“局部良性”。没有片段携带危害，剩下的看起来像普通的良性流量。我们将其形式化为一个“可观测性边界”：监控器只能捕捉其视角中能与良性流量区分开的内容。我们证明，一旦片段在监控视图中看起来是良性的，任何基于该视图的检测器都无法捕捉它们，无论其多么强大。通过一个受控测试平台、一个外部基准测试以及端到端的智能体运行，局部监控器随着局部证据的消失而完全丢失信号，并且只有当监控器看到组装后的对象时信号才恢复。仅在良性流量上训练的监控器能够恢复攻击在未见过编码中的代码结构（平均AUROC 0.874）。给定编码家族的解码视图门控可以阻止所有测试的攻击。但看到更多并不够：完整轨迹的监控器和解码器仍然失败，除非它们到达负载暴露的表示层。当危害是组合性的时，局部安全并非全局安全，而开放问题正是如何找到那个表示层。

## Abstract
As multi-agent, tool-using LLM systems are deployed, a common safety net is a runtime monitor that checks each message, tool call, or step on its own. We show this net has a fundamental hole. A distributed backdoor splits a harmful payload across agents, so every local check passes while the assembled object is the attack. The monitor can be right on every step and still miss the attack. The problem is not splitting itself: split fragments can still leak suspicious tokens or provenance edges. The hard case is \emph{local benignness}. No fragment carries the harm, and what is left looks like ordinary benign traffic. We formalize this as an \emph{observability boundary}: a monitor catches only what its view can tell apart from benign traffic. We prove that once the fragments look benign in the monitored view, no detector on that view can catch them, however strong it is. Across a controlled testbed, an external benchmark, and end-to-end agent runs, local monitors lose the signal exactly as local evidence disappears, and it returns only when the monitor sees the assembled object. A monitor trained only on benign traffic recovers the attack's code structure across held-out encodings (0.874 mean AUROC). A decoded-view gate, given the encoding family, blocks every tested attack. But seeing more is not enough: full-trace monitors and decoders still fail unless they reach the representation where the payload is exposed. Local safety is not global safety when harm is compositional, and the open problem is finding that representation.

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：在多智能体、工具使用型LLM系统中，运行时监控器通常逐个检查每条消息、工具调用或步骤，认为只要每个局部步骤安全，整体就安全。但分布式后门攻击将有害载荷拆分成多个片段，每个片段通过本地检查，组合后形成恶意代码。这导致局部监控器在每一步都正确，却遗漏了整体攻击。
- **背景**：现有防御如逐步提示注入检测、污染/溯源追踪等均受限于“观察窗口”（即监控器能看到的信息量）。问题不是简单的拆分，而是当每个片段在监控器的视图下看起来完全“局部良性”（与正常流量不可区分）时，任何局部检测器都无法捕获攻击。
- **整体含义**：局部安全不等于全局安全。论文提出了**可观测性边界**（observability boundary）概念，形式化解释了局部监控器失效的根本原因，并指明恢复检测的唯一途径是改变监控器所观察的表示层。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程
- **核心思想**：监控器只能捕获其视图下能与良性流量区分开的内容。若攻击的每个片段在局部视图下与良性流量统计上不可区分（ε-局部不可区分），则任何基于该视图的决策规则都无检测优势。
- **关键技术细节**：
  - **形式化定义**：
    - **局部良性（Local benignness）**：每个独立片段`v(x_i)`在观测函数`v`下判断为良性（`h(v(x_i))=0`），但组装对象`C(x_1,...,x_K)`为恶意（`h(C)=1`）。
    - **ε-局部不可区分（ε-local indistinguishability）**：每个片段位置`i`上，攻击与良性分布的Total Variation距离`TV(P_{ben,i}^v, P_{att,i}^v) ≤ ε`。
  - **理论界限**：命题1证明，在ε-局部不可区分下，任何局部决策规则的单片段检测优势（TPR - FPR）≤ ε。因此，仅靠更强的局部分类器无法跨越边界。
  - **可观测性边界**：通过改变监控器的观察窗口（从单片段 → 全部片段 → 组装对象 → 解码后的表示），可以跨越边界。论文构建了从“可见边”→“局部伪影”→“局部良性”的逐步消除局部证据的测试平台（Collaborative Shadows）。
  - **监控器类型**：
    - **局部视图**：每步统计、污染/溯源。
    - **组装视图**：标记式组装（已知标记）、标记自由组装（仅用良性数据训练的一次性分类器，基于代码结构密度、部分解析性等特征）。
    - **解码视图门控**：给定编码族，解码后检查代码结构并阻止执行（作为诊断上限）。

## 6. 主要结论与发现
1. **拆分不是关键，局部良性才是**：只要每个片段在监控视图下与良性流量不可区分，任何局部检测器（包括训练后的神经检测器）都无法捕获。
2. **可观测性边界**：在劣质的局部视图上，信号随着局部证据消失而丢失（图2）；只有在监控器改变观察对象（从片段到组装后的表示）时，信号才恢复。
3. **结构恢复无需攻击样本**：仅用良性流量训练的标记自由组装监控器能从组装代码结构中恢复信号（平均AUROC 0.874），且跨未见过编码家族泛化。
4. **没有解码的广度是不够的**：完全轨迹的盲编码监控器、通用解码器集成、无编解码LLM搜索均无法抵御局部良性攻击（ASR保持在无防御水平）。只有到达解码后的表示层（给定编码族）才能完全拦截（ASR 0/50）。
5. **局部安全≠全局安全**：当危害是组合性的，监控器必须观察到成分形成时的表示层。开放问题是找到那个表示层（例如自然语言无序重组、信念、记忆等）。

## 7. 优点
- **理论贡献**：形式化定义“可观测性边界”，给出命题1的硬界限，解释为何局部检测器失败不是偶然。
- **实验设计精细**：通过可控的局部证据梯度（从可见边到局部良性）准确诊断失败原因，而非仅报告平均性能。
- **跨平台验证**：在多个测试平台（CS、AgentDojo、DemonAgent）和多个模型上复现结论，提高了泛化性。
- **指出关键方向**：将监控设计问题转化为“表示层发现”问题，为后续研究提供新视角。
- **实用导向**：虽解码视图门控需已知编码族，但标记自由组装可直接用作预警信号（无需攻击样本）。

## 8. 不足与局限
- **负载类型局限**：主要研究代码payload（Python外泄脚本），自然语言有害指令未能解决（AUROC降至0.406），且未探索其他形式的组合危害（如信念、权限等）。
- **场景假设**：实验默认K=3固定碎片，更长视野、动态K、会话中介组装、完全自适应攻击者不在主要声明范围内。
- **解码视图门控依赖性**：需预知编码族，限制了实际部署；无编解码方案（如LLM搜索）仅能捕获部分攻击（6/50），且产生大量假阳性。
- **计算资源报告不足**：未提供训练/推理的具体GPU小时数、显存需求等，不利于复现成本评估。
- **未证明通用性**：结论虽理论支撑，但实测仅覆盖有限编码族和模型；对不同架构（如闭源API模型）适用性未验证。
