## 摘要
LLM智能体日益通过外部工具执行自主行动，导致复杂且不断演变的安全风险。然而，现有的安全测试针对专家设计的安全违规，其相应结果由硬编码规则评估，这使得随着智能体演变而扩展的代价高昂。为此，我们提出Vera，一个端到端的自动化安全测试框架，通过一个三阶段自强化流水线，将软件工程测试原则实例化应用于非确定性智能体。首先，文献驱动的探索持续发现新兴风险并将其结构化为安全风险、攻击方法和工具执行环境的分类体系。其次，跨分类维度的组合构成产生可执行的安全用例，每个用例指定一个具体安全目标、一个程序化构建的初始状态，以及一个基于可观察工件的基础确定性验证谓词。第三，自适应执行在隔离沙箱中运行异构智能体，其中控制智能体基于运行时观察引导多轮交互，而基于证据的验证器从环境状态和工具调用证据（而非模型自述）判断结果。我们在四个生产级智能体框架（OpenClaw、Hermes、Codex、Claude Code）上评估Vera，揭示了严重的安全缺陷，多通道攻击下平均攻击成功率达到93.9%；我们还发布了Vera-Bench，包含跨越三个执行环境共124个风险类别的1600个可执行安全用例。这些结果表明，模块化、可执行的测试基础设施对于快速演进的智能体系统在规模上进行严格且可维护的安全评估至关重要。代码公开于https://github.com/Yunhao-Feng/Vera。

## Abstract
LLM agents increasingly perform autonomous actions through external tools, leading to complex and evolving safety risks. However, existing safety testing targets expert-designed safety violations, and the corresponding outcomes are evaluated by hard-coded rules, making them costly to extend as agents evolve. To this end, we present Vera, an end-to-end automated safety testing framework that instantiates software engineering testing principles for non-deterministic agents through a three-stage, self-reinforcing pipeline. First, a literature-driven exploration continuously discovers and structures emerging risks into taxonomies of safety risks, attack methods, and tool execution environments. Second, combinatorial composition across taxonomy dimensions produces executable safety cases, each specifying a concrete safety goal, a programmatically constructed initial state, and a deterministic verification predicate grounded in observable artifacts. Third, adaptive execution runs heterogeneous agents in isolated sandboxes where a control agent steers multi-turn interaction based on runtime observations, while evidence-grounded verifiers judge outcomes from environment state and tool-call evidence rather than model self-report. We evaluate Vera on four production agent frameworks (OpenClaw, Hermes, Codex, Claude Code), revealing substantial safety weaknesses, with average attack success rates reaching 93.9\% under multi-channel attacks; we also release Vera-Bench, comprising 1600 executable safety cases spanning 124 risk categories across three execution settings. These results indicate that modular, executable testing infrastructure is essential for rigorous and maintainable safety evaluation of rapidly evolving agentic systems at scale. The code is publicly available at https://github.com/Yunhao-Feng/Vera.

1. **严重安全缺陷**：在四个生产级agent框架上，攻击成功率（ASR）平均达90.6%（单通道）和93.9%（多通道），表明当前agent系统存在巨大安全漏洞。
2. **自适应交互的价值**：从良性变体（70.5%）到单通道（90.6%）的20.1个百分点提升，证实了适应性多轮交互能有效绕过静态防御。
3. **多通道攻击增量有限但具结构意义**：平均仅提升3.3个百分点，但揭示不同agent在用户消息通道和工具结果通道上的差异化鲁棒性（如Claude Code双通道检测更均衡，OpenClaw工具注入绕过效果更明显）。
4. **跨agent差异**：Claude Code总体ESR最高（88.6%），OpenClaw最低（70.3%），表明能力越强的agent（任务完成能力强）往往更易受攻击，存在“能力-脆弱性对齐”现象。
5. **下游分类增益**：在Vera-Bench上微调的Qwen3-Guard将F1从0.637提升至0.941，且在外部基准R-Judge上达到最高准确率61.7%，证明Vera产生的数据能有效训练跨分布泛化的安全分类器。

## 7. 优点

1. **端到端自动化**：从风险发现到执行验证全流程无需人工干预，支持持续迭代更新。
2. **可执行安全用例**：区别于抽象分类，每个用例包含可编程构建的初始状态和确定性验证器，保证可复现性和客观性。
3. **证据验证层次**：优先使用环境状态>工具调用>代理响应，避免虚假阳性（答而不做）和虚假阴性（做而不答）。
4. **自适应控制智能体**：非固定prompt模板，能根据运行时观察动态调整攻击策略，更真实地评估agent鲁棒性。
5. **模块化架构**：通过统一适配器支持异构agent框架，沙箱隔离、MCP网关、服务环境均可独立扩展。
6. **大规模覆盖**：分类体系细粒度（124+77+30），组合后生成大量可执行用例，数据密度分布合理。
7. **公开代码和基准**：代码和Vera-Bench公开，促进可复现研究。

## 8. 不足与局限

1. **良性变体ESR较低（70.5%）**：表明程序化生成的验证器可能过于严格或初始状态构造存在偏差，导致合法任务也被判为失败，可能高估了良性场景的难度。
2. **文献来源偏差**：风险探索主要依赖学术论文（arXiv/OpenReview），可能遗漏工业界未报道的漏洞或非公开攻击手法，未整合CVE、威胁情报等实时来源（虽提及可扩展性但未实施）。
3. **仅限推理时攻击**：排除了训练阶段攻击（如微调投毒、后门注入），无法覆盖模型生命周期的全环节风险。
4. **环境模拟有限**：当前只部署了5种自托管服务（邮件、代码托管、支付、IM、搜索），未涵盖真实世界中更广泛的工具生态（如云服务、数据库、RPA）。
5. **计算开销与可扩展性**：每个沙箱需要12个容器，大规模并行测试时基础设施需求高；控制智能体调用LLM增加了测试成本（中位数输入15.5万tokens，输出3千tokens），论文未讨论成本控制。
6. **跨agent对比的不公平性**：不同框架只能使用兼容的后端模型（如Codex仅支持OpenAI），模型差异可能影响ESR比较，无法完全分离框架设计与模型能力的影响。
7. **下游任务泛化有限**：在R-Judge上微调模型准确率仅61.7%，虽然领先基线但仍有大量提升空间；仅测试了一个外部基准，泛化性证据还不够充分。
8. **缺乏对拒绝模式的深入分析**：论文报告了攻击成功率，但未详细分析agent拒绝模式（如何时拒绝、拒绝后是否仍执行有害操作），可能隐藏了某些防御机制的有效性。
